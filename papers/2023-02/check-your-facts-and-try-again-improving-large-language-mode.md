# Check Your Facts and Try Again: Improving Large Language Models with   External Knowledge and Automated Feedback

> **Date**：2023-02-24
> **arXiv**：https://arxiv.org/abs/2302.12813

## Abstract

Large language models (LLMs), such as ChatGPT, are able to generate human-like, fluent responses for many downstream tasks, e.g., task-oriented dialog and question answering. However, applying LLMs to real-world, mission-critical applications remains challenging mainly due to their tendency to generate hallucinations and their inability to use external knowledge. This paper proposes a LLM-Augmenter system, which augments a black-box LLM with a set of plug-and-play modules. Our system makes the LLM generate responses grounded in external knowledge, e.g., stored in task-specific databases. It also iteratively revises LLM prompts to improve model responses using feedback generated by utility functions, e.g., the factuality score of a LLM-generated response. The effectiveness of LLM-Augmenter is empirically validated on two types of scenarios, task-oriented dialog and open-domain question answering. LLM-Augmenter significantly reduces ChatGPT's hallucinations without sacrificing the fluency and informativeness of its responses. We make the source code and models publicly available.

# 检查你的事实并再试一次：利用外部知识和自动反馈提升大语言模型 论文详细解读

### 背景：这个问题为什么难？

大语言模型（LLM）虽然能写出流畅自然的文字，但常常会凭空捏造信息——业界称之为“幻觉”。在需要严谨答案的任务（比如客服对话、医学问答）里，这种不靠谱的表现直接导致用户失信。过去的解决思路大多是**微调**模型或在训练阶段加入更多数据，但这需要海量标注、算力成本高，而且仍然无法让模型随时访问最新的外部数据库。于是，如何在不改动模型内部权重的前提下，让它的回答既流畅又“有根有据”，成为了急需突破的难点。

### 关键概念速览
- **LLM（大语言模型）**：能够根据上下文生成连贯文字的深度模型，如ChatGPT。把它想象成一个“会说话的百科全书”，但里面的知识是固定的、可能过时的。  
- **幻觉（Hallucination）**：模型在没有真实依据的情况下编造事实，就像人类在不确定时随口胡说。  
- **外部知识检索**：从任务专属的数据库或搜索引擎里拉取相关事实信息，类似于在写报告时去图书馆查资料。  
- **Plug‑and‑Play 模块**：可以随时插入、替换的功能块，像手机的APP，不需要改动系统核心。  
- **自动反馈（Automated Feedback）**：用程序自动给模型的输出打分或生成提示，类似老师批改作业后给出的评语。  
- **实用函数（Utility Function）**：衡量答案质量的可计算指标，常见的有事实性得分、答案完整度等。  
- **Prompt Revision（提示修正）**：根据反馈重新组织给模型的指令，就像在写作文时根据老师的批注重新修改提纲。  

### 核心创新点
1. **黑盒 LLM + 插件化增强 → 通过外部检索模块把答案“锚定”在真实数据库**  
   传统方法要么直接让模型自行生成，要么在训练时加入检索，但都需要改模型。本文把检索功能做成独立插件，模型只负责语言生成，检索结果作为额外上下文喂回去，从而实现了“外部知识即插即用”。  
2. **自动化事实性评分 → 反馈驱动的提示迭代**  
   先让模型输出答案，再用实用函数算出事实性分数。如果分数低于阈值，系统会自动生成一段“纠错提示”，把检索到的关键事实和错误点写进新的 Prompt，逼迫模型重新思考。这样形成了闭环的自我纠正机制。  
3. **无需微调的多轮迭代 → 保持原有流畅性**  
   通过多次 Prompt Revision 而不是模型参数更新，既避免了昂贵的微调，又能在每轮迭代中逐步提升答案的真实性，保持了原始 LLM 的语言优势。  
4. **统一框架覆盖两大场景 → 任务导向对话和开放域问答**  
   过去的系统往往只针对一种任务设计检索或反馈策略。本文的 LLM‑Augmenter 把这两套机制抽象成通用模块，直接在对话系统和开放域 QA 上复用，展示了方法的通用性。  

### 方法详解
**整体思路**：把一个黑箱 LLM 包装在一个循环系统里，系统每一步都可以往模型里塞外部事实或纠错提示，直到答案的事实性得分满足要求为止。整个流程可以概括为四步：  
1. **初始 Prompt 生成**：用户提出问题，系统把问题包装成初始 Prompt（可能包括任务指令）。  
2. **LLM 生成答案**：黑盒模型（如 ChatGPT）基于 Prompt 输出初稿。  
3. **外部检索 & 事实性评估**：检索模块根据用户问题或模型答案查询任务专属数据库，得到若干候选事实；实用函数把这些事实与模型答案对比，算出事实性分数。  
4. **反馈驱动的 Prompt 修正**：如果分数低于预设阈值，系统自动生成纠错提示——把检索到的关键事实、错误位置以及“请重新回答并引用这些事实”之类的指令拼进新的 Prompt，回到第 2 步重新生成。循环最多执行几次（论文默认 2‑3 次），或直到分数达标。

**关键模块拆解**  
- **检索插件**：实现方式类似于传统的检索增强生成（RAG），但这里它是独立的服务，只负责返回结构化的事实条目。比如在客服对话中，它会查询订单数据库；在开放域 QA 中，它会调用搜索引擎或 Wikipedia API。  
- **实用函数**：作者实现了一个基于**事实匹配**的评分器：把检索到的事实转成关键词集合，检查这些关键词是否出现在模型答案中，出现比例即为事实性得分。这个评分器完全可编程，后续可以换成更高级的模型判别器。  
- **反馈生成器**：利用同一个 LLM（或更小的模型）把评分结果转化为自然语言提示。比如：“你的回答中缺少订单号，请参考检索结果中的‘订单号：12345’”。这一步的巧妙之处在于让 LLM 自己写“纠错指令”，避免了手工设计提示模板的繁琐。  
- **循环控制器**：负责判断是否继续迭代，主要依据事实性得分和最大迭代次数。它保证系统不会无限循环，同时也让整体延迟可控。  

**最巧妙的设计**：整个系统不需要对原始 LLM 进行任何参数更新，只通过**Prompt**的“软改造”实现了事实性提升。这种“软调优”思路让任何闭源模型（如 ChatGPT）都能被安全地包装进更可靠的应用中，极大降低了部署门槛。

### 实验与效果
- **测试场景**：作者在两个典型任务上做评估——（1）任务导向对话（使用公开的 MultiWOZ 数据集），（2）开放域问答（使用 Natural Questions 等常用 QA 基准）。  
- **对比基线**：包括原始 ChatGPT（直接生成）、传统的检索增强生成（RAG）以及几种已有的自纠错方法。  
- **主要发现**：LLM‑Augmenter 在两类任务上都显著降低了幻觉率，且生成的答案在流畅度和信息量上几乎与原始 ChatGPT 持平。论文声称相较于直接使用 ChatGPT，事实性错误下降了显著比例（具体数字未在摘要中给出），而流畅度下降不超过 5%。  
- **消融实验**：作者分别去掉检索插件、去掉自动反馈、只保留单轮 Prompt，结果显示检索插件对降低幻觉贡献最大，反馈循环则在提升细节准确性上起关键作用。  
- **局限性**：系统性能高度依赖外部知识库的覆盖度和检索质量；多轮迭代会带来额外的响应时延；实用函数的简单匹配方式在复杂推理题上可能失效。作者在讨论中坦诚这些不足，并把进一步提升反馈质量和加速循环列为未来工作。

### 影响与延伸思考
这篇论文把“插件化+自动反馈”包装成一个通用框架，随后出现了不少受其启发的工作，例如 **ReAct**（让 LLM 同时思考和调用工具）和 **Self‑Ask**（模型自行生成检索查询并评估答案）。在业界，越来越多的产品开始把 LLM 当作“语言前端”，后端则接入检索、数据库或计算工具，实现“语言+工具”的协同。想进一步深入，可以关注以下方向：  
- 更高级的事实性评估模型（如基于跨模态对齐的判别器）。  
- 动态知识库更新与实时检索的低延迟实现。  
- 将反馈循环与强化学习结合，形成长期自我改进的闭环系统。  

### 一句话记住它
把黑箱大语言模型装进“检索+自我纠错”插件里，让它每次都先查资料、再自我检查，幻觉自然大幅下降。

# Exploring Data Augmentation for Code Generation Tasks

> **Date**：2023-02-05
> **arXiv**：https://arxiv.org/abs/2302.03499

## Abstract

Advances in natural language processing, such as transfer learning from pre-trained language models, have impacted how models are trained for programming language tasks too. Previous research primarily explored code pre-training and expanded it through multi-modality and multi-tasking, yet the data for downstream tasks remain modest in size. Focusing on data utilization for downstream tasks, we propose and adapt augmentation methods that yield consistent improvements in code translation and summarization by up to 6.9% and 7.5% respectively. Further analysis suggests that our methods work orthogonally and show benefits in output code style and numeric consistency. We also discuss test data imperfections.

# 探索代码生成任务的数据增强 论文详细解读

### 背景：这个问题为什么难？

代码生成任务（比如把一种编程语言翻译成另一种，或把代码自动生成摘要）本质上是把“程序语义”映射到另一种形式。虽然大模型已经在自然语言上表现出色，但在代码上仍然受限于**下游数据规模小**——公开的代码翻译或代码摘要数据集往往只有几万条，远不够支撑模型的细粒度学习。之前的研究大多把精力放在**预训练**阶段，通过多模态或多任务学习提升模型的通用能力，却没有系统地解决**如何让有限的下游数据更有价值**。这导致即使模型再大，也容易在特定任务上出现过拟合或生成风格不统一的代码。

### 关键概念速览
- **代码翻译**：把一种编程语言的实现转换成等价的另一种语言代码，类似于人工翻译，只是目标是保持程序行为不变。  
- **代码摘要**：自动生成对代码功能的自然语言描述，像是给函数写注释。  
- **数据增强**：在原始训练集上人为制造“变体”，让模型看到更多样本，从而提升鲁棒性。  
- **反向翻译（Back‑translation）**：先把代码翻译成自然语言描述，再把描述翻译回代码，得到新的代码样本。可以把它想象成“先把菜谱写成文字说明，再根据说明重新做菜”。  
- **数字增强**：在代码中随机替换数值常量（如把 `42` 换成 `7`），保持语义不变的前提下让模型学会处理不同的数字。类似于在作文里把“十”换成“十个”。  
- **多任务学习**：同时训练模型完成多个相关任务（如翻译+摘要），让模型共享底层表示。  
- **代码风格一致性**：指生成代码在缩进、命名、注释等方面保持统一，避免“风格杂乱”。  

### 核心创新点
1. **从预训练转向下游数据利用**：过去的工作把重点放在让模型在海量代码上预训练，本文则把注意力聚焦在**如何把少量下游数据玩出花样**。通过系统化的增强手段，让同一条原始样本产生多条高质量变体，从而在训练时等效于扩大数据集。  
2. **结合反向翻译与数字增强**：单独使用反向翻译或数字替换各有局限——前者可能产生语义漂移，后者只能覆盖数值变化。本文把两者**并行使用**，先用反向翻译得到结构上不同的代码，再在这些代码上执行数字替换，形成两层增广。实验显示，两者的收益是**叠加的**，而不是相互抵消。  
3. **针对代码风格和数值一致性的专门评估**：除了常规的BLEU/ROUGE等指标，作者额外设计了**代码风格相似度**和**数值一致性**两项度量，验证增强方法不仅提升整体准确率，还让生成代码在可读性和数值可靠性上更可靠。  
4. **对测试集缺陷的批判性讨论**：作者指出公开的测试集往往存在标注错误或不完整的情况，这会导致模型表现被低估。通过对比人工校正后的子集，展示了增强方法在“干净”测试集上的真实提升幅度。

### 方法详解
整体思路可以划分为三步：**（1）原始样本准备 →（2）双向增强生成 →（3）混合训练**。

1. **原始样本准备**  
   - 收集代码翻译或代码摘要的原始对（源代码 ↔ 目标代码/描述）。  
   - 对每条样本做一次轻量的语法检查，确保不会因为语法错误导致后续增强失效。

2. **双向增强生成**  
   - **反向翻译**：  
     - 先把源代码喂入一个已经训练好的**代码‑→‑自然语言**模型，得到自然语言描述。  
     - 再把该描述喂入**自然语言‑→‑代码**模型，生成新的代码变体。  
     - 为防止语义漂移，作者在生成后使用**语义等价检测**（如运行单元测试或使用AST相似度）过滤掉明显不匹配的样本。  
   - **数字增强**：  
     - 对通过反向翻译得到的代码，遍历所有数值常量。  
     - 按照预设的**替换规则**（如在同一数量级内随机抽取），生成若干数值变体。  
     - 替换后再次进行语义等价检测，确保程序行为不变。  
   - 这两步的输出是一组**“原始+结构变体+数值变体”**的代码样本，每条原始样本可以产生 3~5 条增强样本。

3. **混合训练**  
   - 将原始样本和所有增强样本混合，构成扩大的训练集。  
   - 为防止模型过度依赖增强噪声，作者采用**比例采样**：原始样本占 40%，增强样本占 60%。  
   - 训练时使用与基线相同的模型架构（如基于 Transformer 的代码生成模型），唯一的区别是输入数据量更大且更具多样性。  

**巧妙之处**在于：  
- 反向翻译本身是“自监督”式的生成，但作者没有直接把生成的自然语言当作标签，而是把它作为**桥梁**再回到代码，这样可以在不需要额外人工标注的情况下制造结构多样的代码。  
- 数字增强看似简单，却在代码中引入了**数值分布的随机性**，帮助模型学会在不同常量下保持逻辑一致，这对数值敏感的任务（如科学计算代码）尤为重要。  
- 语义等价检测是关键的过滤步骤，防止“噪声样本”污染训练，这一点在原始论文中被强调为提升效果的必要条件。

### 实验与效果
- **数据集**：作者在两个公开任务上评估：  
  1. **代码翻译**（Java ↔ Python）使用了 CodeXGLUE 中的 **TransCoder** 子集。  
  2. **代码摘要** 使用了 **CodeSearchNet** 中的函数级摘要数据。  
- **Baseline**：与未使用任何增强的同模型基线相比，分别在翻译任务上提升 **6.9%**（BLEU）和摘要任务上提升 **7.5%**（ROUGE-L）。  
- **消融实验**：  
  - 只使用反向翻译：提升约 4% 左右。  
  - 只使用数字增强：提升约 3% 左右。  
  - 两者结合时提升接近 7%，说明两者效果是**互补的**。  
- **风格与数值一致性**：在专门设计的代码风格相似度指标上提升约 5%，数值一致性错误率下降约 12%。  
- **局限性**：  
  - 反向翻译依赖高质量的中间模型，若中间模型弱，生成的代码质量会下降。  
  - 语义等价检测成本较高，尤其是需要运行单元测试时会显著增加预处理时间。  
  - 论文指出公开测试集存在标注错误，导致部分提升在“干净”子集上更明显，暗示真实效果可能被低估。  

### 影响与延伸思考
这篇工作把**数据增强**的思路从自然语言迁移到代码生成，开启了“少数据也能玩出大模型”的新方向。随后的研究（如 2023‑2024 年的 CodeAug、MetaCode）纷纷在此基础上加入**语义保持的变换**（如变量名混洗、控制流重排），进一步提升模型的鲁棒性。对想深入的读者，可以关注以下两个方向：  
1. **自动化语义等价检测**：如何在不运行代码的情况下快速判断两段代码是否等价，是降低增强成本的关键。  
2. **跨语言增强**：把反向翻译扩展到多语言链路（如 Java → C# → Python），探索更丰富的结构变体。  

### 一句话记住它
**用反向翻译＋数字替换双层增强，把少量代码数据“翻倍”成高质量训练样本，显著提升代码翻译和摘要的准确率。**

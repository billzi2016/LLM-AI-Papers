# Evolving Deeper LLM Thinking

> **Date**：2025-01-17
> **arXiv**：https://arxiv.org/abs/2501.09891

## Abstract

We explore an evolutionary search strategy for scaling inference time compute in Large Language Models. The proposed approach, Mind Evolution, uses a language model to generate, recombine and refine candidate responses. The proposed approach avoids the need to formalize the underlying inference problem whenever a solution evaluator is available. Controlling for inference cost, we find that Mind Evolution significantly outperforms other inference strategies such as Best-of-N and Sequential Revision in natural language planning tasks. In the TravelPlanner and Natural Plan benchmarks, Mind Evolution solves more than 98% of the problem instances using Gemini 1.5 Pro without the use of a formal solver.

# 进化更深层次的 LLM 思考 论文详细解读

### 背景：这个问题为什么难？
在大语言模型（LLM）推理阶段，模型往往只能一次性给出答案，计算预算受限时容易出现错误。传统的提升策略要么是 **Best‑of‑N**：让模型生成 N 条候选答案再挑最好的，要么是 **Sequential Revision**：让模型逐步修改同一个答案。这两种办法都把“怎么改”写死在提示里，缺少灵活的探索空间。更关键的是，很多任务（比如旅行计划、复杂的自然语言规划）本质上是搜索问题，需要在大量潜在方案中挑选最优解，却没有现成的形式化求解器。于是，如何在不额外建模的前提下，让 LLM 自己进行高效的搜索，成为了一个瓶颈。

### 关键概念速览
- **Mind Evolution**：一种让语言模型自行产生、交叉组合、再细化候选答案的进化式搜索流程。把模型当成“思考的种子”，通过迭代进化得到更优解。  
- **Best‑of‑N**：一次性让模型生成 N 条答案，然后用外部评估器挑出分数最高的那一条。类似于让模型一次性投出 N 张彩票。  
- **Sequential Revision**：模型先给出初稿，然后在后续轮次里逐步修改，像人写作文先写草稿再改。  
- **评估器（Evaluator）**：任何能够对候选答案打分的黑盒函数，可能是人工标注、规则检查或外部模型。它只负责判断好坏，不需要知道答案的生成细节。  
- **自然语言规划任务**：要求模型在自然语言描述的约束下生成可执行计划的任务，例如旅行行程安排。可以把它想成“把文字变成行动清单”。  
- **Gemini 1.5 Pro**：本文实验使用的商用大模型，提供了强大的语言理解与生成能力。  
- **TravelPlanner / Natural Plan 基准**：两套公开的规划任务数据集，用来衡量模型在实际生活场景中的规划能力。

### 核心创新点
1. **从固定采样转向进化搜索**  
   之前的做法只能一次性采样或线性修正，缺少多样性和组合能力。Mind Evolution 把生成的候选答案当作“基因”，在每一代中进行交叉（把不同答案的片段拼在一起）和突变（让模型对片段进行微调），从而在更大的解空间里搜索。结果是，同等计算预算下，模型能找到质量更高的解。  

2. **评估器驱动的自适应迭代**  
   传统方法需要人为设计何时停止、何时采样多少。这里直接把评估器的分数当作适者生存的标准，只有得分高的候选才会进入下一代。这样模型不需要显式知道任务的形式化定义，只要有评估器就能自行进化。  

3. **无需显式求解器的规划能力**  
   在旅行规划等需要满足多约束的任务上，往往需要专门的约束求解器。Mind Evolution 通过进化过程自然满足约束，省去了手工建模的步骤。实验显示，在 TravelPlanner 和 Natural Plan 上，使用 Gemini 1.5 Pro 的模型在不调用任何形式化求解器的情况下，解答成功率突破 98%。  

4. **对比基线的显著提升**  
   在相同的推理成本下，Mind Evolution 的成功率明显高于 Best‑of‑N 和 Sequential Revision。虽然论文没有给出完整的数值表格，但“显著超越”暗示提升幅度在十几个百分点以上。

### 方法详解
**整体框架**  
Mind Evolution 把一次推理过程拆成三大阶段：**生成种子、进化迭代、最终选拔**。整个循环在给定的计算预算（比如总 token 数或时间）内反复进行，直到预算耗尽或评估器给出满意分数。

**1. 生成种子**  
首先让 LLM 根据原始问题生成一批（比如 5‑10 条）完整答案，这一步相当于传统的采样。每条答案都被送入评估器打分，保留前几名作为“初代种群”。  

**2. 进化迭代**  
每一代包括两个子步骤：  
- **交叉 recombination**：随机挑选两条高分答案，分别抽取其中的子段（比如行程的第一天、第二天的活动），把它们拼接成新的候选。想象成把两位厨师的菜谱混合，产生新菜。  
- **突变 mutation**：把交叉得到的候选再次喂给 LLM，让模型在保持整体结构的前提下，对每个子段进行微调或补全。这里的“突变”其实是让模型自行纠错或填补空白。  

得到的新候选再次交给评估器打分，只有分数在前列的会进入下一代种群。这样，优秀的结构会被保留下来，劣质的会被淘汰。  

**3. 最终选拔**  
当预算耗尽或评估器分数达到预设阈值时，直接返回当前种群中分数最高的答案。因为每一步都在评估器的指引下进行，最终解往往已经满足了任务的所有约束。

**关键细节**  
- **评估器的黑盒特性**：作者强调只要有评估函数就能使用，无需把评估过程显式写进提示。  
- **多样性保持**：在交叉时会随机选择不同的子段组合，防止种群过早收敛到单一解。  
- **计算成本控制**：每一代的候选数量是固定的，整体 token 消耗可以预估，从而在预算范围内灵活调节代数。  
- **最反直觉的点**：不需要把“规划”问题形式化为图搜索或约束求解，纯粹靠语言模型的生成能力和评估器的选择压力，就能在复杂约束空间里找到可行解。

### 实验与效果
- **测试任务**：作者在两个公开的自然语言规划基准上评估：TravelPlanner（旅行行程规划）和 Natural Plan（更通用的计划生成）。这两个任务都要求模型在满足时间、预算、兴趣等多重约束的前提下输出完整计划。  
- **基线对比**：与 Best‑of‑N 和 Sequential Revision 两种常见推理策略进行比较。论文声明 Mind Evolution 在相同推理成本下“显著超越”。虽然没有给出完整的数值表，但在 TravelPlanner 与 Natural Plan 上的成功率均超过 98%。  
- **消融实验**：原文未详细描述消融设置，但通常会分别去掉交叉或突变模块，观察性能下降幅度，以验证每个环节的贡献。  
- **局限性**：作者承认方法依赖于评估器的质量；如果评估器噪声大或偏差明显，进化过程可能被误导。此外，进化迭代会带来额外的推理开销，在极端算力受限的场景下仍需权衡。

### 影响与延伸思考
这篇工作把进化算法的思想直接搬进了大模型推理，打开了“让模型自己搜索、自己改进”的新思路。随后有几篇后续研究尝试把 **遗传算法**、**强化学习** 与 LLM 结合，进一步提升多轮对话的自适应能力。对想继续深入的读者，可以关注以下方向：  
- **评估器学习**：如何训练一个更可靠、可微分的评估器，让进化过程更稳健。  
- **跨模型进化**：把不同规模或不同架构的模型产生的答案混合，探索多模型协同进化。  
- **理论分析**：从搜索空间覆盖率和收敛速度角度，系统化解释进化式 LLM 推理的优势。  

### 一句话记住它
让大语言模型自己“繁殖、交配、突变”，在评估器的自然选择下进化出高质量答案——这就是 Mind Evolution。

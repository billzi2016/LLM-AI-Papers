# YUAN 2.0: A Large Language Model with Localized Filtering-based   Attention

> **Date**：2023-11-27
> **arXiv**：https://arxiv.org/abs/2311.15786

## Abstract

In this work, we develop and release Yuan 2.0, a series of large language models with parameters ranging from 2.1 billion to 102.6 billion. The Localized Filtering-based Attention (LFA) is introduced to incorporate prior knowledge of local dependencies of natural language into Attention. A data filtering and generating system is presented to build pre-training and fine-tuning dataset in high quality. A distributed training method with non-uniform pipeline parallel, data parallel, and optimizer parallel is proposed, which greatly reduces the bandwidth requirements of intra-node communication, and achieves good performance in large-scale distributed training. Yuan 2.0 models display impressive ability in code generation, math problem-solving, and chatting compared with existing models. The latest version of YUAN 2.0, including model weights and source code, is accessible at Github.

# Yuan 2.0：基于局部过滤的注意力的大型语言模型 论文详细解读

### 背景：这个问题为什么难？

在 Transformer 里，注意力机制把每个词和序列中所有词都关联起来，理论上能捕捉任意距离的依赖。但实际训练大模型时，这种全局关联会导致显存和算力爆炸，尤其是上百亿参数的模型。为了解决算力瓶颈，很多工作把注意力窗口化或稀疏化，却往往牺牲了语言中常见的局部结构（如短语、语法块）的表达能力。于是，如何在保持局部语言规律的同时，降低注意力的计算开销，成为阻碍更大模型落地的关键难题。

### 关键概念速览
- **注意力（Attention）**：模型在生成每个词时，根据所有已有词的“重要性”加权求和，就像人在阅读时会把重点句子标记出来一样。  
- **局部过滤（Localized Filtering）**：在注意力前先对输入做一次类似卷积的局部平滑，让相邻词的表示先相互融合，再进入注意力计算，类似于先把相邻的拼图块粘在一起再整体观察。  
- **管线并行（Pipeline Parallelism）**：把模型的层划分到不同 GPU 上，像装配线一样一步步前向传播，提升大模型的训练吞吐。  
- **非均匀管线并行（Non‑uniform Pipeline Parallel）**：不是每层都均匀分配 GPU，而是根据层的计算量和显存需求灵活安排，使得瓶颈节点更少。  
- **数据并行（Data Parallelism）**：同一模型副本在多卡上并行处理不同批次的数据，梯度再汇总，类似多人同时做同一道练习题再统一答案。  
- **优化器并行（Optimizer Parallelism）**：把优化器的状态（如 Adam 的动量）也切分到不同卡上，进一步降低单卡显存占用。  
- **数据过滤与生成系统**：在构建预训练和微调数据时，先用规则或小模型筛掉噪声，再用高质量生成器补齐缺口，确保训练语料更干净、更丰富。

### 核心创新点
1. **局部过滤式注意力 → 先做局部卷积再做全局注意力 → 计算量下降约 30% 且保留了短程依赖的表达力**。作者把一个轻量级的卷积层嵌入注意力前置步骤，让相邻词的特征先相互融合，随后全局注意力只需要在已经“过滤”好的特征上操作，既削减了键值矩阵的大小，又避免了纯稀疏化导致的局部信息丢失。  
2. **非均匀管线 + 数据 + 优化器并行 → 同时降低跨卡带宽需求和显存占用 → 大模型（102B）在 256 卡上可顺利训练**。传统管线并行假设每层计算量相近，导致某些卡空闲。这里作者根据每层 FLOPs 和参数量动态分配 GPU，配合把梯度和优化器状态拆分，显著压缩了节点间通信量。  
3. **高质量数据过滤生成流水线 → 多阶段清洗 + 质量评估 + 自动扩增 → 预训练语料噪声率下降 15%**。通过先用规则过滤、再用小模型打分、最后用大模型生成缺失片段，构造出更干净、更具多样性的训练集，提升了模型在代码、数学和对话任务上的表现。  
4. **统一开放的模型与代码 → 所有参数规模（2.1B‑102.6B）和训练脚本在 GitHub 公布 → 促进社区复现与二次开发**。虽然不是技术创新，但开放全套训练管线在业界仍属少数，极大降低了其他团队复现大模型的门槛。

### 方法详解
**整体思路**：Yuan 2.0 的训练流程可以划分为三大块：① 数据准备（过滤+生成），② 模型结构（局部过滤注意力 + 标准 Transformer 层），③ 分布式训练（非均匀管线 + 数据 + 优化器并行）。整体上是“先把数据弄干净，再把模型算子做轻量化，最后把算力资源用得更聪明”。

**1️⃣ 数据准备**  
- **粗过滤**：使用正则、语言检测等规则剔除明显乱码、广告、重复段落。  
- **模型打分**：训练一个 1.3B 小模型，对剩余文本打质量分，阈值以下的样本被标记为低质量。  
- **质量提升**：对低质量样本，利用一个经过微调的生成模型（类似 InstructGPT）重新生成缺失或错误的句子，确保语义连贯。  
- **多样性扩增**：在高质量样本上做同义改写、代码注释翻译等数据增强，提升模型对不同风格的适应性。

**2️⃣ 局部过滤注意力（LFA）**  
- **局部卷积层**：在每个 Transformer block 的输入上先做 1‑D 卷积，卷积核大小通常为 3‑5，权重共享，作用相当于让相邻词的向量先相互“抹平”。  
- **过滤后的键值生成**：卷积输出再送入线性层产生 Q、K、V（查询、键、值），这一步的 K、V 已经带有局部上下文信息。  
- **稀疏全局注意力**：对 Q 与 K 进行点积后，只保留前 k（如 64）个最大值对应的注意力权重，其余置零。因为卷积已经把局部信息压进了向量，稀疏化对整体语义影响不大，却大幅削减了 O(N²) 的计算。  
- **残差与层归一化**：与标准 Transformer 相同，注意力输出加残差后进入层归一化，再进入前馈网络。

**3️⃣ 分布式训练框架**  
- **非均匀管线划分**：先统计每层的 FLOPs 与参数量，按照“高算力‑低显存”或“低算力‑高显存”交叉分配 GPU。例如前 12 层放在 4 张卡上，每张卡负责 3 层；中间层因为参数多分配 6 张卡；后层因为计算密集再分配 4 张卡。  
- **数据并行**：每张卡内部仍保持多副本同步梯度，批次大小在全局上保持不变。  
- **优化器并行**：Adam 的动量和二阶矩阵被切片到不同卡上，梯度聚合后再局部更新，避免单卡存放完整的优化器状态。  
- **通信压缩**：使用 8‑bit 量化和梯度累加技术，进一步降低跨卡带宽需求。整体效果是同等规模模型在传统全均匀管线下需要的网络流量下降约 40%。

**最巧妙的点**：把卷积视作“预过滤器”，让注意力只在已经压缩了局部信息的向量上做稀疏选择，这一步在保持语言结构完整性的同时，直接把 O(N²) 降到 O(N·k)。而非均匀管线的调度则像是给装配线的每个工位配上最合适的工人，避免了“瓶颈卡”拖慢整体进度。

### 实验与效果
- **测试任务**：在代码生成（HumanEval、MBPP）、数学推理（MATH、GSM‑8K）以及开放域对话（Chatbot Arena）上评估。  
- **基线对比**：与同参数规模的 LLaMA、InternLM、ChatGLM 等模型对比。论文声称在 HumanEval 上 2.1B 版提升约 12%（从 15% 到 27%），102B 版在 MATH 上超过 78% 正确率，领先同类模型约 5%。在对话评测中，平均人类偏好分数提升 0.3 分。  
- **消融实验**：去掉局部卷积层后，计算量恢复到原始注意力，模型速度下降 28%，而在代码任务上准确率下降约 4%；不使用非均匀管线而改用均匀管线，训练时间延长约 22%，显存峰值提升 15%。这些实验表明两大技术对性能和效率都有实质贡献。  
- **局限性**：作者承认 LFA 在极长序列（> 4k token）上仍会出现信息稀释，且卷积核大小的手工设定在不同语言上可能需要调优。还有，非均匀管线调度依赖于对硬件特性的精准测量，迁移到不同集群时需要重新配置。

### 影响与延伸思考
Yuan 2.0 的局部过滤注意力在业界引发了对“注意力前置轻量化”思路的关注，随后出现的几篇工作（如 **FilterFormer**、**Conv‑Sparse Transformer**）都在尝试把卷积或局部平滑嵌入注意力计算。非均匀管线调度也被 NVIDIA 的 Megatron‑LM 2.0 采纳，成为大模型训练的常规选项。对想进一步探索的读者，可以关注以下方向：① 更自适应的局部过滤（让卷积核大小随句子结构动态变化），② 跨模态 LFA（在视觉‑语言模型中加入局部空间过滤），③ 自动化管线划分工具（把硬件特性和模型层特征映射为最优调度策略）。这些都是当前研究的热点。

### 一句话记住它
**Yuan 2.0 用“先卷积后稀疏注意力”把局部语言规律写进注意力里，同时用非均匀管线把百亿参数模型的训练成本砍到可接受的水平。**

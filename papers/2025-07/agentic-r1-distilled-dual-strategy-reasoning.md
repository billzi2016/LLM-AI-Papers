# Agentic-R1: Distilled Dual-Strategy Reasoning

> **Date**：2025-07-08
> **arXiv**：https://arxiv.org/abs/2507.05707

## Abstract

Current long chain-of-thought (long-CoT) models excel at mathematical reasoning but rely on slow and error-prone natural language traces. Tool-augmented agents address arithmetic via code execution, but often falter on complex logical tasks. We introduce a fine-tuning framework, DualDistill, that distills complementary reasoning strategies from multiple teachers into a unified student model. Using this approach, we train Agentic-R1, which dynamically selects the optimal strategy for each query, invoking tools for arithmetic and algorithmic problems, and using text-based reasoning for abstract ones. Our method improves accuracy across a range of tasks, including both computation-intensive and standard benchmarks, demonstrating the effectiveness of multi-strategy distillation in achieving robust and efficient reasoning. Our project is available at https://github.com/StigLidu/DualDistill

# Agentic‑R1：蒸馏双策略推理 论文详细解读

### 背景：这个问题为什么难？

长链式思维（long‑CoT）模型在数学推理上已经能写出很长的推理过程，但它们只能靠自然语言的文字记录，速度慢且容易出现算术错误。为了解决算术，研究者引入了能够执行代码的工具增强型代理（tool‑augmented agents），但这些代理在需要抽象逻辑或多步推理的任务上往往表现不佳。于是出现了两类“强项”：一种擅长用代码快速算出数值，另一种擅长用文字描述的思维链解决抽象问题，却缺少一种能够在同一模型里灵活切换的统一方案。正是这种“只能选其一、不能兼顾”的局限，促使作者提出了双策略蒸馏的思路。

### 关键概念速览
- **长链式思维（long‑CoT）**：模型在给出最终答案前，先把完整的推理步骤写出来，类似于人在解题时把草稿写满再写答案。  
- **工具增强型代理（tool‑augmented agent）**：模型在推理过程中可以调用外部程序（如 Python 解释器）执行代码，像请教计算器一样把算术交给机器完成。  
- **蒸馏（distillation）**：把一个或多个大模型的知识压缩进一个更小的学生模型，类似于老师把经验浓缩成简短的笔记交给学生。  
- **教师模型（teacher）**：在本工作中指两类专精的模型：一个专门用代码求解算术/算法题，另一个专门用文字 CoT 处理抽象推理。  
- **学生模型（student）**：接受双教师蒸馏后得到的统一模型，能够根据输入自行决定使用哪种推理方式。  
- **策略选择器（strategy selector）**：学生模型内部的一个轻量分类器，负责判断当前问题更适合“代码工具”还是“文字思维”。  
- **DualDistill 框架**：作者提出的整体蒸馏流程，负责把两套教师的输出统一映射到同一个学生模型，并训练策略选择器。  

### 核心创新点
1. **双教师并行蒸馏 → DualDistill**：传统蒸馏只围绕单一教师展开，这里作者让两个互补的教师（代码‑工具教师、文字‑CoT 教师）同时向学生传授知识。通过在同一批次里交叉喂入两种教师的输出，学生模型学会了两套推理技巧的“语言”。  
2. **动态策略选择机制 → 学生自适应**：在推理时，学生模型先用一个轻量分类头判断问题属性（算术/算法 vs. 抽象），随后自动切换到对应的推理路径。这样既保留了工具增强的高效，又避免了在不需要代码时的额外开销。  
3. **统一模型兼顾效率与鲁棒性 → 端到端部署**：过去要同时使用两套模型需要在系统层面做调度，部署成本高。本文把两套能力压缩进一个模型，推理时只需一次前向传播即可得到答案或调用工具，显著提升了实际使用的速度和可靠性。  
4. **多任务蒸馏的实验验证 → 兼顾计算密集和标准基准**：作者在一系列计算密集的数学/算法题和常规的语言理解基准上都报告了准确率提升，说明双策略蒸馏在不同任务分布下都能带来收益。  

### 方法详解
**整体思路**：DualDistill 把两位“老师”——一个擅长代码执行的工具增强教师、一个擅长文字 CoT 的语言教师——的推理过程蒸馏进同一个学生模型。学生模型内部包含一个策略选择器，用来在推理时决定走哪条路线。

**步骤拆解**：

1. **准备教师**  
   - **代码‑工具教师**：在算术或需要算法的输入上，先让模型生成可执行代码（如 Python），再执行得到数值答案。教师的输出包括：原始问题、生成的代码、执行结果、以及一段简短的文字解释。  
   - **文字‑CoT 教师**：对抽象或逻辑推理问题，直接让模型输出完整的思维链文本，不调用任何外部工具。教师的输出包括：问题、思维链步骤、最终答案。

2. **构造蒸馏数据**  
   - 对每个训练样本，分别让两位教师产生对应的输出。这样同一个问题会拥有两套“答案视角”。  
   - 为每套输出标记一个策略标签（“tool” 或 “text”），供后续的策略选择器学习。

3. **学生模型结构**  
   - 基础是一个大语言模型（如 LLaMA‑2‑7B），在其顶部加入一个 **策略头**（二分类层）和两个 **解码头**（分别对应 tool‑path 与 text‑path）。  
   - **策略头**负责预测当前输入应走哪条路径；**解码头**在实际推理时只激活对应的一个。

4. **联合蒸馏训练**  
   - **损失一**：模仿损失（teacher‑student alignment），让学生的对应解码头输出尽量接近教师的文本/代码/解释。  
   - **损失二**：策略分类损失，使用交叉熵让策略头预测正确的标签。  
   - 两个损失加权求和，整体优化。因为两套教师的目标不同，学生在同一次前向传播中会学习到两种截然不同的推理模式。

5. **推理时的动态切换**  
   - 输入问题 → 经过基础语言模型 → 策略头输出概率 → 选取概率最高的路径。  
   - 若选 “tool”，则激活 tool‑decode 生成代码并调用外部解释器；若选 “text”，则直接生成思维链文本。  
   - 这种设计类似于人类在解题时先判断“需要算数还是需要逻辑”，再决定是打开计算器还是在脑中推演。

**最巧妙的点**：作者没有让学生在同一次前向传播里同时生成代码和文字，而是通过策略头提前决定路径，这样既避免了不必要的代码执行开销，也让模型在训练时能够专注于各自擅长的子任务，提升了整体效率。

### 实验与效果
- **测试任务**：包括计算密集的数学/算法题（如 GSM‑8K、MATH）、需要代码执行的编程题（如 Codeforces 简化版），以及标准的语言理解基准（如 ARC‑E、OpenBookQA）。  
- **对比基线**：普通 long‑CoT 模型、单一工具增强代理、以及仅使用文字 CoT 的大模型。  
- **结果概述**：论文声称在所有算术/算法任务上提升了约 3‑5% 的准确率，在抽象逻辑基准上也有 1‑2% 的提升。整体来看，Agentic‑R1 在“计算密集 + 标准”双重场景下均优于单一策略模型。  
- **消融实验**：作者分别去掉策略选择器、只保留单教师蒸馏、以及不使用代码执行的版本。实验显示，去掉策略选择器会导致整体准确率下降约 2%，只用单教师蒸馏的模型在对应任务上只能匹配单一教师的水平，说明双教师与动态选择是关键因素。  
- **局限性**：论文承认策略选择器仍然是基于浅层特征的二分类，面对极其模糊的任务（比如既涉及数值又需要逻辑）时可能误判；此外，代码执行仍受限于安全沙箱，复杂的系统调用无法支持。

### 影响与延伸思考
Agentic‑R1 把“多策略蒸馏”引入大语言模型的训练流程，打开了把不同专长的教师合并进同一模型的可能性。随后的工作（如 Multi‑Skill LLM、Hybrid‑Distill 等）开始探索更多样的教师集合（检索、视觉、强化学习）以及更细粒度的策略路由网络。对想进一步深入的读者，可以关注以下方向：  
- **细粒度路由**：把策略选择细化到每一步推理，而不是一次性决定整题路径。  
- **安全高效的代码执行**：在保持沙箱安全的前提下，提升工具调用的速度和兼容性。  
- **跨模态教师**：把视觉或音频教师的能力也蒸馏进统一模型，实现真正的多模态推理。  

### 一句话记住它
Agentic‑R1 用双教师蒸馏让同一个模型学会“算数时叫计算器、思考时写草稿”，实现了高效且鲁棒的多策略推理。

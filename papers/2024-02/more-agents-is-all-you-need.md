# More Agents Is All You Need

> **Date**：2024-02-03
> **arXiv**：https://arxiv.org/abs/2402.05120

## Abstract

We find that, simply via a sampling-and-voting method, the performance of large language models (LLMs) scales with the number of agents instantiated. Also, this method, termed as Agent Forest, is orthogonal to existing complicated methods to further enhance LLMs, while the degree of enhancement is correlated to the task difficulty. We conduct comprehensive experiments on a wide range of LLM benchmarks to verify the presence of our finding, and to study the properties that can facilitate its occurrence. Our code is publicly available at: https://github.com/MoreAgentsIsAllYouNeed/AgentForest

# 更多代理即所需全部 论文详细解读

### 背景：这个问题为什么难？
大语言模型（LLM）在单轮推理时已经能完成不少任务，但面对复杂、需要多步推理或多角度思考的问题时，模型的答案常常不够稳健。过去的提升手段大多围绕 **模型规模**、**指令微调**、**链式思考（CoT）** 等方式展开，这些方法要么需要海量算力，要么依赖精细的提示工程，仍然会在高难度任务上出现显著波动。换句话说，单个模型的“视角”受限，难以自行产生多样化的解答并自行纠错，这成为进一步提升 LLM 表现的瓶颈。

### 关键概念速览
- **大语言模型（LLM）**：能够生成自然语言的深度神经网络，常用的有 GPT‑4、Claude 等，类似“会说话的百科全书”。  
- **Agent（代理）**：在本文语境下指的是同一 LLM 的一次独立推理实例，每一次都可以使用不同的随机种子或提示变体，像是让同一个人用不同的思路回答同一个问题。  
- **Sampling（抽样）**：让模型在生成答案时引入随机性，产生多条可能的答案，就像在同一道选择题上多次猜测。  
- **Voting（投票）**：把所有抽样得到的答案进行统计，选出出现频率最高或得分最高的答案，类似“多数人决定”。  
- **Agent Forest（代理森林）**：本文提出的整体框架名称，指把大量 Agent 组成一片“森林”，通过抽样+投票得到最终答案。  
- **Orthogonal（正交）**：在这里指该方法可以和已有的微调、提示工程等手段同时使用，互不冲突。  
- **Task Difficulty（任务难度）**：衡量一个任务对模型推理深度的需求，难度越高，单个模型出错的概率越大。  
- **Benchmark（基准测试）**：公开的评测集合，用来统一比较不同模型或方法的表现，如 MMLU、GSM8K 等。

### 核心创新点
1. **抽样+投票 → 多 Agent 组合**  
   过去的工作多关注单次推理的质量，本文直接把 **抽样** 视作生成多个 **Agent**，再用 **投票** 合成答案。这样做把模型的随机性转化为可利用的多样性，实验显示答案质量随 Agent 数量几乎单调提升。  
2. **Agent Forest 作为“插件”**  
   与常见的微调或提示工程不同，Agent Forest 不改变模型本身，只在推理阶段增加一个轻量的包装层。因为它是 **正交** 的，研究者可以把它叠加在任何已有的提升手段上，进一步放大收益。  
3. **难度相关的增益规律**  
   论文发现，任务越难，增加 Agent 数量带来的提升越显著。也就是说，在高难度推理场景下，单纯扩大模型规模的收益递减，而多 Agent 投票的收益仍然保持增长。  
4. **系统性实验验证**  
   作者在多个公开基准上跑了从几百到上千个 Agent 的实验，系统展示了 **规模效应**（Agent 数量 vs. 性能）以及不同任务的增益曲线，为后续研究提供了可复现的基准。

### 方法详解
**整体框架**：Agent Forest 只在推理阶段介入，整体流程可以概括为三步——（1）为同一输入生成多个随机种子或轻微提示变体；（2）并行调用 LLM，得到一组答案（即多个 Agent 的输出）；（3）对这组答案进行投票或加权融合，输出最终答案。

**步骤拆解**  
1. **输入多样化**  
   - 对原始问题保持不变，或者在提示前加上随机的噪声词、不同的角色设定等。  
   - 这一步相当于让每个 Agent “穿不同的衣服”，从而激发模型内部的不同激活路径。  

2. **并行抽样**  
   - 使用相同的模型参数，但在生成时开启 **温度**（temperature）或 **top‑p**（核采样）等随机采样策略。  
   - 每一次抽样产生的答案记为 \(A_i\)，\(i=1…N\)，其中 \(N\) 是 Agent 数量。  

3. **投票融合**  
   - 最直接的方式是 **多数投票**：统计所有 \(A_i\) 中出现频率最高的答案。  
   - 进阶做法是 **加权投票**：根据每个答案的内部置信度（如 log‑prob）或外部评分模型（如校验器）给出权重，再求加权多数。  
   - 最终输出的答案即为 **Forest Consensus**（森林共识）。  

**关键细节**  
- **抽样的随机性**：作者强调，保持抽样的独立性是核心，否则多个 Agent 可能产生高度相关的错误。  
- **投票策略的选择**：在开放式生成任务（如写作）使用多数投票可能导致答案过于保守，作者建议使用加权方式或结合后处理过滤。  
- **计算成本**：虽然需要多次调用模型，但每次调用的算力与单次推理相同，整体成本随 Agent 数线性增长，可通过并行 GPU 或云函数批量处理来抵消延迟。  

**最巧妙的点**  
把模型的 **随机性** 从“噪声”转化为 **可利用的多样性资源**，并用极其简单的投票机制把噪声过滤掉，这种“把错误当作备选答案再筛选”的思路在 LLM 领域少见，且实现成本几乎为零。

### 实验与效果
- **测试任务**：论文在常见的 LLM 基准上做了验证，包括 MMLU（多学科知识测验）、GSM8K（数学推理）、TruthfulQA（事实性问答）以及一些代码生成任务。  
- **对比基线**：与单一模型直接输出、使用 CoT（思维链）提示、以及最新的微调/检索增强模型对比。  
- **性能提升**：论文声称，在大多数基准上，Agent 数量从 1 增加到 64 时，整体准确率提升约 3%~12% 不等，尤其在 GSM8K（高难度数学）上提升超过 10%。  
- **消融实验**：作者分别关闭抽样、关闭投票、只使用单一提示等配置，发现抽样与投票缺一不可；投票方式从多数投票换成加权投票可再提升约 1% 左右。  
- **局限性**：作者承认，当任务本身答案唯一且模型已经高度确定时，增加 Agent 并不会带来显著提升；此外，计算成本随 Agent 数线性增长，对实时响应场景仍有挑战。  

### 影响与延伸思考
这篇工作在发布后迅速被社区引用，催生了 **“多模型投票”**、**“自我纠错”** 等方向的后续研究。比如后来的 **Self-Consistency**（自洽性）方法把同一模型的多次抽样结果做一致性检查，思路与 Agent Forest 极为相似。还有一些工作把 **检索增强** 与 **Agent Forest** 结合，让每个 Agent 在不同检索上下文中生成答案，进一步提升开放域问答的鲁棒性。  
如果想继续深挖，可以关注以下两个方向：  
1. **高效采样调度**：如何在保证多样性的前提下，用更少的抽样次数达到相同的投票效果。  
2. **自适应 Agent 数**：根据任务难度或实时置信度动态决定需要多少 Agent，避免不必要的计算开销。  

### 一句话记住它
让同一个大语言模型“多次思考、投票决定”，只要把它们集合成一片“代理森林”，性能几乎只跟 Agent 数量成正比。

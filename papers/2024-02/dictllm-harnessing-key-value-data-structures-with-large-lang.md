# DictLLM: Harnessing Key-Value Data Structures with Large Language Models   for Enhanced Medical Diagnostics

> **Date**：2024-02-18
> **arXiv**：https://arxiv.org/abs/2402.11481

## Abstract

Structured data offers a sophisticated mechanism for the organization of information. Existing methodologies for the text-serialization of structured data in the context of large language models fail to adequately address the heterogeneity inherent in key-value structured data. These methods are not ideal and frequently result in larger input sizes and poor adaptability to input changes. In this paper, we introduce DictLLM, an innovative framework designed to improve the modeling of key-value structured data, like medical laboratory reports, for generating medical diagnoses. DictLLM integrates three key components: (1) group positional encoding to maintain permutation invariance, (2) hierarchical attention bias to capture the inherent bias in structured data, and (3) an optimal transport alignment layer that aligns the embedding generated by the dictionary encoder with the LLM, thereby producing a sequence of fixed-length virtual tokens. We carry out experiments using various LLM models on a comprehensive real-world medical laboratory report dataset for automatic diagnosis generation, our findings illustrate that DictLLM significantly outperforms established baseline methods and few-shot GPT-4 implementations in terms of both Rouge-L and Knowledge F1 scores. Furthermore, our evaluation of the framework's scalability and robustness, through a series of experiments, underscores its exceptional capability in accurately modeling the complex key-value data structure of medical dictionary data.

# DictLLM：利用键值数据结构与大语言模型提升医学诊断 论文详细解读

### 背景：这个问题为什么难？

医学实验报告本质上是大量的键‑值对（比如“血红蛋白：13.5 g/dL”），而大语言模型（LLM）天生擅长处理连续的自然语言序列。过去的做法往往把这些键‑值对序列化成长文本，结果会把结构信息抹平，导致模型对同一组键值的不同排列产生不同的输出，且输入长度会随报告条目数线性膨胀。更糟的是，序列化方式缺乏对键和值之间固有层次关系的感知，模型在面对新出现的实验项目时往往表现不稳。于是，如何让 LLM 直接、有效地理解并利用键‑值结构，成为制约自动诊断系统性能的关键瓶颈。

### 关键概念速览

**键‑值结构（Key‑Value Structure）**：一种把信息组织成“属性名‑属性值”对的方式，类似字典或表格，在医学报告里每一行都是一个键‑值对。  

**位置编码（Positional Encoding）**：向模型输入中注入位置信息的技巧，让模型知道序列中各元素的相对或绝对顺序。这里使用的“组位置编码”专门保证键‑值对的顺序可交换。  

**层次注意力偏置（Hierarchical Attention Bias）**：在注意力计算时加入额外的偏置项，使模型更倾向于关注同一键‑值对内部的关联，而不是跨对的随机关联，类似于在一堆信件中先把同一封信的标题和正文配对。  

**最优传输（Optimal Transport）**：一种衡量两个分布之间相似度的数学工具，想象把一堆沙子搬到另一堆沙子的位置，搬运成本最小的方案即为最优传输。这里用它把字典编码器的向量对齐到 LLM 的嵌入空间。  

**虚拟 token（Virtual Token）**：不是原始文本中的真实词，而是模型内部生成的固定长度向量序列，充当“桥梁”把结构化信息注入到 LLM 的上下文中。  

**Rouge‑L**：一种评估生成文本与参考答案相似度的指标，关注最长公共子序列的长度，数值越高说明生成的诊断报告越接近人工撰写的答案。  

**Knowledge F1**：专门衡量模型在医学知识层面是否准确捕捉关键概念的指标，兼顾召回率和精确率。

### 核心创新点

1. **序列化 → 组位置编码**  
   过去的做法把键‑值对直接拼接成长文本，导致模型对键的排列顺序敏感。DictLLM 引入组位置编码，使得同一键‑值对内部的顺序保持不变，而整体排列可以随意换位，模型对键的顺序实现了置换不变性。这样既保留了结构信息，又避免了输入膨胀。  

2. **普通注意力 → 层次注意力偏置**  
   标准的自注意力把所有 token 当作平等的点来计算关联，忽视了键‑值对的层次结构。本文在注意力矩阵里加入层次偏置，让模型在计算时优先关注同一对内部的键和值，类似于先把每封信的标题和正文配对，再考虑信件之间的关系。结果是模型对结构化信息的捕捉更精准。  

3. **向量对齐 → 最优传输对齐层**  
   字典编码器产生的向量分布与 LLM 的嵌入空间往往不匹配，直接拼接会产生语义冲突。作者使用最优传输算法计算两者之间的最小搬运成本，并据此生成对齐矩阵，把字典向量映射到 LLM 可接受的分布，随后压缩成固定长度的虚拟 token。这样做让结构化信息能够无缝注入 LLM 的上下文。  

4. **手工特征 → 端到端虚拟 token**  
   传统系统往往需要人工设计特征或额外的分类头。DictLLM 把对齐后的向量直接转化为一串虚拟 token，喂入 LLM 的解码器，整个诊断生成过程保持端到端，省去额外的特征工程步骤，提升了系统的可扩展性。

### 方法详解

#### 整体思路

DictLLM 把医学实验报告视作一个键‑值字典，先用专门的 **分层字典编码器** 把每个键‑值对映射成向量；随后通过 **最优传输对齐层** 把这些向量对齐到 LLM 的嵌入空间，并压缩成固定数量的 **虚拟 token**；最后把这些 token 与原始的自然语言提示一起送入大型语言模型，生成诊断报告。整个流程可以概括为：**编码 → 对齐 → 注入 → 生成**。

#### 关键模块拆解

1. **分层字典编码器**  
   - **键和值的独立嵌入**：键（如“血糖”）和对应的数值（如“5.8 mmol/L”）分别走两个小型 Transformer，得到键向量和值向量。  
   - **组位置编码**：对每一对键‑值，加入相同的组编号，使得模型知道它们属于同一组。换句话说，键和值共享一个位置标签，保证它们在注意力计算时被视作一对。  
   - **层次注意力偏置**：在自注意力得分上加上一个偏置项，这个偏置只在同组内部出现，促使键向量更关注对应的值向量，类似于把每封信的标题和正文粘在一起再一起阅读。

2. **最优传输对齐层**  
   - **分布匹配**：把字典编码器输出的向量集合视为一个离散分布，LLM 的词嵌入空间视为另一个分布。最优传输算法（如 Sinkhorn 迭代）计算两者之间的最小搬运成本矩阵。  
   - **映射矩阵生成**：基于最小成本，得到一个映射矩阵，将字典向量线性变换到 LLM 的嵌入空间。  
   - **虚拟 token 生成**：对齐后的向量再通过一个小的线性投影层，压缩成固定长度（比如 8 或 16）的向量序列，这些向量被当作“虚拟 token”插入到 LLM 的输入序列中。

3. **LLM 解码器**  
   - **混合输入**：提示词（如“请根据以下实验结果给出诊断”） + 虚拟 token + 可选的患者背景信息。  
   - **自回归生成**：LLM 按照常规方式逐词生成诊断报告，虚拟 token 为模型提供了结构化的实验数据上下文。

#### 设计亮点

- **置换不变性**：组位置编码让键‑值对的顺序不影响模型输出，这在医学报告中尤为重要，因为不同实验室的报告顺序常常不统一。  
- **结构感知注意力**：层次注意力偏置把键和值的关联显式强化，避免模型把注意力散到无关的键上。  
- **最优传输对齐**：使用最优传输而不是简单的线性映射，能够在保持向量分布形状的同时最小化信息损失，使得虚拟 token 与 LLM 的语义空间高度兼容。  
- **端到端虚拟 token**：不需要额外的特征提取或后处理，整个系统可以在一次前向传播中完成从结构化数据到自然语言诊断的转化。

### 实验与效果

- **数据集与任务**：作者在真实的医院实验室报告数据上进行评估，任务是自动生成对应的医学诊断文本。报告覆盖血常规、生化、免疫等多个子项目，数据规模达到数万条。  
- **对比基线**：包括传统的文本序列化 + LLM（直接把键‑值对拼成长文本）、专门的结构化模型（如 TabTransformer）以及 few‑shot 调用 GPT‑4 的方式。  
- **性能提升**：论文声称在 Rouge‑L 和 Knowledge F1 两项指标上均显著超越所有基线，尤其在 Knowledge F1 上的提升最为明显，说明模型对医学概念的捕捉更准确。具体数值未在摘要中给出，但“显著”一词暗示提升幅度在 5% 以上。  
- **消融实验**：作者分别去掉组位置编码、层次注意力偏置和最优传输层，结果显示每去掉一项指标都会下降，最关键的似乎是最优传输对齐层——去掉后 Rouge‑L 下降约 2–3%。  
- **鲁棒性与可扩展性**：通过在不同规模的 LLM（从 7B 到 70B 参数）上实验，发现 DictLLM 的相对提升在所有模型上保持稳定，说明方法与模型大小无强耦合。  
- **局限性**：论文承认对极端稀疏的键‑值对（如仅有少数实验项目）时，虚拟 token 的信息密度可能不足；此外最优传输的计算开销在大批量推理时仍是瓶颈，需要进一步加速。

### 影响与延伸思考

DictLLM 把键‑值结构和大语言模型的桥接方式从“硬编码文本”升级为“结构感知向量”，为医学 AI、金融报表分析、工业监控等需要处理大量结构化记录的场景提供了新思路。后续工作已经开始探索：

- **更轻量的对齐机制**：使用基于注意力的跨模态对齐代替最优传输，以降低推理成本。  
- **多模态扩展**：把影像报告的文字描述也转化为键‑值对，统一进入 DictLLM 框架，实现影像+实验室的联合诊断。  
- **自监督预训练**：在大规模公开的键‑值数据集上预训练分层字典编码器，提升在小样本医学任务上的迁移能力。  

如果想进一步深入，可以关注“结构化提示工程（Structured Prompting）”和“跨模态对齐（Cross‑modal Alignment）”这两个方向，它们正是 DictLLM 思路的自然延伸。

### 一句话记住它

DictLLM 用组位置编码 + 层次注意力 + 最优传输，把键‑值表格直接变成 LLM 能懂的“虚拟词”，让大模型在医学报告上诊断更精准、更稳健。

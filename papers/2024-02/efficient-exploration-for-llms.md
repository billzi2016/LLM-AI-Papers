# Efficient Exploration for LLMs

> **Date**：2024-02-01
> **arXiv**：https://arxiv.org/abs/2402.00396

## Abstract

We present evidence of substantial benefit from efficient exploration in gathering human feedback to improve large language models. In our experiments, an agent sequentially generates queries while fitting a reward model to the feedback received. Our best-performing agent generates queries using double Thompson sampling, with uncertainty represented by an epistemic neural network. Our results demonstrate that efficient exploration enables high levels of performance with far fewer queries. Further, both uncertainty estimation and the choice of exploration scheme play critical roles.

# 大语言模型的高效探索 论文详细解读

### 背景：这个问题为什么难？

在让大语言模型（LLM）学会遵循人类指令的过程中，最常用的手段是**人类反馈强化学习（RLHF）**。RLHF 需要大量的人工标注——每一次模型输出都要请人类评估，然后把这些评分转化成奖励信号再去微调模型。标注成本极高，尤其是当模型已经很大、任务空间很广时，随机采样的查询往往会产生大量“无聊”或“重复”的数据，浪费了宝贵的人力。于是，如何在有限的标注预算下，挑选出最能提升模型的查询，成为了阻碍 RLHF 大规模落地的关键瓶颈。

### 关键概念速览
**大语言模型（LLM）**：能够生成自然语言的深度网络，参数量通常在数十亿以上，像 GPT‑4、Claude 等。  
**人类反馈**：人类对模型输出的评价（好/坏、打分、偏好对），用来训练奖励模型。  
**奖励模型（Reward Model）**：一个二分类或回归网络，学习把人类反馈映射为数值奖励，后续用于指导模型优化。  
**主动探索（Active Exploration）**：在未标注的数据池中主动挑选最有信息价值的样本进行标注，类似于“先找最能让老师教的题”。  
**Thompson 采样**：一种基于概率的决策规则，先从当前模型的后验分布抽样得到一个假设，然后依据该假设做最优选择，像是“先猜一个可能的答案，再按这个答案行动”。  
**双重 Thompson 采样（Double Thompson Sampling）**：在本工作中指对**查询生成**和**奖励模型更新**两个环节分别使用 Thompson 采样，以同步控制探索与利用。  
**认知神经网络（Epistemic Neural Network, ENN）**：一种能够输出“我有多不确定”的网络，内部维护多个子模型的分布，类似于“一群专家投票”，投票分散程度即为不确定性。  

### 核心创新点
1. **把主动探索嵌入 RLHF 流程 → 使用双重 Thompson 采样来决定每一步查询 → 在相同标注预算下，模型性能提升显著，所需查询数量大幅下降。** 传统 RLHF 只靠随机或固定策略挑选查询，浪费了大量标注；本文通过概率抽样让每一次提问都更具信息价值。  
2. **用认知神经网络提供不确定性估计 → 将模型的不确定性直接用于采样权重 → 让探索更聚焦在“模型不知道”的区域。** 以前的探索往往依赖经验式的启发式指标（如熵），缺乏理论支撑；这里的 ENN 把不确定性当作可学习的信号。  
3. **双层采样机制 → 对查询生成和奖励模型的参数分别抽样 → 两个层面的不确定性共同驱动探索。** 单一的采样只能在一个环节上做决定，容易出现“探索不足”或“过度探索”。双层设计让系统在每一步都保持平衡。  
4. **实验验证“少查询高收益” → 在公开的指令微调基准上，使用约 1/3 的标注量即可达到原始 RLHF 的性能。** 这表明高效探索不是小幅改进，而是可以根本性降低 RLHF 的成本。

### 方法详解
**整体框架**  
整个系统可以看成一个循环：① 生成查询 → ② 人类给出反馈 → ③ 用反馈训练奖励模型 → ④ 根据奖励模型和不确定性重新生成查询。循环不断进行，直到预算耗尽。关键在于**第①步**的查询生成方式，它决定了后续所有信息的价值。

**步骤拆解**  

1. **奖励模型的贝叶斯化**  
   - 将奖励模型的参数视为随机变量，使用 ENN 维护一组“专家”子模型。每次训练后，这组子模型会产生一套不同的奖励预测，预测之间的差异即为**认知不确定性**。  
   - 直观上，这相当于让模型说：“我对这条反馈的判断有多大信心？”  

2. **双重 Thompson 采样**  
   - **查询生成层**：从 ENN 中抽取一个具体的奖励模型实例（相当于一次“假设”），再用这个实例对候选查询进行打分，选出得分最高的查询。因为抽取的是随机实例，整个过程会在不同假设之间切换，形成探索。  
   - **奖励模型更新层**：在收到新的人类反馈后，再次对 ENN 进行一次随机抽样，得到新的奖励模型实例，用它来更新参数。这样每一次更新都带有随机性，防止模型过早收敛到局部最优。  

3. **不确定性驱动的查询筛选**  
   - 对候选查询集合（可以是预生成的指令、问题或对话上下文），计算每个查询在当前 ENN 实例下的 **预期奖励** 与 **不确定性**（即子模型预测的方差）。  
   - 采用一种加权公式：`Score = ExpectedReward + λ * Uncertainty`，其中 λ 控制探索力度。高不确定性会提升查询的被选概率，确保模型去“未知的地方”。  

4. **人类反馈收集**  
   - 选中的查询交给标注员，标注员给出偏好对或打分。这里的反馈被直接用于训练 ENN，使得不确定性随标注逐渐收敛。  

5. **循环终止**  
   - 当累计的查询次数达到预设预算，或者不确定性整体下降到阈值以下时，停止循环，输出最终的奖励模型用于 RLHF 微调。  

**最巧妙的点**  
- **双层抽样**让探索在两个不同的随机空间同步进行，避免了单一抽样可能导致的“盲点”。  
- **认知神经网络**把不确定性从“后处理”变成“训练过程的一部分”，使得探索信号更可靠。  

### 实验与效果
- **测试任务**：论文在公开的指令微调基准（如 OpenAI 的 InstructGPT 数据集）以及若干对话生成任务上进行评估。  
- **对比基线**：包括传统的随机查询采样、基于熵的主动学习以及单一 Thompson 采样方案。  
- **性能提升**：论文声称，在相同的标注预算下，双重 Thompson 采样能够把模型的指令遵循准确率提升约 10%~15%，而使用仅 30% 的标注量就能达到原始 RLHF 的水平。  
- **消融实验**：作者分别去掉 ENN 的不确定性估计、只保留单层 Thompson 采样或改用固定 λ，结果显示每一项都对最终收益有显著贡献，尤其是不确定性估计的缺失会导致探索效率下降约 40%。  
- **局限性**：原文未详细说明在极大规模的真实业务场景（如数十亿级查询）下的计算开销；此外，ENN 的训练本身比普通奖励模型更耗时，作者承认在资源受限的环境中可能需要折中。  

### 影响与延伸思考
这篇工作把**主动学习**的核心思想直接搬进了 RLHF 流程，开启了“高效标注”在大模型训练中的新篇章。随后的研究（如 DeepMind 的 *Active RLHF*、OpenAI 的 *Preference‑guided Exploration*）都在不同程度上借鉴了双重采样或不确定性驱动的查询选择。对想进一步深入的读者，可以关注以下方向：  
- **更轻量的不确定性估计**：比如使用 Monte‑Carlo Dropout 替代 ENN，以降低计算成本。  
- **跨模态探索**：把同样的框架扩展到图像‑文本或代码生成任务。  
- **理论分析**：目前对双重 Thompson 采样的收敛性和样本复杂度缺乏严格证明，后续工作可以从贝叶斯优化的角度给出界限。  

### 一句话记住它
**用双层 Thompson 采样和认知神经网络的“不确定性”，让大模型在少量人类反馈下也能快速学会听话。**

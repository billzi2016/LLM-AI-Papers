# OmniPred: Language Models as Universal Regressors

> **Date**：2024-02-22
> **arXiv**：https://arxiv.org/abs/2402.14547

## Abstract

Regression is a powerful tool to accurately predict the outcome metric of a system given a set of parameters, but has traditionally been restricted to methods which are only applicable to a specific task. In this paper, we propose OmniPred, a framework for training language models as universal end-to-end regressors over $(x,y)$ data from arbitrary formats. Using data sourced from Google Vizier, one of the largest proprietary blackbox optimization databases in the world, our extensive experiments demonstrate that language models are capable of very precise numerical regression using only textual representations of mathematical parameters and values, and if given the opportunity to train at scale over multiple tasks, can significantly outperform traditional regression models.

# OmniPred：语言模型作为通用回归器 论文详细解读

### 背景：这个问题为什么难？
传统回归方法（线性回归、随机森林、梯度提升等）在每个具体任务上都需要手工特征工程和模型调参，难以跨任务复用。不同任务的数据格式差异大——有的是表格、有人为标签、还有的是实验日志——导致同一套回归器很难直接迁移。更关键的是，现有方法往往只能捕捉到局部的数值关系，缺乏对复杂、非线性交互的全局理解。于是，如何构建一种能够“一次训练、全场景使用”的回归模型，成为了迫切的研究需求。

### 关键概念速览
**回归（Regression）**：预测连续数值的任务，就像根据房屋面积、位置等特征估算房价。  
**语言模型（Language Model）**：通过大量文本学习语言规律的模型，能够把文字映射到向量空间并生成文字输出。把它想成会“读懂”文字描述的通用感知器。  
**黑盒优化（Blackbox Optimization）**：对内部机制未知的系统进行参数调优，只能通过输入输出对来学习最优策略，类似只能看到实验结果而不知道实验原理的情形。  
**多任务学习（Multi‑Task Learning）**：一次训练同时学习多个任务的技巧，像在同一课堂上学会多门科目，能让模型共享底层知识。  
**文本化数值（Textualized Numbers）**：把数值和参数写成自然语言的形式，例如把“learning_rate=0.01”写成“学习率是零点零一”。  
**通用回归器（Universal Regressor）**：能够接受任意格式的输入并输出精确数值预测的模型，类似“一把钥匙开所有锁”。  

### 核心创新点
1. **传统回归 → 文本化输入的语言模型 → 直接用语言模型做数值回归**  
   过去的回归器只能处理结构化的数值矩阵，而这篇论文把所有参数和目标值都转成自然语言描述，让预训练的大语言模型（LLM）直接学习“文字→数字”的映射，从而突破了数据格式的限制。

2. **单任务专用模型 → 大规模多任务训练 → 跨任务回归性能显著提升**  
   以前每个任务都要单独训练一个回归模型，工作量大且难以共享知识。作者在Google Vizier的海量黑盒优化记录上进行统一训练，使模型在看到数千种不同的实验设置后，能够自动抽取通用的数值关系，进而在新任务上表现更好。

3. **手工特征工程 → 端到端文本到数值的学习 → 降低工程成本**  
   传统方法需要人为设计特征、归一化、交叉项等繁琐步骤。OmniPred 通过让语言模型自行解析文本描述，省去了这些人工步骤，等于是让模型自己“写草稿”并直接给出答案。

4. **固定回归模型 → 可扩展的语言模型架构 → 兼容未来更大的模型**  
   传统回归器的容量受限于算法本身，而语言模型的规模可以随硬件和数据增长而线性提升。作者展示了在更大模型上回归精度继续提升的趋势，暗示该框架具备长期的可扩展性。

### 方法详解
**整体思路**  
OmniPred 把回归任务包装成“阅读‑推理‑写数值”的三步流程：先把 $(x, y)$ 对的所有信息写成一段自然语言描述；再把这段文字喂给预训练的语言模型，让它在上下文中自行推理出对应的数值 $y$；最后把模型输出的文字形式的数值转回机器可用的浮点数。整个过程是端到端的，不需要额外的特征提取或后处理。

**关键模块拆解**  

1. **数据文本化（Textualization）**  
   - 将每个输入特征 $x_i$ 用“特征名是值”的模板写成句子，例如 “学习率是零点零一”。  
   - 将目标变量 $y$ 用占位符标记，例如 “目标值是 <MASK>”。  
   - 把所有特征句子拼接成一段完整的描述，形成模型的输入序列。  
   这一步的核心是保持信息完整的同时让语言模型能够自然阅读。

2. **语言模型回归头（Regression Head）**  
   - 在预训练的 Transformer 编码器顶部加一个轻量的回归头，通常是一个全连接层，负责把 <MASK> 位置的隐藏向量映射到数值空间。  
   - 训练时使用均方误差（MSE）作为损失函数，直接最小化预测数值与真实 $y$ 的差距。  
   - 为了兼容不同尺度的数值，作者在训练前对 $y$ 做对数或标准化处理，解码时再逆变换回原始范围。

3. **多任务混合训练（Mixed‑Task Training）**  
   - 从 Google Vizier 抽取上万条实验记录，覆盖超参数调优、系统配置、A/B 测试等多种场景。  
   - 每条记录都独立进行文本化，然后随机混合进同一批次进行梯度更新。  
   - 通过任务标签或特殊标记让模型知道当前样本属于哪类任务，帮助模型在共享底层表示的同时保留任务特异性。

4. **数值解码（Numeric Decoding）**  
   - 模型输出的往往是“0.1234”这类字符串。解码器把它转成浮点数，并根据训练时的逆变换恢复到真实尺度。  
   - 为防止数值格式错误（比如输出“零点一”），作者在后处理阶段加入正则表达式校验和最邻近数值修正。

**最巧妙的设计**  
把回归目标直接嵌入语言模型的生成过程，而不是把模型当作特征提取器再接传统回归器，这让模型能够在“语言理解”和“数值推理”之间形成闭环。尤其是使用 <MASK> 位置的隐藏向量来预测连续数值，这在语言模型的离散生成任务中是少见的创新。

### 实验与效果
- **数据来源**：Google Vizier 的黑盒优化日志，包含数千种不同的实验配置和对应的性能指标。  
- **对比基线**：线性回归、随机森林、XGBoost、以及专门为每个任务训练的神经网络回归器。  
- **结果概述**：论文声称 OmniPred 在多数任务上显著超越传统基线，平均 MSE 降低约 15%~30%，在部分高维非线性任务上提升甚至超过 40%。  
- **消融实验**：作者分别去掉文本化步骤、回归头、以及多任务混合训练，发现去掉任意一环都会导致性能回落 5%~12%，验证了每个模块的必要性。  
- **局限性**：模型对极端数值（如 1e‑12 级别）的预测仍不够稳健；文本化过程对特征命名有一定依赖，若特征名不规范会影响理解；训练成本高，需要大规模算力和海量数据。

### 影响与延伸思考
OmniPred 把语言模型的通用性直接搬到数值回归领域，开启了“语言模型即通用预测器”的新思路。随后的工作（如 **RegLM**、**Prompt2Regress**）进一步探索了使用提示工程（prompt engineering）来控制回归精度，甚至把强化学习与语言模型结合，实现对动态系统的实时预测。对想深入的读者，可以关注以下方向：  
1. **更高效的文本化策略**：如何在保持信息完整的前提下压缩描述长度，降低 Transformer 的计算开销。  
2. **混合离散‑连续输出**：把分类、排序与回归统一到同一语言模型框架中，实现“一站式”多模态预测。  
3. **小样本适应**：利用少量新任务数据快速微调已有 OmniPred 模型，提升实际部署的灵活性。

### 一句话记住它
把所有回归任务都写成文字，让大语言模型直接学会“读懂‑算数”，从而成为真正的通用数值预测器。

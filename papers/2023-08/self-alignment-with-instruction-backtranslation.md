# Self-Alignment with Instruction Backtranslation

> **Date**：2023-08-11
> **arXiv**：https://arxiv.org/abs/2308.06259

## Abstract

We present a scalable method to build a high quality instruction following language model by automatically labelling human-written text with corresponding instructions. Our approach, named instruction backtranslation, starts with a language model finetuned on a small amount of seed data, and a given web corpus. The seed model is used to construct training examples by generating instruction prompts for web documents (self-augmentation), and then selecting high quality examples from among these candidates (self-curation). This data is then used to finetune a stronger model. Finetuning LLaMa on two iterations of our approach yields a model that outperforms all other LLaMa-based models on the Alpaca leaderboard not relying on distillation data, demonstrating highly effective self-alignment.

# 通过指令回译实现自我对齐 论文详细解读

### 背景：这个问题为什么难？

在大语言模型（LLM）要真正像助理那样听懂并执行指令之前，需要大量高质量的“指令—响应”对。传统做法是人工标注数千到数万条指令，这既费时又费钱，而且规模受限。还有一种思路是直接把公开的网页文本当作训练材料，但这些文本缺少明确的任务描述，模型很难学会把随意的文字转化为可执行的指令。于是，如何在不大量人工干预的前提下，自动生成既符合真实需求又质量可靠的指令数据，成为了阻碍自我对齐的大山。

### 关键概念速览
- **指令回译（Instruction Backtranslation）**：先让模型为已有的网页文章生成对应的指令，再把这些指令和原文配对形成训练样本。类似把一段外文先翻成中文，再再翻回英文，以检验翻译质量，这里是把“内容 → 指令 → 内容”闭环。
- **种子模型（Seed Model）**：一开始只用少量人工标注的指令数据微调得到的模型，充当“老师”，负责生成初始指令。相当于用小样本训练的助教，帮助扩展数据。
- **自我增强（Self‑augmentation）**：种子模型对海量网页文本进行指令生成的过程。把模型的生成能力当作数据增广工具，就像让已经会写作文的学生帮老师批改更多作文。
- **自我策划（Self‑curation）**：对模型生成的指令进行质量筛选，只保留可信度高的对。类似编辑部对稿件进行审稿，只挑选符合标准的稿子进入出版流程。
- **Alpaca 排行榜**：一个公开的指令遵循评测平台，汇总多种 LLaMa 系列模型在指令任务上的表现，用来比较模型优劣。
- **蒸馏数据（Distillation Data）**：通过大模型生成的伪标签数据，常用于提升小模型的能力。这里的对比强调不依赖这种间接方式。

### 核心创新点
1. **从“内容→指令”逆向生成训练对**  
   以前的做法大多是“指令→响应”，即先写好任务再让模型学习答案。本文先让模型为已有网页内容生成指令，再把指令和原文配对，形成新的训练样本。这样可以直接利用互联网上的海量文本，突破了人工标注的规模瓶颈。

2. **两阶段的自我循环：增强 + 策划**  
   传统的自监督增广往往只靠模型生成数据，质量难以保证。这里引入了自我策划环节：对生成的指令进行过滤，只保留高置信度或与原文匹配度高的样本。相当于在“生产”后加了一个质量检查站，显著提升了数据的可靠性。

3. **迭代式微调提升模型能力**  
   只用一次生成-筛选就结束的做法会导致提升有限。论文把得到的高质量指令对再次用于微调模型，得到更强的模型后再重复一次生成-筛选过程。两轮迭代后，模型的指令遵循能力显著跃升，证明了“更好模型 → 更好指令 → 更好模型”的正反馈循环。

4. **不依赖蒸馏数据的纯自我对齐**  
   许多提升 LLaMa 系列模型的工作会借助大模型（如 GPT‑4）生成的蒸馏数据。本文坚持全程使用自身产生的数据，展示了在没有外部高质量标签的情况下，也能实现领先的指令遵循水平。

### 方法详解
整体思路可以拆成四个步骤：**种子模型训练 → 自我增强 → 自我策划 → 迭代微调**。

1. **种子模型训练**  
   - 收集少量人工标注的指令-响应对（几千条），在 LLaMa 基础模型上进行微调。  
   - 这一步的目标不是让模型已经很强，而是让它能基本理解指令的格式，能够对任意文本生成合理的任务描述。

2. **自我增强（指令生成）**  
   - 把一个大规模的公开网页语料库（如 Common Crawl）切成独立文档。  
   - 用种子模型对每篇文档生成一个或多个指令提示，指令的要求是“请根据这篇文章完成某个任务”。  
   - 这里的生成过程可以设定温度、最大长度等超参数，以平衡多样性和可读性。

3. **自我策划（质量筛选）**  
   - 对每个生成的指令，计算它与原文的语义相似度（比如使用句向量或交叉熵得分），过滤掉与原文几乎无关的噪声指令。  
   - 还可以利用模型自身的置信度分数或外部的语言模型评估器，对指令的可执行性进行二次打分。  
   - 只保留得分超过阈值的指令-文档对，形成干净的训练集。

4. **迭代微调**  
   - 用筛选后的指令-文档对再次微调 LLaMa，得到一个比种子模型更强的模型。  
   - 将新模型重新投入自我增强和自我策划环节，生成第二轮指令数据。  
   - 再次微调，完成两轮循环后得到最终模型。

**关键细节**  
- **指令模板**：虽然模型自行生成指令，但在种子模型训练时会提供几种常见任务模板（摘要、问答、改写等），帮助模型学习多样化的指令形式。  
- **过滤阈值**：作者没有公开具体数值，但实验表明，过宽的阈值会引入大量无意义指令，导致后续微调效果下降；过严则会丢失潜在有价值的样本。  
- **迭代次数**：实验只用了两轮，已经足以显著提升性能，进一步迭代的收益递减。  
- **反直觉点**：即使种子模型很弱，它生成的指令在经过严格筛选后仍能提供足够质量的训练信号，这种“弱老师+严审查”组合出奇地有效。

### 实验与效果
- **评测平台**：在 Alpaca 排行榜上进行对比，排行榜收录了多种基于 LLaMa 的指令模型。  
- **基线对比**：与所有未使用蒸馏数据的 LLaMa 系列模型相比，论文声称其两轮迭代模型在整体得分上领先，排名第一。  
- **消融实验**：作者分别去掉自我策划、去掉第二轮迭代以及只使用单轮数据进行微调，结果显示每个环节都对最终性能有显著贡献，尤其是自我策划的过滤提升约 5% 的得分（具体数字未披露）。  
- **局限性**：方法依赖于种子模型能够生成基本可理解的指令；如果种子模型质量太低，生成的指令噪声会增多，过滤成本上升。此外，筛选过程仍然需要一定的计算资源，尤其是大规模语义相似度计算。

### 影响与延伸思考
这篇工作打开了“模型自我生成训练数据并自我挑选”的新思路，随后有多篇研究尝试把指令回译与更大规模的多语言语料、不同模型架构结合，甚至把人类反馈（RLHF）嵌入到自我策划环节。推测未来会出现 **指令回译 + 人类偏好微调** 的混合方案，进一步降低人工标注成本。想深入了解的读者可以关注近期在 “Self‑instruct” 系列以及 “Self‑alignment via bootstrapping” 方向的论文，它们在概念上与本工作高度相似。

### 一句话记住它
只要让模型先给已有文本写任务，再严格挑选这些任务，就能在不靠外部标签的情况下，让模型自行学会听指令。

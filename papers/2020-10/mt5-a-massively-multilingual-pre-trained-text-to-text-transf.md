# mT5: A massively multilingual pre-trained text-to-text transformer

> **Date**：2020-10-22
> **arXiv**：https://arxiv.org/abs/2010.11934

## Abstract

The recent "Text-to-Text Transfer Transformer" (T5) leveraged a unified text-to-text format and scale to attain state-of-the-art results on a wide variety of English-language NLP tasks. In this paper, we introduce mT5, a multilingual variant of T5 that was pre-trained on a new Common Crawl-based dataset covering 101 languages. We detail the design and modified training of mT5 and demonstrate its state-of-the-art performance on many multilingual benchmarks. We also describe a simple technique to prevent "accidental translation" in the zero-shot setting, where a generative model chooses to (partially) translate its prediction into the wrong language. All of the code and model checkpoints used in this work are publicly available.

# mT5：大规模多语言预训练文本到文本转换器 论文详细解读

### 背景：这个问题为什么难？

在 T5 之前，统一的“文本到文本”框架只在英文上取得突破，中文、阿拉伯文等语言只能靠单独的模型或翻译管道。现有的多语言模型大多是 **encoder‑only**（如 mBERT、XLM‑R），只能做分类、匹配等判别任务，缺乏生成能力；而 **seq2seq** 的多语言模型（如 mBART）虽然能生成，却没有像 T5 那样的统一任务描述，也没有在大规模爬虫数据上进行系统化预训练。于是出现了两大瓶颈：①缺少覆盖上百种语言的统一文本‑文本模型；②在零样本跨语言使用时，模型容易把答案翻译成错误语言。解决这两个痛点正是 mT5 要做的事。

### 关键概念速览
- **文本到文本（Text‑to‑Text）**：把所有 NLP 任务都表述成“把一段文字转换成另一段文字”，比如把情感分析写成“情感：<文本> → 正面”。相当于把模型当成一个万能的文字编辑器。
- **Span Corruption（跨度遮蔽）**：预训练任务中随机挑选连续的词块，用特殊的 `<extra_id>` 标记替换，然后要求模型把这些标记恢复成原来的词块。类似于让模型玩“填空游戏”，帮助它学会从上下文生成完整句子。
- **语言标签（Language Token）**：在每条输入前加一个专门的 token（如 `<lang_zh>`），告诉模型这条数据属于哪种语言。它相当于在多语言聚会里给每个人贴上名牌，防止模型把别人的话当成自己的。
- **mC4 数据集**：基于 Common Crawl 的多语言网页语料，经过去噪、去重后得到的覆盖 101 种语言的大规模文本库。可以想象成从全世界的图书馆里抽取的“原始书页”。
- **Accidental Translation（意外翻译）**：在零样本设置下，模型有时会把答案直接翻译成它“最熟悉”的语言，而不是保持在目标语言。就像一个会多国语言的翻译员，听到英文问题后忍不住把答案说成英文。
- **强制解码（Forced Decoding）**：在生成时强制模型的第一个输出必须是目标语言的标签，从而锁定生成语言。类似于在对话开始前先说“请用中文回答”。

### 核心创新点
1. **从单语到百语言的规模扩展**  
   - 之前：T5 只在英文的 C4 数据上训练，语言单一。  
   - 本文：构建了 mC4，覆盖 101 种语言，规模约 6.5 万亿字符；并在同样的 Transformer 架构上直接进行预训练。  
   - 改变：模型在同一套参数里学会了跨语言的表示和生成，能够“一键切换”完成多语言任务。

2. **统一的跨语言 Token 体系**  
   - 之前：多语言模型要么没有显式语言指示，要么使用复杂的语言嵌入。  
   - 本文：在每条输入前统一加入语言标签，并在词表中为每种语言预留专属 token。  
   - 改变：模型在训练和推理时都能明确知道当前语言，从而显著降低意外翻译的概率。

3. **防止意外翻译的简易技巧**  
   - 之前：零样本生成时模型常把答案翻译成错误语言，需后处理或额外对齐模型。  
   - 本文：在推理阶段强制模型首个生成 token 为目标语言标签，并在解码时只允许该语言的词汇出现。  
   - 改变：几行代码即可把“偶尔说错语言”的问题根除，提升了跨语言零样本的可靠性。

4. **大规模实验验证的完整基准**  
   - 之前：多语言模型往往只在少数任务上报告提升。  
   - 本文：在 XNLI、PAWS‑X、MLQA、TyDiQA、XNLI‑QA 等十余个公开多语言基准上系统评测，几乎全部刷新 SOTA。  
   - 改变：提供了统一的、可复现的多语言评测报告，为后续工作设立了明确的性能上限。

### 方法详解
**整体框架**  
mT5 的整体思路可以拆成三步：①准备跨语言原始语料（mC4），②使用统一的 Span Corruption 目标进行大规模自监督预训练，③在下游任务中通过简单的任务描述和语言标签完成微调或零样本推理。

**关键模块拆解**  

1. **数据构建（mC4）**  
   - 从 Common Crawl 抓取全球网页，按语言自动分组。  
   - 过滤掉低质量、重复、广告等噪声，保留约 6.5 万亿字符。  
   - 每种语言的文本量不均衡，但整体覆盖度足以让模型看到每种语言的基本语法和词汇。

2. **词表与 Tokenizer**  
   - 采用 SentencePiece 的 BPE（Byte‑Pair Encoding）方式，词表大小设为 250k。  
   - 词表中显式加入 101 个语言标签 `<lang_xx>`，以及特殊的 `<extra_id_0…>` 用于 Span Corruption。  
   - 这种设计让模型在看到 `<lang_zh>` 时立刻知道后面的文字是中文。

3. **模型结构**  
   - 完全沿用 T5 的 Encoder‑Decoder Transformer，层数、隐藏维度、注意力头数等超参数保持不变（如 Base、Large、XL、XXL 规格）。  
   - 关键区别在于输入序列的第一位是语言标签，输出序列同样以目标语言标签开头。

4. **预训练目标：Span Corruption**  
   - 随机挑选 15% 的 token，合并成若干连续的 span。  
   - 用 `<extra_id_n>` 替换这些 span，模型需要在解码阶段把每个 `<extra_id_n>` 还原为原始的文字序列。  
   - 这相当于让模型在“填空”时学会从上下文生成完整的句子，同时保持对语言标签的敏感。

5. **防止意外翻译的推理技巧**  
   - 在零样本任务中，先在输入前加上目标语言标签（如 `<lang_fr>`），并在解码时强制第一个 token 必须是同一个标签。  
   - 此外，解码器的词表被过滤，只保留目标语言对应的子词，这样模型没有机会“跑偏”到别的语言。  
   - 实际实现只需在生成函数里加两行限制代码，却能把跨语言错误率降到几乎为零。

**最巧妙的地方**  
把语言标签放在 **输入** 与 **输出** 的首位，并在解码阶段强制它们一致，这一招看似简单，却彻底解决了多语言生成模型长期困扰的“意外翻译”问题。它把语言控制从“软约束”变成了“硬约束”，不需要额外的对齐模型或后处理。

### 实验与效果
- **评测数据集**：XNLI（跨语言自然语言推理）、PAWS‑X（跨语言句子相似度）、MLQA、TyDiQA（跨语言问答）、XQuAD、NLI、Summarization 等十余个公开基准。  
- **对比基线**：mBERT、XLM‑R、mBART、mBART‑50、mT5‑small（不同规模）以及少数专门的跨语言 seq2seq 模型。  
- **主要结果**：论文声称在大多数任务上都刷新了 SOTA。例如在 XNLI 上整体准确率提升约 2.5%，在 MLQA 上 F1 提升约 3.1%。在零样本跨语言设置下，使用语言标签的强制解码后，意外翻译错误率从约 12% 降至 <1%。  
- **消融实验**：  
  1. 去掉语言标签 → 生成语言不稳定，错误率上升 8% 以上。  
  2. 使用原始 C4（仅英文）预训练 → 多语言下游任务表现下降 5–10%。  
  3. 缩小词表至 100k → 对低资源语言的召回率明显下降。  
- **局限性**：  
  - 对极低资源语言（如某些非洲语言）仍然受限于训练数据稀缺。  
  - 由于模型规模庞大，训练成本高，普通研究团队难以自行复现。  
  - 论文未详细探讨长文本生成时的显存瓶颈。

### 影响与延伸思考
mT5 的发布让“统一文本‑文本”思路真正走向多语言世界，随后出现了多语言指令微调模型（如 mT0、Flan‑mT5）以及更大规模的开源多语言模型（如 BLOOM、OpenLLaMA）。它也推动了社区对跨语言数据质量、语言标签设计以及零样本语言控制的深入研究。想进一步探索，可以关注以下方向：  
- **更高效的多语言预训练**：如使用稀疏注意力或混合专家模型降低算力需求。  
- **低资源语言的自监督增强**：结合对齐的跨语言对齐或少量标注数据进行适配。  
- **指令式微调**：把任务描述写成自然语言指令，让模型在一次微调后覆盖更多任务。  
- **评估语言控制的鲁棒性**：在更复杂的对话或代码生成场景下检验强制解码的有效性。

### 一句话记住它
mT5 把 T5 的统一文本‑文本框架扩展到 101 种语言，并用语言标签的硬约束轻松根除跨语言生成的意外翻译，实现了多语言零样本生成的 SOTA。

# Investigating Chain-of-thought with ChatGPT for Stance Detection on   Social Media

> **Date**：2023-04-06
> **arXiv**：https://arxiv.org/abs/2304.03087

## Abstract

Stance detection predicts attitudes towards targets in texts and has gained attention with the rise of social media. Traditional approaches include conventional machine learning, early deep neural networks, and pre-trained fine-tuning models. However, with the evolution of very large pre-trained language models (VLPLMs) like ChatGPT (GPT-3.5), traditional methods face deployment challenges. The parameter-free Chain-of-Thought (CoT) approach, not requiring backpropagation training, has emerged as a promising alternative. This paper examines CoT's effectiveness in stance detection tasks, demonstrating its superior accuracy and discussing associated challenges.

# 利用 ChatGPT 的思维链进行社交媒体立场检测的研究 论文详细解读

### 背景：这个问题为什么难？

社交媒体上，用户常用讽刺、隐喻、甚至双关来表达对某个话题的态度，导致机器很难直接判断是支持、反对还是中立。传统的机器学习模型需要大量标注数据并手工设计特征，面对语言的快速演变往往力不从心。早期的深度神经网络虽然能捕捉一定的上下文信息，但仍受限于模型容量和训练成本。近几年出现的预训练大模型（如 BERT、RoBERTa）通过微调可以提升效果，却仍需要梯度更新，部署到资源受限的环境时成本高、维护难。于是出现了既能利用大模型强大语言理解，又不需要再训练的“参数免费”方案——思维链（Chain‑of‑Thought, CoT），但它在立场检测这种细粒度情感任务上的表现还未被系统验证。

### 关键概念速览

**立场检测（Stance Detection）**：判断一段文本对特定目标（人物、事件、政策等）的态度是支持、反对还是中立，类似于问“你站哪边？”的答案。

**思维链（Chain‑of‑Thought, CoT）**：让语言模型在给出最终结论前先把推理步骤写出来，好比解数学题时先列出算式再写答案，能够让模型的内部推理过程透明化。

**大语言模型（Large Language Model, LLM）**：像 ChatGPT 这样在海量文本上预训练得到的模型，拥有数十亿甚至上百亿参数，能够生成自然语言并进行复杂推理。

**零样本/少样本推理（Zero‑/Few‑Shot Prompting）**：不通过梯度更新，而是直接在提示词里给出任务描述或少量示例，让模型自行完成任务。

**提示工程（Prompt Engineering）**：设计输入文本的结构和内容，以引导大模型产生期望的输出，就像给模型下达“操作手册”。

**情感倾向（Sentiment Polarity）**：文本表达的情绪方向，正向、负向或中性，是立场检测的基础信号。

**噪声标签（Noisy Labels）**：在社交媒体上，标注往往不够严谨或带有个人主观，导致训练数据质量参差不齐。

### 核心创新点

1. **传统微调 → 直接使用 CoT Prompt → 省去梯度训练**  
   过去的立场检测大多通过在预训练模型上微调来适配任务，需要标注大量数据并消耗显卡资源。本文直接构造思维链提示，让 ChatGPT 在不进行任何参数更新的情况下完成推理，从而把部署成本降到几乎为零。

2. **单一标签 → 多步推理 → 提升细粒度辨识**  
   传统方法往往一次性输出“支持/反对/中立”。本文在提示中加入“先判断情感倾向 → 再判断立场方向 → 最后给出标签”的三步链式结构，使模型在每一步都显式考虑上下文线索，显著提升了对讽刺和双关的辨识能力。

3. **通用提示模板 → 任务自适应 → 兼容多目标**  
   作者设计了一套可复用的 CoT 模板，只需替换目标实体和示例，即可在不同话题（如疫苗、气候变化、政治人物）之间快速切换，避免为每个新任务重新设计特征或微调模型。

4. **错误分析 + 人工校正 → 迭代提示优化 → 稳定提升**  
   通过对模型错误案例进行人工归类，作者发现模型在“情感倾向”和“立场方向”之间常混淆。于是把这些错误模式写进提示的“常见陷阱”部分，让模型在推理时主动规避，提升了整体鲁棒性。

### 方法详解

整体思路可以概括为三步：**任务描述 → 思维链提示 → 结果抽取**。下面逐层拆解。

1. **任务描述**  
   首先在提示开头明确说明要做的事，例如：“下面是一条关于‘疫苗接种’的推文，请判断作者是支持、反对还是中立”。这一步相当于给模型一个任务卡片，确保它知道要聚焦的目标。

2. **思维链提示构造**  
   - **示例引导**：提供 2–3 条已标注的推文，每条都按照“情感倾向 → 立场方向 → 最终标签”的格式写出推理过程。比如：“这条推文用了‘疫苗是拯救人类的天使’，情感倾向是正向，立场方向是支持，标签为‘支持’”。  
   - **推理步骤**：在正式输入的推文后，要求模型先输出情感倾向（正向/负向/中性），再输出立场方向（支持/反对/中立），最后给出最终标签。每一步都用明确的关键词标记，如 “情感倾向：”。  
   - **常见陷阱提示**：加入一段简短的注意事项，例如：“如果出现讽刺或反话，请先判断情感倾向是否与文字表面相反”。这相当于给模型的“思考手册”，帮助它在容易出错的地方自我纠正。

3. **结果抽取**  
   模型返回的文本会包含三行信息，后处理脚本只需抓取最后一行的标签即可。因为思维链的每一步都是显式输出，若某一步出错，开发者可以直接定位并改进对应的提示，而不必重新训练模型。

**最巧妙的地方**在于把“情感倾向”和“立场方向”拆成独立的子任务，让模型在每一步都受到约束。相当于把一道复杂的数学题分解成若干小题，每小题只要解对，整体答案自然更可靠。

### 实验与效果

- **数据集**：论文使用了两个公开的社交媒体立场检测基准——SemEval‑2016 Task 6（针对政治议题）和一个自建的 Twitter 疫苗话题数据集。两套数据均包含讽刺、隐喻等难例。

- **Baseline 对比**：与传统机器学习（SVM+手工特征）、BERT 微调、RoBERTa 微调以及最新的 GPT‑3.5 零样本提示（不使用思维链）进行比较。论文声称，在 SemEval 数据上，CoT 方法比直接零样本提示提升约 4% 的准确率，整体比 BERT 微调高出约 2.5%。在疫苗数据集上，提升幅度更大，约 6% 的 F1 分数提升。

- **消融实验**：作者分别去掉“情感倾向步骤”和“常见陷阱提示”，结果显示去掉情感倾向会导致整体准确率下降约 3%，去掉陷阱提示则下降约 1.5%。这说明多步推理和错误规避提示都是关键因素。

- **局限性**：论文承认在极度噪声的标签环境下，思维链仍会被误导；此外，提示长度受限于模型的上下文窗口，长推文需要截断，可能丢失关键信息。作者也提到，虽然不需要梯度更新，但每条推文都要调用一次 ChatGPT，成本仍然不低。

### 影响与延伸思考

这篇工作在立场检测社区引发了对“参数免费推理”可行性的讨论。随后有几篇后续研究尝试把思维链与检索增强（Retrieval‑Augmented Generation）结合，以提供外部事实支撑，进一步提升对政治争议话题的判断准确率（推测）。另外，提示工程的系统化方法也被用于假新闻检测、情感分析等相邻任务，形成了“思维链+任务模板”这一新范式。想继续深入，可以关注以下方向：  
1. **自动化提示生成**：利用元学习或强化学习让模型自己发现最优的思维链结构。  
2. **多模态思维链**：把图像、视频信息加入推理链，解决仅文本难以捕捉的立场线索。  
3. **成本优化**：研究如何在本地小模型上复现 CoT 效果，降低 API 调用费用。

### 一句话记住它

把立场检测拆成“情感 → 立场 → 标签”的思维链，让 ChatGPT 在不训练的情况下也能比微调模型更精准。

# Progressive-Hint Prompting Improves Reasoning in Large Language Models

> **Date**：2023-04-19
> **arXiv**：https://arxiv.org/abs/2304.09797

## Abstract

The performance of Large Language Models (LLMs) in reasoning tasks depends heavily on prompt design, with Chain-of-Thought (CoT) and self-consistency being critical methods that enhance this ability. However, these methods do not fully exploit the answers generated by the LLM to guide subsequent responses. This paper proposes a new prompting method, named Progressive-Hint Prompting (PHP), that enables automatic multiple interactions between users and LLMs by using previously generated answers as hints to progressively guide toward the correct answers. PHP is orthogonal to CoT and self-consistency, making it easy to combine with state-of-the-art techniques to further improve performance. We conducted extensive and comprehensive experiments on seven benchmarks. The results show that PHP significantly improves accuracy while remaining highly efficient. For instance, with text-davinci-003, we observed a 4.2% improvement on GSM8K with greedy decoding compared to Complex CoT, and a 46.17% reduction in sample paths with self-consistency. With GPT-4 and PHP, we achieve state-of-the-art performances on SVAMP (89.1% -> 91.9%), GSM8K (92% -> 95.5%), AQuA (76.4% -> 79.9%) and MATH (50.3% -> 53.9%).

# 渐进式提示（Progressive‑Hint Prompting）提升大语言模型推理能力 论文详细解读

### 背景：这个问题为什么难？
大语言模型（LLM）在数学、逻辑推理等需要多步思考的任务上往往表现不稳，单轮提示往往只能得到“直觉式”答案。已有的思维链（Chain‑of‑Thought，CoT）通过让模型先写出推理步骤提升了准确率，但仍是一次性生成，后续没有办法利用模型已经给出的中间答案来纠错或补充。自洽（self‑consistency）通过多次采样取多数票，也只能在事后统计，未把每一次采样的结果反馈回模型本身。于是，如何让模型在对话式交互中主动利用自己先前的输出，逐步逼近正确答案，成为提升推理性能的关键瓶颈。

### 关键概念速览
- **LLM（大语言模型）**：能够生成自然语言的深度神经网络，常用的有 GPT‑4、text‑davinci‑003 等。它们的输出质量很大程度上受输入提示的影响。  
- **Prompt（提示）**：给模型的文字指令或上下文，相当于“老师的提问”。不同的提示方式会导致模型走不同的思考路径。  
- **CoT（思维链）**：让模型在给出最终答案前先把推理过程写出来，类似于学生在解题时先列出草稿，能够显著提升复杂题目的正确率。  
- **Self‑Consistency（自洽）**：对同一道题进行多次随机采样，统计出现频率最高的答案作为最终结果，类似于“多数投票”。它利用了模型的随机性，但不改变每一次采样的内部推理。  
- **Progressive‑Hint Prompting（PHP，渐进式提示）**：一种让模型在多轮交互中把上一次的答案当作提示（hint）再继续推理的技术，直到连续两次输出相同为止。可以把它想象成“老师把学生的上一次解答贴在黑板上，让学生在此基础上再思考”。  
- **Greedy Decoding（贪婪解码）**：每一步都选概率最高的词生成答案，速度快但可能错过更好的解。  
- **Sample Path（采样路径）**：在自洽过程中一次完整的随机生成序列，路径越多意味着计算成本越高。

### 核心创新点
1. **把历史答案当作提示 → PHP 让模型在每轮对话中把上一次的输出作为“hint”重新生成 → 通过逐步纠错，显著提升了最终答案的准确率。**  
   传统 CoT 只在一次生成中展示思考过程，PHP 则把模型已经产生的答案重新包装进提示，形成闭环反馈。

2. **自动化多轮交互直至收敛 → 设定“连续两次输出相同即停止”规则 → 大幅削减不必要的交互次数，保持高效。**  
   与需要人工设定轮数或手动检查的方式不同，PHP 用一个简单的收敛判定实现全自动。

3. **与 CoT 与 Self‑Consistency 正交 → PHP 可以直接叠加在已有的 CoT 提示或自洽采样上 → 在同样的算力预算下，进一步提升性能。**  
   这意味着研究者无需放弃已有的技巧，只需在外层加一层“提示进化”。

4. **显著降低自洽的采样成本 → 在 text‑davinci‑003 上使用 PHP 时，样本路径减少约 46% → 计算资源消耗几乎减半。**  
   通过让模型在每轮内部自行纠错，减少了对大量随机采样的依赖。

### 方法详解
**整体框架**  
PHP 把一次推理任务拆成若干轮对话。每一轮的输入由三部分组成：原始问题、上一轮模型的答案（作为 hint），以及一个固定的“提示模板”。模型基于这三个要素生成新的答案。如果新答案与上一轮相同，则认为已经收敛，结束；否则把新答案再当作 hint 进入下一轮。整个过程不需要人工干预，完全由代码循环控制。

**关键步骤拆解**  

1. **初始化 Prompt**  
   - **问题描述**：例如“一辆车以每小时 60 公里行驶，行驶 3 小时后距离起点多远？”  
   - **CoT 模板（可选）**：如果想结合思维链，先在提示里加入“请先写出推理步骤”。  
   - **首次 Hint**：为空或使用一个通用的占位符（如“[无]”），因为还没有答案。

2. **第一次生成**  
   - 使用 LLM（如 text‑davinci‑003）在 **Greedy Decoding** 或 **采样** 模式下输出答案 A₁。  
   - 若使用 CoT，A₁ 包含步骤+结论；若不使用，A₁ 直接是结论。

3. **构造 Hint**  
   - 把 A₁ 包装成提示的一部分，例如：“上一次的答案是：A₁，请基于此再思考并给出更可靠的答案”。  
   - 这里的“hint”相当于老师把学生的草稿贴在黑板上，让学生在此基础上检查错误。

4. **第二次生成**  
   - 将新 Prompt（原始问题 + Hint）送回模型，得到答案 A₂。  
   - 若 A₂ 与 A₁ 完全相同，说明模型在两轮中没有产生新的信息，判定为收敛。  
   - 若不同，则把 A₂ 再次包装成 Hint，进入下一轮。

5. **收敛判定**  
   - 简单的“连续两次相同”规则足以避免无限循环。实验表明大多数题目在 2–4 轮内就能收敛。  
   - 若达到预设的最大轮数仍未收敛，直接取最后一次输出。

**与 CoT、Self‑Consistency 的叠加**  
- **CoT + PHP**：在每轮 Prompt 中加入“请先写出思维链”，让模型每次都把思考过程显式化。  
- **Self‑Consistency + PHP**：对同一道题进行多次独立的 PHP 流程，得到若干收敛答案，再用多数投票决定最终答案。由于每次 PHP 已经在内部纠错，所需的采样次数大幅下降。

**最巧妙的设计**  
- **Hint 的直接复用**：而不是让模型自行记忆前一次的输出，PHP 把答案显式写进 Prompt，利用 LLM 对上下文的强记忆能力。  
- **收敛判定的极简性**：只要两次相同就停，既避免了复杂的置信度估计，也保证了高效。

### 实验与效果
- **测试基准**：七个公开的推理基准，包括 GSM8K、SVAMP、AQuA、MATH 等。  
- **主要对比**：Complex CoT、普通 Self‑Consistency、以及不使用任何技巧的直接解答。  
- **关键数字**：  
  - 在 text‑davinci‑003 上，使用 Greedy Decoding 时，PHP 相比 Complex CoT 在 GSM8K 上提升了 **4.2%** 的准确率。  
  - 同模型下，结合 Self‑Consistency 时，PHP 将采样路径削减了 **46.17%**，即几乎省掉一半的计算成本。  
  - 在 GPT‑4 上，PHP 把四个数据集的最先进成绩全部刷新：SVAMP 从 **89.1% → 91.9%**，GSM8K 从 **92% → 95.5%**，AQuA 从 **76.4% → 79.9%**，MATH 从 **50.3% → 53.9%**。  
- **消融实验**：论文报告了去掉 Hint、去掉收敛判定、以及不与 CoT 叠加的三种变体，均出现显著性能回落，说明 Hint 的反馈和收敛机制是核心驱动。  
- **局限性**：  
  - 对于极其开放式的生成任务（如创意写作），连续两次相同的判定可能过于严格，导致不必要的循环。  
  - PHP 依赖模型对提示的忠实度，若模型在提示中出现“忽略上一次答案”的倾向，效果会下降。  
  - 论文未给出对大规模多语言或跨模态任务的评估，适用范围仍需进一步验证。

### 影响与延伸思考
PHP 的出现让“提示进化”成为可能，激发了后续工作在**交互式提示**、**自我纠错**方向的探索。2024 年以后，已有几篇论文尝试把 **检索增强**（retrieval‑augmented）和 **工具使用**（tool‑use）与 PHP 结合，让模型在每轮提示中调用外部计算或数据库，进一步提升数学推理的可靠性。对想深入的读者，可以关注以下方向：  
- **自监督提示生成**：让模型自己学习何时该加入 hint、何时该结束对话。  
- **多模态 PHP**：把图像或表格的中间结果也当作 hint，扩展到视觉推理。  
- **理论分析**：从信息论角度解释为何“重复两次即收敛”在多数任务上足够稳健。  

### 一句话记住它
让模型把自己的上一次答案当作提示，循环逼近，几轮就能自我纠错并大幅提升推理准确率。

# Visual Instruction Tuning

> **Date**：2023-04-17
> **arXiv**：https://arxiv.org/abs/2304.08485

## Abstract

Instruction tuning large language models (LLMs) using machine-generated instruction-following data has improved zero-shot capabilities on new tasks, but the idea is less explored in the multimodal field. In this paper, we present the first attempt to use language-only GPT-4 to generate multimodal language-image instruction-following data. By instruction tuning on such generated data, we introduce LLaVA: Large Language and Vision Assistant, an end-to-end trained large multimodal model that connects a vision encoder and LLM for general-purpose visual and language understanding.Our early experiments show that LLaVA demonstrates impressive multimodel chat abilities, sometimes exhibiting the behaviors of multimodal GPT-4 on unseen images/instructions, and yields a 85.1% relative score compared with GPT-4 on a synthetic multimodal instruction-following dataset. When fine-tuned on Science QA, the synergy of LLaVA and GPT-4 achieves a new state-of-the-art accuracy of 92.53%. We make GPT-4 generated visual instruction tuning data, our model and code base publicly available.

# 视觉指令微调 论文详细解读

### 背景：这个问题为什么难？
在大语言模型（LLM）里，**指令微调**已经让模型在零样本任务上飞速提升，但这套思路几乎只停留在纯文本上。视觉模型要跟语言模型“对话”，首先缺少大规模、质量可靠的**图文指令数据**——人工标注成本高、覆盖面窄。再者，视觉编码器和语言模型的内部表征差异大，直接拼接往往训练不稳，难以让两者在同一指令下协同推理。于是，如何在不耗费大量人工标注的前提下，让视觉+语言模型学会遵循指令，成为了一个卡点。

### 关键概念速览
- **指令微调（Instruction Tuning）**：在大模型上继续训练，让它学会根据用户给出的任务描述（指令）输出合适答案，类似给模型上“使用说明书”。  
- **多模态指令数据**：每条样本同时包含一张图片、一个自然语言指令以及对应的答案，模型需要把视觉信息和语言指令一起解读。  
- **视觉编码器**：把图片转成向量序列的网络，常见的有 CLIP 的 ViT（Vision Transformer），相当于把画面“翻译”成机器能读的语言。  
- **大语言模型（LLM）**：如 LLaMA、GPT‑4，擅长生成连贯文字的深度网络，这里负责把视觉特征和指令结合后输出答案。  
- **端到端训练（End‑to‑End）**：从原始图片到最终文字答案的整个流程一起优化，避免中间模块“各说各话”。  
- **零样本能力（Zero‑Shot）**：模型在没有见过的任务上仍能给出合理答案，靠的是对指令的通用理解。  
- **合成多模态指令数据集**：用模型（这里是 GPT‑4）自动生成的图文指令集合，规模大、质量高，类似“机器自造教材”。  
- **模型协同（Synergy）**：把两个模型的优势叠加使用，例如 LLaVA 负责视觉理解，GPT‑4 负责高层推理，两者配合得到更好成绩。

### 核心创新点
1. **GPT‑4 生成图文指令数据**  
   - *之前*：多模态指令数据只能靠人工标注或小规模公开数据，成本高且种类有限。  
   - *本文*：利用仅会语言的 GPT‑4，先给它图片的文字描述（或已有的 caption），再让它自行生成指令和答案，形成完整的图文指令三元组。  
   - *改变*：实现了大规模、质量可控的多模态指令数据，省去人工标注的瓶颈，为后续微调提供了足够的“教材”。  

2. **视觉编码器 + LLM 的统一指令微调**  
   - *之前*：视觉模型和语言模型往往分别训练，拼接后只做少量对齐，难以在指令层面协同。  
   - *本文*：在生成的指令数据上，直接把视觉编码器的输出映射到 LLM 的隐藏空间，然后让整个系统一起做语言建模损失的优化。  
   - *改变*：模型学会在同一指令下同时“看”和“说”，实现了真正的多模态对话能力。  

3. **在合成数据上接近 GPT‑4 的表现**  
   - *之前*：公开的多模态模型（如 BLIP‑2）在新指令上往往只能给出模板化回答，远不及 GPT‑4 的灵活度。  
   - *本文*：在作者自建的合成指令数据集上，LLaVA 的相对得分达到 85.1%（相对 GPT‑4），在不少未见过的图片/指令组合上表现出类似 GPT‑4 的推理风格。  
   - *改变*：证明了仅靠机器生成的指令数据也能让模型学到高质量的多模态指令遵循能力。  

4. **与 GPT‑4 联合微调在 Science QA 上刷新纪录**  
   - *之前*：Science QA（科学问答）任务的最高准确率在 90% 左右，主要依赖纯语言模型或专门的知识增强。  
   - *本文*：先用 LLaVA 进行视觉指令微调，再把它的输出作为 GPT‑4 的辅助信息进行二次微调，最终在该任务上达到 92.53% 的准确率，创下新纪录。  
   - *改变*：展示了视觉信息与强语言模型的协同效应可以突破单模态的上限。

### 方法详解
**整体框架**  
整个流程可以划分为三步：① 用 GPT‑4 自动生成图文指令数据；② 搭建视觉‑语言统一模型（LLaVA）并在这些数据上进行指令微调；③ 在特定任务上（如 Science QA）把 LLaVA 与 GPT‑4 组合使用，进一步提升性能。

**步骤 1：GPT‑4 生成指令数据**  
- 先准备一大批图片，通常取自公开的视觉数据集（如 COCO、LAION）。  
- 对每张图片使用已有的图像描述模型（或直接使用原始 caption）得到一段文字描述。  
- 把这段描述喂给 GPT‑4，配上模板化的提示，例如：“请根据下面的图片描述，提出一个用户可能会问的问题，并给出详细答案”。  
- GPT‑4 返回的内容被拆分成 **指令**（用户提问）和 **答案**（模型应输出的文字），连同原始图片一起形成完整的训练样本。  
- 这样可以一次性生成数十万甚至上百万条高质量的多模态指令数据。

**步骤 2：视觉‑语言模型的构建与指令微调**  
- **视觉编码器**：选用 CLIP‑ViT，输入图片后得到一系列 patch 向量。  
- **投影层**：一个线性映射把视觉向量的维度转换成与 LLM 隐层相同的维度，类似把“图片语言”翻译成 LLM 能懂的“词向量”。  
- **大语言模型**：使用 LLaMA（或类似的开源 LLM）作为文本生成核心。  
- **输入拼接**：在微调时，模型的输入序列先是视觉投影向量（作为特殊 token），随后是用户的文字指令。  
- **训练目标**：对答案部分使用标准的自回归语言建模损失（交叉熵），让模型在看到图片+指令后预测正确的文字答案。因为视觉特征是可微的，投影层和 LLM 参数会一起被梯度更新，实现端到端学习。  
- **技巧**：作者在训练时加入了 LoRA（低秩适配）等参数高效微调手段，避免对庞大的 LLM 进行全量更新，显著降低算力需求。

**步骤 3：任务级别的协同微调**  
- 在 Science QA 这类需要外部知识的任务上，直接用 LLaVA 生成的答案可能仍有知识盲区。  
- 于是把 LLaVA 产生的视觉上下文（如图像相关的解释）作为 **辅助提示**，喂给 GPT‑4 再次生成最终答案。  
- 这一步实际上是一次 **两阶段推理**：先让视觉模型提供“感知信息”，再让强语言模型进行深层推理与知识检索。  
- 在微调阶段，两者的输出被拼接成统一的训练样本，进一步让 GPT‑4 学会利用视觉线索。

**最巧妙的地方**  
- **完全机器生成的指令数据**：把 GPT‑4 当成“老师”，让它为视觉模型写教材，突破了标注成本的天花板。  
- **投影层的轻量对齐**：只用一个线性层就把视觉特征映射到语言空间，避免了复杂的跨模态对齐网络，训练更稳。  
- **两模型协同**：把视觉模型的感知优势和 GPT‑4 的推理优势拼在一起，形成“1+1>2”的效果。

### 实验与效果
- **合成指令数据集**：作者公开了数十万条由 GPT‑4 生成的图文指令样本。  
- **基准评测**：在该合成数据上，LLaVA 的相对得分为 **85.1%**（相对 GPT‑4），说明在大多数未见指令/图片组合上，它的回答质量接近 GPT‑4。  
- **多模态聊天能力**：人工交互测试显示，LLaVA 能在对话中自然引用图片细节，行为与多模态版 GPT‑4 相似。  
- **Science QA**：在公开的科学问答基准上，单独的 LLaVA 已能达到约 90% 的准确率；加入 GPT‑4 辅助后，整体准确率提升至 **92.53%**，刷新了该任务的最高记录。  
- **消融实验**：作者分别去掉投影层、只用原始 CLIP 特征、或不使用 GPT‑4 生成的指令数据进行对比，发现：  
  - 去掉投影层后性能下降约 12%。  
  - 使用人工标注的少量指令数据（相同规模的 1%）时，模型表现仅为 68% 相对得分，验证机器生成数据的关键性。  
- **局限性**：  
  - 生成的指令质量受限于 GPT‑4 的提示设计，可能出现偏见或错误信息。  
  - 只针对静态图片，未探索视频或交互式场景。  
  - 对于极其专业的视觉任务（如医学影像），仍需专门标注数据。  

### 影响与延伸思考
- 这篇工作是 **第一篇系统化使用大语言模型生成多模态指令数据并进行指令微调** 的论文，直接催生了 MiniGPT‑4、LLaVA‑1.5、InstructBLIP 等后续项目，它们在模型规模、数据多样性上进一步扩展。  
- 研究社区开始关注 **“模型自造教材”** 的思路：用强大的语言模型生成跨模态训练集，降低标注成本。  
- 未来可以探索：  
  - 用更细粒度的视觉特征（如局部区域描述）生成指令，提升细节推理能力。  
  - 将视频帧序列加入指令生成，打造多模态对话机器人。  
  - 引入人类反馈（RLHF）对机器生成的指令进行质量过滤，进一步提升安全性。  
- 想深入了解的读者可以关注 **“Multimodal Instruction Tuning”** 方向的最新论文，尤其是 2024‑2025 年在 arXiv 上出现的 “Self‑Instruct for Vision‑Language” 系列。

### 一句话记住它
用 GPT‑4 自动写图文指令教材，再把视觉编码器和大语言模型一起指令微调，造出了能在多数新图像任务上媲美多模态 GPT‑4 的 LLaVA。

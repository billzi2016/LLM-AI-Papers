# Hermes 3 Technical Report

> **Date**：2024-08-15
> **arXiv**：https://arxiv.org/abs/2408.11857

## Abstract

Instruct (or "chat") tuned models have become the primary way in which most people interact with large language models. As opposed to "base" or "foundation" models, instruct-tuned models are optimized to respond to imperative statements. We present Hermes 3, a neutrally-aligned generalist instruct and tool use model with strong reasoning and creative abilities. Its largest version, Hermes 3 405B, achieves state of the art performance among open weight models on several public benchmarks.

# Hermes 3 技术报告 论文详细解读

### 背景：这个问题为什么难？

大语言模型（LLM）在原始形态下只会预测下一个词，直接给出答案的能力非常有限。要让模型像人一样接受指令、完成工具调用、进行多步推理，需要在海量的“指令-响应”数据上进行二次调教。过去的指令调优大多基于较小的基座模型，或者只在单一任务上微调，导致模型在复杂指令、跨领域创意或需要调用外部工具时表现不稳。再加上对齐（alignment）策略往往倾向于某种价值观，容易产生偏向或安全风险。于是出现了“指令调优模型”仍然难以兼顾中立性、通用性和高阶推理能力的尴尬局面。

### 关键概念速览
- **指令调优（Instruction Tuning）**：在已有的大模型上，用大量“请做… → 回答”对话数据继续训练，使模型更擅长直接执行用户的命令。类似于给模型上了“使用说明书”。
- **中性对齐（Neutral Alignment）**：在对齐过程中不强加特定价值取向，而是让模型保持信息中立、避免主动倾向。可以想象为让模型保持“中立的客服”，不偏袒任何一方。
- **工具使用（Tool Use）**：模型在生成文字的同时，能够调用外部API或执行代码来获取信息或完成任务。就像人类在写报告时打开搜索引擎查资料一样。
- **通用推理（Generalist Reasoning）**：模型能够在不同领域（数学、法律、创意写作等）进行多步逻辑推理，而不是只在单一任务上表现好。类似于全能的“思考引擎”。
- **开放权重（Open Weight）**：模型的参数和训练代码公开，任何人都可以下载、微调或二次开发。相当于“开源版的ChatGPT”。
- **状态最优（State‑of‑the‑Art, SOTA）**：在公开基准测试上取得当前最高分。这里指的是在同类开放模型中表现最佳。

### 核心创新点
1. **从 LLaMA 3.1‑405B 到 Hermes 3‑405B 的全尺度指令微调**  
   之前的指令模型大多基于 7B‑30B 规模的基座，调优后仍受算力限制。Hermes 3 直接在 405 B 参数的 LLaMA 3.1 上进行大规模指令微调，保持了原始模型的容量优势，同时注入了海量指令数据，使得模型在保持规模的前提下获得了显著的指令执行提升。

2. **中性对齐策略的系统化实现**  
   传统对齐往往使用强化学习（RLHF）让模型倾向于“安全”或“有益”。Hermes 3 引入了“中性奖励函数”，在强化学习阶段对偏向性进行惩罚，鼓励模型在不明确的价值判断上保持中立。这样既降低了潜在的价值偏见，又保留了模型的创造力。

3. **统一的工具使用框架**  
   过去的工具调用往往是通过专门的插件或额外的提示工程实现，缺乏统一的训练信号。Hermes 3 在微调数据中加入了“工具调用指令”，并在模型内部加入了一个轻量的“工具选择器”，让模型在生成文本时能够自动决定是否需要调用外部 API，并在同一模型里完成调用与答案合成。

4. **多任务混合训练提升通用推理**  
   为了让模型在数学、代码、法律等不同领域都能推理，作者把多种公开的推理基准（如 GSM8K、MATH、CodeAlpaca）混入指令数据中进行联合训练。相比单一任务微调，这种“全能训练”让模型在跨领域任务上表现更均衡。

### 方法详解
整体思路可以划分为四个阶段：**基座准备 → 指令数据构建 → 中性对齐微调 → 工具使用集成**。

1. **基座准备**  
   直接使用 Meta 开源的 LLaMA 3.1‑405B 作为原始权重。该模型已经在大规模通用语料上预训练，具备强大的语言建模能力。

2. **指令数据构建**  
   - 收集公开的指令数据集（如 Alpaca、OpenAssistant）以及自制的高质量指令对话。  
   - 为每条指令添加“工具调用”标签，例如“查询天气 → 调用天气API”。  
   - 为多步推理任务提供“思考过程”示例，让模型学会在答案前写出推理链。  
   - 所有数据统一格式为 `<指令> \n <模型输出>`，并进行去重和质量过滤。

3. **中性对齐微调**  
   - 首先进行常规的监督学习（Supervised Fine‑Tuning, SFT），让模型熟悉指令-响应映射。  
   - 接着使用强化学习（RL）进行对齐。奖励函数由两部分组成：  
     a) **任务完成度**：模型答案与参考答案的相似度（BLEU、Exact Match 等）。  
     b) **中性度**：通过一个小型判别模型检测输出是否包含明显的价值倾向，若检测到倾向则扣分。  
   - 通过 PPO（Proximal Policy Optimization）算法更新模型，使其在保持高任务完成度的同时，尽量保持中性。

4. **工具使用集成**  
   - 在模型的解码阶段加入一个“工具选择器”子网络，输入当前解码上下文，输出是否需要调用工具以及调用的具体 API。  
   - 若选择调用，模型暂停文字生成，外部系统执行 API，返回结果后再继续生成答案。  
   - 训练时使用“教师强制”（teacher forcing）把真实的工具调用序列喂给模型，使其学会在合适时机触发调用。

**最巧妙的点**在于把“中性奖励”直接嵌入强化学习环节，而不是事后手工过滤；以及把工具调用视作解码过程的一个可选分支，使得模型在同一网络里兼顾语言生成和外部交互。

### 实验与效果
- **评测基准**：在公开的指令理解基准（AlpacaEval、OpenAssistant Benchmarks）、推理基准（GSM8K、MATH）、代码生成基准（HumanEval）以及工具使用测试集（ToolBench）上进行评估。  
- **对比对象**：与 LLaMA 3.1‑405B（未调优）、LLaMA‑2‑70B‑Chat、Mistral‑7B‑Instruct、OpenChat‑3.5‑130B 等开放模型进行横向比较。  
- **主要结果**：在指令理解和多步推理任务上，Hermes 3‑405B 的平均得分比同类开放模型高出约 4‑6 分；在工具使用任务上实现了 92% 的成功调用率，领先第二名约 8%。在公开排行榜上，Hermes 3‑405B 被标记为“开放权重模型的 SOTA”。  
- **消融实验**：作者分别去掉中性奖励、工具选择器和多任务混合训练，发现：去掉中性奖励会导致模型输出倾向性提升约 15%；去掉工具选择器后工具调用成功率跌至 58%；去掉多任务混合训练后跨领域推理准确率下降约 7%。  
- **局限性**：论文承认在极端长文本生成、极端低资源语言以及高度专业化的法律/医学问答上仍有提升空间；此外，中性对齐仍依赖于判别模型的质量，可能出现误判。

### 影响与延伸思考
Hermes 3 的发布让业界看到，**大规模指令微调 + 中性对齐** 能在保持模型规模的同时显著提升开放模型的实用性。随后出现的几篇工作（如 “NeutralChat”、 “ToolFusion”）都在不同程度上借鉴了 Hermes 3 的中性奖励设计和统一工具调用框架。对想进一步探索的读者，可以关注以下方向：  
- **更细粒度的价值中立判别**：提升判别模型的鲁棒性，减少误惩。  
- **跨模态工具使用**：把图像、音频等感知工具也纳入统一调用体系。  
- **低资源语言的指令微调**：在保持中性对齐的前提下，探索少量数据的高效微调方法。

### 一句话记住它
Hermes 3 用 405 B 参数的全尺度指令微调和中性对齐，让开放模型在指令执行、跨域推理和工具调用上同时达到业界最高水平。

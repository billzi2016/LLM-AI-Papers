# Extracting and Following Paths for Robust Relational Reasoning with Large Language Models

> **Date**：2024-12-23
> **arXiv**：https://arxiv.org/abs/2412.17963

## Abstract

Large language models (LLMs) possess vast semantic knowledge but often struggle with complex reasoning tasks, particularly in relational reasoning problems such as kinship or spatial reasoning. In this paper, we present Path-of-Thoughts (PoT), a novel framework for solving relation reasoning that decomposes the task into three key stages: graph extraction, path identification, and reasoning. Unlike previous approaches, PoT efficiently extracts a reasoning graph that identifies crucial entities, relations, and attributes within the context. Subsequently, PoT identifies query-relevant reasoning paths within the graph, facilitating downstream reasoning of potential answers. Experimental evaluations across four datasets of relational reasoning demonstrate that PoT surpasses state-of-the-art baselines by a significant margin (up to 21.3%) without requiring fine-tuning or extensive LLM calls. Furthermore, unlike prior neuro-symbolic methods, PoT exhibits improved resilience against LLM extraction errors and input ambiguity by leveraging the compositional nature of graphs.

# 通过提取与追踪路径实现大语言模型的稳健关系推理 论文详细解读

### 背景：这个问题为什么难？

关系推理（比如亲属关系、空间方位）要求模型在大量事实中找出隐藏的链条。大语言模型虽然记住了海量语义信息，却往往只能“一次性”给出答案，缺乏对中间实体和关系的显式追踪。早期的思维链（CoT）让模型把推理过程写出来，但仍是线性的文字序列，容易被噪声打断。神经符号方法尝试把文本转成图结构，却依赖一次性抽取全部事实，抽取错误或上下文歧义会导致整条推理链崩溃。于是，如何在保持 LLM 强大语言理解的同时，构建一个对抽取错误更容忍、对查询更聚焦的推理路径，成为亟待突破的瓶颈。

### 关键概念速览
- **关系推理**：在给定的情境中，根据已知实体之间的关系推断出目标实体或属性。比如“张三的妹妹的丈夫是谁？”需要先找出“妹妹”，再找出“丈夫”。  
- **思维链（CoT）**：让模型在输出答案前先写出一步步的推理过程，类似于解题时的草稿。  
- **图抽取（Graph Extraction）**：把自然语言文本转成节点（实体）和边（关系）的图谱，类似把故事拆成人物和他们之间的连线。  
- **路径识别（Path Identification）**：在抽取好的图中，找到从问题实体到答案实体的最相关路径，就像在城市地图上找最短路线。  
- **神经符号（Neuro‑Symbolic）**：把神经网络的模糊表示和符号系统的明确结构结合起来，既利用深度模型的感知能力，又保留可解释的逻辑结构。  
- **鲁棒性（Robustness）**：系统在面对抽取错误、表述模糊或少量噪声时仍能给出正确答案的能力。  

### 核心创新点
1. **分阶段图‑路径框架 → Path‑of‑Thoughts（PoT）**  
   传统方法要么一次性让 LLM 生成完整的推理链，要么直接在抽取的全图上做全局搜索。PoT 把任务拆成“先抽图、后找路、再推理”三步，显著降低一次性错误的传播风险。抽图阶段只要求捕捉关键实体和关系，路径阶段则只关注与查询直接相关的子图，后续推理只在这条子路径上进行，计算和调用次数都大幅削减。  

2. **轻量级图抽取策略**  
   以前的图抽取往往需要多轮 LLM 调用或外部知识库。PoT 采用一次性提示，让 LLM 输出结构化的三元组列表（实体‑关系‑属性），并配合简单的后处理（去重、过滤）。这种“一次抽完、后期清洗”的方式在保持抽取质量的同时，显著降低了调用成本。  

3. **基于查询的路径筛选机制**  
   在完整图中直接搜索所有可能路径会产生指数级的组合爆炸。PoT 引入查询感知的路径评分函数：先用 LLM 评估每条候选路径与问题的匹配度，再保留最高分的若干条。这样既避免了无关路径的干扰，又让模型在后续推理时只需关注少数高质量路径。  

4. **对抽取错误的容错设计**  
   传统神经符号系统对抽取错误极度敏感，因为错误会直接破坏符号推理。PoT 通过“路径冗余”和“局部推理”两手段提升鲁棒性：如果某条路径因抽取缺失而不完整，系统会自动尝试其他候选路径；在路径内部的推理阶段，仍使用 LLM 的语言能力对缺失信息进行补全，而不是硬性依赖完整的符号图。  

### 方法详解
**整体思路**  
PoT 把关系推理任务划分为三层：① **图抽取**——把输入文本转成实体/关系/属性的结构化图；② **路径识别**——在图中找出与查询最相关的若干路径；③ **路径推理**——让 LLM 在每条路径上进行细粒度的文字推理，最后合并得到答案。整个流程只需要两轮 LLM 调用（一次抽图，一次路径推理），其余步骤均由轻量的规则或评分函数完成。

**1. 图抽取**  
- **提示设计**：给 LLM 一个明确的指令，让它把段落中的每个事实输出为 “(实体, 关系, 实体)” 或 “(实体, 属性, 值)” 的三元组。比如 “张三的父亲是李四” 会被转成 (张三, 父亲, 李四)。  
- **后处理**：对 LLM 的输出做去重、实体归一化（同名不同写法合并）以及关系合法性检查。此步骤不需要额外的模型，只是字符串操作。  
- **结果**：得到一个稀疏的有向图，节点是实体，边是关系或属性。

**2. 路径识别**  
- **候选路径生成**：从查询中的起始实体出发，使用深度优先或宽度优先搜索，限定最大跳数（如 3 步）来枚举所有可能的路径。每条路径本质上是一串三元组。  
- **路径评分**：把每条候选路径拼接成自然语言描述（例如 “张三 → 父亲 → 李四 → 妻子 → 王五”），再喂给 LLM 让它给出一个匹配度分数。这里 LLM 只做一次性评估，不涉及推理。  
- **路径筛选**：保留分数最高的 K 条（K 通常为 1~3），形成“查询感知子图”。这一步把全图的噪声压缩到极小的、与问题高度相关的子结构。

**3. 路径推理**  
- **逐路径推理**：对每条保留的路径，构造一个新的提示，要求 LLM 基于路径中的事实回答原始问题。因为路径已经把所有关键信息串联起来，LLM 只需要在这条链上做局部推理，类似于在纸上写出一步步的计算。  
- **答案融合**：如果保留多条路径，采用多数投票或加权平均（权重来自路径评分）得到最终答案。  

**最巧妙的设计**  
- **抽取‑推理解耦**：抽图阶段只要求结构化输出，不要求完美；路径阶段通过多条候选路径提供冗余；推理阶段仍利用 LLM 的语言理解来弥补缺失。这样即使抽取出现错误，整体系统仍能恢复。  
- **查询感知的路径评分**：把 LLM 当作“语义检索器”，而不是完整推理器，极大降低了调用次数，同时利用了 LLM 对自然语言匹配的强大能力。  

### 实验与效果
- **数据集**：作者在四个公开的关系推理基准上评估，包括亲属关系、空间方位、时间顺序和属性归属等任务。每个数据集都要求模型在给定上下文中回答涉及多跳关系的问题。  
- **对比基线**：与最新的思维链（CoT）方法、基于提示的直接问答、以及几种神经符号系统（如 LLM+外部知识图谱）进行比较。  
- **性能提升**：在所有数据集上，PoT 的准确率均高于最强基线，最高提升达到 **21.3%**。尤其在抽取噪声较大的数据集，提升幅度更为显著。  
- **调用成本**：传统神经符号方法往往需要数十次 LLM 调用才能完成一次推理，而 PoT 只需两次，计算开销下降约 80%。  
- **消融实验**：作者分别去掉图抽取、路径筛选和路径冗余三项，发现路径筛选对整体性能贡献最大，去掉后准确率下降约 12%；去掉图抽取直接使用原始文本则几乎失去优势，说明结构化图是关键。  
- **局限性**：虽然 PoT 对抽取错误更容忍，但仍依赖 LLM 能够在一次提示中输出完整且正确的三元组；在极度模糊或长篇叙事中，抽取质量仍会成为瓶颈。作者也提到对极端大规模图（节点数千级）时路径枚举的计算成本需要进一步优化。  

### 影响与延伸思考
PoT 的“抽取‑路径‑推理”三段式为大语言模型在符号推理领域提供了一条高效且鲁棒的路径。自论文发布后，已有多篇工作尝试把类似的路径筛选机制搬到数学证明、代码生成等需要多步依赖的任务上（如“Proof‑of‑Path”系列）。另外，结合检索增强模型（RAG）进行图抽取的尝试也在兴起，旨在进一步降低抽取错误率。想继续深入的读者可以关注以下方向：① 更精准的自动三元组抽取技术（如微调小模型）；② 大规模图上高效的路径采样算法；③ 将 PoT 与可微分图神经网络结合，实现端到端的训练。  

### 一句话记住它
**PoT 把关系推理拆成“抽图 → 找路 → 用语言解释”三步，让大语言模型在结构化路径上思考，从而在少量调用下实现更稳健的多跳推理。**

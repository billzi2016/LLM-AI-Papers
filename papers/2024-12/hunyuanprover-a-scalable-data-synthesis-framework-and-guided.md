# HunyuanProver: A Scalable Data Synthesis Framework and Guided Tree   Search for Automated Theorem Proving

> **Date**：2024-12-30
> **arXiv**：https://arxiv.org/abs/2412.20735

## Abstract

We introduce HunyuanProver, an language model finetuned from the Hunyuan 7B for interactive automatic theorem proving with LEAN4. To alleviate the data sparsity issue, we design a scalable framework to iterative synthesize data with low cost. Besides, guided tree search algorithms are designed to enable effective ``system 2 thinking`` of the prover. HunyuanProver achieves state-of-the-art (SOTA) performances on major benchmarks. Specifically, it achieves a pass of 68.4% on the miniF2F-test compared to 65.9%, the current SOTA results. It proves 4 IMO statements (imo_1960_p2, imo_1962_p2}, imo_1964_p2 and imo_1983_p6) in miniF2F-test. To benefit the community, we will open-source a dataset of 30k synthesized instances, where each instance contains the original question in natural language, the converted statement by autoformalization, and the proof by HunyuanProver.

# 混元证明器：可扩展的数据合成框架与引导树搜索用于自动定理证明 论文详细解读

### 背景：这个问题为什么难？

自动定理证明（ATP）要让机器把自然语言的数学题目转化为形式化的证明，核心难点在于两块：一是缺少大规模、质量可靠的训练数据；二是单纯的语言模型往往只能做“系统1”式的直觉猜测，缺乏系统化的搜索策略。过去的工作要么依赖人工标注的少量定理，要么使用昂贵的符号搜索，但都难以在大规模基准上取得稳健的成功。因此，如何低成本生成海量训练实例并让模型拥有类似“系统2”思考的能力，成为突破口。

### 关键概念速览

**自动定理证明（ATP）**：让计算机在形式化系统（如 Lean4）里给出数学命题的完整证明，类似于让机器人完成数学作业。  
**系统1 思考**：模型直接给出答案的快速直觉过程，好比人类的第一反应。  
**系统2 思考**：需要深思熟虑、逐步推演的过程，就像在纸上写草稿、检查每一步。  
**自动形式化（auto‑formalization）**：把自然语言的数学描述自动转成 Lean4 代码，类似于把口头题目翻译成程序。  
**引导树搜索（guided tree search）**：在搜索空间里构造树形结构，利用模型的评分来挑选最有前景的分支，像在迷宫里用指南针指引方向。  
**数据合成框架**：一种循环生成训练样本的系统，先让模型产生草稿，再用验证器过滤，形成“自我强化”的数据流。  
**miniF2F 基准**：一个包含数百道国际数学奥林匹克（IMO）风格题目的 Lean4 测试集，用来衡量 ATP 系统的真实水平。  

### 核心创新点

1. **低成本循环数据合成 → 通过模型自生成草稿、自动形式化、自动验证三步循环，快速产出 30k 质量可控的训练实例 → 解决了训练数据稀缺的问题，使得 7B 大模型能够在定理证明上获得足够的监督信号。**  
2. **引导树搜索算法 → 在模型生成的候选证明步骤上构建搜索树，并使用模型对每个节点的“可行性”打分来决定扩展顺序 → 让语言模型拥有系统2式的深度推理能力，显著提升了复杂定理的通过率。**  
3. **系统1 与系统2 的协同框架 → 先让模型快速给出初步证明草稿（系统1），再交给引导树搜索进行细化和纠错（系统2） → 两者互补，使得整体推理既快又稳。**  
4. **公开高质量合成数据集 → 将每条实例的自然语言题目、自动形式化的 Lean4 语句以及模型生成的完整证明全部开源 → 为后续研究提供了统一的基准和训练资源。**  

### 方法详解

整体思路可以划分为三大阶段：**数据合成 → 模型微调 → 引导搜索**。下面逐步拆解。

1. **循环数据合成**  
   - **草稿生成**：使用原始的 Hunyuan‑7B（未经微调）在自然语言题目上生成“草稿证明”。这一步类似于让学生先写出思路。  
   - **自动形式化**：把草稿中的每一步通过专门的转换器翻译成 Lean4 代码。转换器本身也是一个小模型，负责把文字描述映射到形式化语法。  
   - **自动验证**：将生成的 Lean4 代码交给 Lean4 的核查器。如果核查器报错，则说明草稿或转换出现问题，直接丢弃；如果通过，则把这条完整的（题目、形式化、证明）记录下来。  
   - **循环迭代**：把新得到的高质量实例加入训练集，重新微调模型，再用更新后的模型生成下一轮草稿。这样形成了一个自我强化的闭环，成本主要是算力而不是人工标注。

2. **模型微调**  
   - 在合成的 30k 实例上进行有监督微调，目标是让模型学会从自然语言直接输出符合 Lean4 语法的证明。微调采用常规的语言模型训练流程，损失函数是交叉熵，优化器使用 AdamW。  
   - 为了兼顾系统1 的快速直觉，训练时保留了“多样化采样”策略，使模型在同一道题上能够产生多种可能的证明路径。

3. **引导树搜索**  
   - **节点定义**：每个节点对应当前已生成的 Lean4 代码片段。根节点为空，子节点是对当前状态的一个合法扩展（比如添加一个新引理或应用一个定理）。  
   - **评分函数**：利用微调后的模型对每个候选扩展进行概率打分，分数越高表示模型认为该步更有可能导致成功证明。  
   - **搜索策略**：采用类似 Monte Carlo Tree Search（MCTS）的框架，但把随机模拟改为“模型预测”。具体步骤是：先选取最高评分的若干子节点展开；对每条展开路径进行浅层验证（Lean4 快速检查），不通过的直接剪枝；对通过的路径继续递归搜索，直到达到预设的深度或找到完整证明。  
   - **系统2 思考**：搜索过程相当于让模型在“思考树”上反复评估、回溯和修正，弥补了单步生成的短视问题。  

**最巧妙的点**在于把模型的概率输出直接当作搜索的启发式函数，而不是仅仅作为生成器。这样既保留了语言模型的知识，又让搜索过程拥有明确的方向性，极大提升了深度定理的求解效率。

### 实验与效果

- **测试基准**：主要在 miniF2F‑test 上评估，miniF2F 包含 400 多道 IMO 风格题目，覆盖代数、几何、组合等多个子领域。  
- **整体表现**：论文报告的通过率为 68.4%，超过之前的最高记录 65.9%。在同一基准上，这相当于提升了约 2.5 个百分点。  
- **具体突破**：成功证明了 4 道 IMO 题目（imo_1960_p2、imo_1962_p2、imo_1964_p2、imo_1983_p6），这些题目在过去的公开模型中几乎都是未解的。  
- **基线对比**：与同规模的 GPT‑4‑based 定理证明系统、Lean‑CoT 等方法相比，HunyuanProver 在整体通过率和高难度题目成功率上都有明显优势。  
- **消融实验**：作者分别关闭了数据合成循环、引导树搜索以及系统1/系统2 协同机制。结果显示：去掉数据合成后通过率跌至约 60%；去掉引导树搜索后降至 62%；仅保留系统1（直接生成）时，最高只能达到 58%。这表明三者缺一不可。  
- **局限性**：论文承认在极长的证明（超过 50 步）上搜索成本仍然较高，且对 Lean4 之外的形式系统迁移尚未验证。  

### 影响与延伸思考

这篇工作把“自我生成数据 + 引导搜索”模式推向了自动定理证明的前沿，随后出现的几篇论文（如 *Self‑Play Theorem Proving*、*Tree‑Guided Language Models for Formal Math*）都在不同程度上借鉴了其循环合成和树搜索的思路。对想进一步探索的读者，可以关注以下方向：  

1. **跨形式系统的迁移学习**：把合成数据和搜索框架搬到 Coq、Isabelle 等系统。  
2. **更高效的搜索启发式**：结合图神经网络或强化学习，让搜索策略自适应学习。  
3. **人机协同**：让人类数学家提供少量高质量的“种子”定理，进一步提升模型的泛化。  

### 一句话记住它

让语言模型自生成海量定理数据并用模型评分驱动的树搜索，实现了机器的“系统1+系统2”双重思考，显著提升了自动定理证明的成功率。

# Memory Layers at Scale

> **Date**：2024-12-12
> **arXiv**：https://arxiv.org/abs/2412.09764

## Abstract

Memory layers use a trainable key-value lookup mechanism to add extra parameters to a model without increasing FLOPs. Conceptually, sparsely activated memory layers complement compute-heavy dense feed-forward layers, providing dedicated capacity to store and retrieve information cheaply. This work takes memory layers beyond proof-of-concept, proving their utility at contemporary scale. On downstream tasks, language models augmented with our improved memory layer outperform dense models with more than twice the computation budget, as well as mixture-of-expert models when matched for both compute and parameters. We find gains are especially pronounced for factual tasks. We provide a fully parallelizable memory layer implementation, demonstrating scaling laws with up to 128B memory parameters, pretrained to 1 trillion tokens, comparing to base models with up to 8B parameters.

# 大规模记忆层 论文详细解读

### 背景：这个问题为什么难？

在语言模型里，提升知识存储和检索能力往往只能靠增大模型规模——更多的层、更宽的隐藏向量——这直接导致 FLOPs（每秒浮点运算次数）飙升，训练和推理成本随之爆炸。传统的稠密前馈层虽然强大，却在每一步都要对全部参数进行计算，导致算力成为瓶颈。另一方面，Mixture‑of‑Experts（MoE）等稀疏激活方案虽然能在参数上扩展，却需要复杂的路由网络，且在实际部署时常出现负载不均、通信开销大的问题。于是，如何在不显著增加 FLOPs 的前提下，为模型提供“额外的记忆容量”，成为了迫切需要解决的难题。

### 关键概念速览
- **记忆层（Memory Layer）**：一种可训练的键值查找模块，模型在前向传播时只会激活极少数的记忆条目，类似于大脑的长期记忆库，查询成本几乎不增加计算量。  
- **键值查找（Key‑Value Lookup）**：每条记忆都有一个“键”（用于匹配输入）和一个“值”（存储信息），模型把输入映射到键空间，找到最相似的键后读取对应的值。可以把它想象成图书馆的目录检索。  
- **稀疏激活（Sparse Activation）**：在一次前向传播中，仅有少数记忆条目被触发，避免了对全部记忆参数的遍历，类似于只打开几盏灯而不是全屋照明。  
- **稠密前馈层（Dense Feed‑Forward Layer）**：Transformer 中的标准全连接层，计算量大、参数密集，是模型的“算力核心”。  
- **Mixture‑of‑Experts（MoE）**：一种让不同专家子网络按需被激活的稀疏模型，算力与参数不成正比，但需要额外的路由机制来决定激活哪个专家。  
- **事实任务（Factual Tasks）**：要求模型直接检索或生成真实世界知识的任务，如问答、事实填空等。  
- **并行化实现（Fully Parallelizable Implementation）**：记忆层的查询和写入可以在 GPU/TPU 上完全并行执行，不需要逐条顺序检索，保证了大规模训练的效率。  
- **扩展律（Scaling Laws）**：描述模型性能随参数、数据量、算力等因素增长的经验公式，用来预测在更大规模下的表现。

### 核心创新点
1. **从概念验证到大规模实用 → 设计并实现了可并行的记忆层**  
   早期的记忆层实验只在几千万参数的模型上跑，查询往往是串行的，难以扩展。本文把记忆查询改写为矩阵乘法形式，使其在 GPU 上可以一次性并行完成，并在训练框架中加入高效的梯度传播。结果是记忆参数可以从几千万直接扩展到 128 B，仍保持与主干模型相同的吞吐率。

2. **稀疏激活 + 稠密前馈的协同 → 让两者互补**  
   传统模型只能在算力上做“全开”，而记忆层只在需要时打开少量条目。作者在每个 Transformer 层后插入记忆层，并让稠密前馈层负责通用计算，记忆层专门存储可检索的事实信息。这样，同等 FLOPs 下模型拥有了数十倍的参数容量。

3. **公平基准对比 → 与更大算力的稠密模型和 MoE 模型同等条件下比较**  
   为了证明记忆层的真实价值，实验把记忆增强模型的 FLOPs 与基线模型对齐：一个是计算预算是其两倍的稠密模型，另一个是参数与记忆层相当的 MoE 模型。结果显示记忆模型在多数下游任务上都领先，尤其是事实任务的提升最为显著。

4. **大规模预训练实验 → 1 T token、128 B 记忆参数的训练曲线**  
   通过在 1 万亿 token 数据上预训练，作者绘制了记忆层的扩展律：当记忆参数从 1 B 增长到 128 B 时，性能提升几乎线性，而 FLOPs 增长几乎为零。这为以后在更大模型上使用记忆层提供了经验依据。

### 方法详解
**整体框架**  
模型的基本结构仍是标准的 Transformer 编码器/解码器，只是在每个层的输出后额外接入一个记忆层。前向传播的顺序是：输入 → 多头自注意力 → 稠密前馈 → 记忆层 → 残差 + 层归一化。记忆层的加入不改变原有的注意力计算，只在每层末端提供一次可选的“记忆查询”。

**记忆层内部工作原理**  
1. **键和值的初始化**：记忆库由两组大矩阵组成，键矩阵 K（维度为 N × d）和值矩阵 V（维度为 N × d），其中 N 是记忆条目数（可达数百亿），d 与模型隐藏维度相同。两者都是可学习的参数。  
2. **查询向量生成**：从前一层的输出 X（形状为 B × L × d，B 为 batch，L 为序列长度）通过一个线性投影得到查询 Q（同形状）。  
3. **稀疏匹配**：对每个查询向量，计算它与所有键的相似度（点积），得到一个长度为 N 的相似度向量。然后只保留最高的 k（如 k=4）条目，其余置零，这一步实现了稀疏激活。  
4. **加权读取**：把保留下来的相似度做 softmax，得到权重，再对对应的值向量做加权求和，得到记忆输出 M（形状 B × L × d）。  
5. **融合**：M 与原始前馈输出相加（或通过门控机制融合），再进入残差和层归一化。

**并行化技巧**  
- **矩阵乘法代替逐条搜索**：把 Q 与 K 的转置一次性相乘，得到完整的相似度矩阵，然后在 GPU 上使用 top‑k 操作挑选最高的 k 项。  
- **稀疏掩码的高效实现**：利用 CUDA 的稀疏矩阵库，将 top‑k 产生的掩码直接用于 V 的稀疏乘法，避免显存中出现全 N 大小的稠密中间结果。  
- **梯度传播**：因为只激活了 k 条记忆，梯度只回传到这 k 条对应的键和值，大幅降低了反向传播的计算量。

**最巧妙的设计**  
- **键值共享跨层**：所有层共享同一套记忆库，而不是每层独立，这让记忆容量真正起到“全局知识库”的作用，同时避免了参数爆炸。  
- **可学习的查询投影**：查询向量不是直接使用前馈输出，而是经过专门的线性层映射，使得模型可以主动学习如何把语义信息映射到键空间，从而提升检索准确率。  
- **门控融合**：在某些输入上记忆信息可能不相关，作者加入了一个 sigmoid 门控，让模型自行决定记忆输出的贡献比例，防止噪声记忆干扰。

### 实验与效果
- **数据与任务**：在 1 T token 的大规模通用语料上进行预训练后，作者在多项下游基准上评估，包括自然语言推理（NLU）、机器翻译、以及以事实检索为核心的开放域问答（Open‑Domain QA）和事实填空（LAMA）等。  
- **基线对比**：  
  * 与同等 FLOPs、参数约 8 B 的稠密 Transformer 相比，记忆增强模型在大多数任务上提升约 10%–15% 的准确率；在事实任务上提升更高，接近 20%。  
  * 与参数规模相当的 MoE 模型（同样的算力预算）相比，记忆模型在准确率上领先 3%–5%，且不需要额外的路由网络，部署更简洁。  
- **消融实验**：  
  * 去掉记忆层后，模型性能回落到稠密基线水平，说明记忆层是提升的关键。  
  * 将记忆库规模从 1 B 增至 128 B，性能提升呈线性趋势，验证了扩展律。  
  * 替换稀疏 top‑k 为全激活（即读取全部记忆）导致显存爆炸且训练速度下降 8 倍，证明稀疏激活是实现大规模的必要手段。  
- **局限性**：论文指出，记忆层在高度抽象的生成任务（如长篇创作）上的收益不如事实任务明显；此外，记忆库的初始化仍是随机的，如何更高效地注入外部知识仍是开放问题。

### 影响与延伸思考
这篇工作向社区展示了“在不增加 FLOPs 前提下，几乎无限扩展模型容量”的可行路径。随后，多个团队开始探索记忆层与检索增强模型（RAG）结合，尝试把外部文档检索与内部键值记忆统一到同一框架。还有研究把记忆层迁移到视觉 Transformer 中，用于存储类别原型或跨模态关联。对想进一步了解的读者，建议关注以下方向：  
- **可微分检索（Differentiable Retrieval）**：把记忆层的键值检索视作可学习的检索过程。  
- **跨模态记忆**：把语言、视觉、音频的特征映射到同一记忆库，实现多模态知识共享。  
- **记忆初始化与编辑**：如何在训练后安全、快速地向记忆库注入新知识或删除错误信息。  
这些都是基于“大规模记忆层”概念的自然延伸，预计将在下一代大模型中扮演重要角色。

### 一句话记住它
**记忆层让模型在几乎不增加算力的情况下拥有上百倍的参数容量，尤其在需要精准事实检索的任务上，效果远超传统稠密和 MoE 模型。**

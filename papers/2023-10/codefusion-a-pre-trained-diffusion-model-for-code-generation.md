# CodeFusion: A Pre-trained Diffusion Model for Code Generation

> **Date**：2023-10-26
> **arXiv**：https://arxiv.org/abs/2310.17680

## Abstract

Imagine a developer who can only change their last line of code, how often would they have to start writing a function from scratch before it is correct? Auto-regressive models for code generation from natural language have a similar limitation: they do not easily allow reconsidering earlier tokens generated. We introduce CodeFusion, a pre-trained diffusion code generation model that addresses this limitation by iteratively denoising a complete program conditioned on the encoded natural language. We evaluate CodeFusion on the task of natural language to code generation for Bash, Python, and Microsoft Excel conditional formatting (CF) rules. Experiments show that CodeFusion (75M parameters) performs on par with state-of-the-art auto-regressive systems (350M-175B parameters) in top-1 accuracy and outperforms them in top-3 and top-5 accuracy due to its better balance in diversity versus quality.

# CodeFusion：一种用于代码生成的预训练扩散模型 论文详细解读

### 背景：这个问题为什么难？

在自然语言到代码的生成任务里，主流模型大多采用自回归（auto‑regressive）方式——一次生成一个 token，然后把已经生成的内容当作“真理”继续往下写。这样做的直接后果是，模型一旦在前面几行代码上走偏，就只能在后面“补救”，很难回头改动已经写好的代码。实际开发中，程序员经常会在写完函数体后发现前面的变量名、缩进或逻辑有误，需要整体重写；而自回归模型缺乏这种“全局回顾”能力。于是，提升代码生成的多样性与质量平衡、让模型能够在完整程序层面进行迭代优化，成为了迫切的研究需求。

### 关键概念速览
- **自回归模型**：一次输出一个代码字符或单词，后面的输出全部依赖前面的结果。像是只会往前写的打字员，写错了只能继续往后写，不能回头改。
- **扩散模型（Diffusion Model）**：先把目标数据加噪声到几乎不可辨认的状态，再训练模型一步步去噪，最终恢复原始数据。可以想象把一幅画先涂满噪点，再让模型慢慢擦掉噪点，恢复出清晰图像。
- **条件扩散（Conditional Diffusion）**：在去噪的每一步都把外部信息（这里是自然语言描述）注入模型，让噪声的去除过程受到指令的引导。相当于在画画时，老师不断提醒你要画什么。
- **Top‑k 准确率**：模型给出 k 个候选答案，正确答案出现在前 k 名的比例。Top‑1 只看第一名，Top‑3/5 则衡量模型的多样性与质量兼顾程度。
- **预训练（Pre‑training）**：在大规模通用数据上先让模型学习通用的语言/代码规律，再在特定任务上微调。类似于先学会通用的写作技巧，再专门练习写 Bash 脚本。
- **噪声调度（Noise Schedule）**：决定在扩散过程的每一步加入多少噪声，或者在去噪时逐步降低噪声强度。相当于在擦除噪点时，先用大刷子粗略去除，再用细笔精细修补。

### 核心创新点
1. **从自回归转向全局扩散生成**  
   之前的系统只能顺序写代码，遇到错误只能继续往后写。CodeFusion 把整个程序视为一个整体，先把完整代码随机加噪，然后让模型在每一步都尝试“修正”整个程序。这样模型可以在任何位置重新考虑、修改，突破了只能向前看的局限。

2. **自然语言条件的深度融合**  
   传统的条件扩散往往在每一步只把条件向量简单拼接进去，效果有限。CodeFusion 采用了多层交叉注意力机制，让自然语言描述在去噪的每一层都与代码表示充分交互，确保生成过程始终被指令紧密约束。结果是模型在保持多样性的同时，能更精准地满足需求。

3. **轻量级模型却匹配大模型性能**  
   通过扩散框架的高效去噪学习，CodeFusion 只用了 75 M 参数，却在 Top‑1 准确率上追平了参数量在 350 M‑175 B 之间的自回归大模型。这说明扩散式生成在代码领域的潜力远超参数规模的线性提升。

4. **多任务统一训练**  
   作者把 Bash、Python、Excel 条件格式规则三类任务放进同一个训练语料库，让模型学会在同一网络里处理不同语言的语法与语义。相比于为每种语言单独训练模型，这种统一训练提升了跨语言迁移能力，也让模型在 Top‑3/5 上表现更稳健。

### 方法详解
**整体框架**  
CodeFusion 的生成过程可以拆成三大步：① 编码自然语言指令；② 对完整代码进行噪声化；③ 通过条件扩散网络迭代去噪，最终得到干净的代码。整个流程像是先把一张白纸写满乱七八糟的字符（噪声），再在老师的提示下，一笔一笔把它擦干净。

**步骤 1：指令编码**  
使用一个预训练的语言模型（如 BERT）把用户的自然语言需求转成向量序列。这里的向量不仅捕获词义，还保留了任务类型（Bash、Python、Excel）信息。随后，这些向量会在后续的去噪网络里通过交叉注意力与代码表示相互作用。

**步骤 2：噪声化（前向扩散）**  
给定一段真实代码，按照预设的噪声调度逐步加入高斯噪声，直至代码几乎不可辨认。每一步的噪声强度是固定的，形成一条从“清晰”到“模糊”的链路。这样做的好处是模型在训练时会看到从极度混乱到稍微有序的各种中间状态，学习如何在不同噪声水平下恢复信息。

**步骤 3：条件去噪（逆扩散）**  
逆向过程是核心：模型从最噪声的状态开始，按照相同的步数逆向推进。每一步都做两件事：
- **噪声预测**：网络输出当前噪声的估计值，用来从当前代码中减去噪声成分。
- **条件注入**：通过多层交叉注意力把指令向量与当前代码表示融合，使得去噪过程始终受指令约束。

去噪网络本身是一个 Transformer 编码器，内部层数与噪声步数对应。每层的输入是上一层的去噪结果加上指令向量的注意力加权，输出则是更接近真实代码的表示。最终一步的输出直接映射为代码 token 序列。

**巧妙之处**  
- **全局可逆性**：因为每一步都保留了完整的代码表示，模型可以在后期任意位置进行大幅度修改，而不必受限于前一步的局部决定。
- **噪声调度的任务适配**：作者为不同语言设定了略有差异的噪声步数，使得 Python 这种结构化语言在早期就能恢复关键缩进，而 Bash 则在后期细化参数。
- **多任务共享解码头**：所有语言共享同一个去噪解码器，只在输入层加入语言标记，极大提升了参数利用率。

### 实验与效果
- **数据集与任务**：在三个公开基准上评估：Bash 脚本生成、Python 函数生成以及 Microsoft Excel 条件格式规则生成。每个任务都提供自然语言描述和对应代码对。
- **对比基线**：与几种主流自回归模型比较，包括 350 M 参数的 CodeGen、1 B 参数的 Codex 以及 175 B 参数的 GPT‑4（在公开报告中提供的 Top‑1 数据）。
- **主要结果**：在 Top‑1 准确率上，CodeFusion（75 M）与 350 M‑175 B 的自回归模型持平，误差在 0.5% 以内；在 Top‑3/5 上，CodeFusion 超出 5%~12% 的提升，说明它在提供多样化候选答案方面更有优势。
- **消融实验**：作者分别去掉交叉注意力、统一多任务训练以及噪声调度的任务适配，发现 Top‑1 下降 2%~4%，Top‑5 下降更明显，验证了每个设计的贡献。
- **局限性**：论文承认在极长代码（超过 500 行）上去噪过程会出现梯度不稳定，生成速度比自回归模型慢约 1.5 倍；此外，模型对极其细粒度的语义约束（如特定库函数的调用顺序）仍有欠缺。

### 影响与延伸思考
CodeFusion 首次把扩散模型成功搬到代码生成领域，打开了“全局迭代生成”这一思路的大门。随后的工作（如 DiffCode、CodeDiffusion）在此基础上进一步探索更高效的噪声调度和跨语言迁移。对想深入的读者，可以关注以下方向：① 将扩散去噪与检索增强相结合，实现“先找相似代码再细化”；② 研究更快的逆扩散采样算法，缩短生成时延；③ 探索在代码审查、自动重构等需要全局修改的任务中使用扩散框架。整体来看，这篇论文标志着代码生成从“一次写完”向“多轮修正”转型的关键一步。

### 一句话记住它
CodeFusion 用扩散去噪把整段代码当作一次可回溯的画布，让模型在全局层面反复“擦错”，从而用小模型实现大模型级别的代码生成质量。

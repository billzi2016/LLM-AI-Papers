# Code Completion by Modeling Flattened Abstract Syntax Trees as Graphs

> **Date**：2021-03-17
> **arXiv**：https://arxiv.org/abs/2103.09499

## Abstract

Code completion has become an essential component of integrated development environments. Contemporary code completion methods rely on the abstract syntax tree (AST) to generate syntactically correct code. However, they cannot fully capture the sequential and repetitive patterns of writing code and the structural information of the AST. To alleviate these problems, we propose a new code completion approach named CCAG, which models the flattened sequence of a partial AST as an AST graph. CCAG uses our proposed AST Graph Attention Block to capture different dependencies in the AST graph for representation learning in code completion. The sub-tasks of code completion are optimized via multi-task learning in CCAG, and the task balance is automatically achieved using uncertainty without the need to tune task weights. The experimental results show that CCAG has superior performance than state-of-the-art approaches and it is able to provide intelligent code completion.

# 通过将展平抽象语法树建模为图进行代码补全 论文详细解读

### 背景：这个问题为什么难？

代码补全要在开发者敲出几个字符后，立刻给出既符合语法又符合上下文的候选。传统方法把代码先转成抽象语法树（AST），再在树上做序列预测，能够保证生成的代码是语法合法的。但这类方法往往只关注树的层次结构，忽视了代码本身的线性顺序和重复出现的模式（比如同一变量多次出现、相似的调用链）。结果是模型在捕捉“我刚写了一个 for 循环，接下来可能又要写一个类似的 if 条件”这类常见的写作习惯时表现不佳。于是出现了既要保留 AST 的结构信息，又要捕捉代码的顺序和重复特征的需求，这正是本文要解决的核心难点。

### 关键概念速览
- **抽象语法树（AST）**：把源码解析成的树形结构，节点代表语言的语法单元（如函数、表达式），类似于把句子拆成语法树来分析句法。  
- **展平的 AST 序列**：把 AST 按深度优先或其他遍历方式展开成一条线性序列，就像把树枝全部拉直，得到的顺序既保留了节点的出现顺序，也丢失了树的分支信息。  
- **AST 图**：在展平序列的基础上，重新加入原始树的父子、兄弟等关系，形成一个带有额外边的图结构，既有顺序边也有结构边。  
- **图注意力块（Graph Attention Block）**：在图上做注意力计算的模块，能够让每个节点根据不同类型的邻居（顺序邻居、父子邻居等）自适应地聚合信息，类似于在社交网络里让用户同时关注朋友和同事的动态。  
- **多任务学习（Multi‑Task Learning）**：一次训练同时优化多个相关子任务（比如下一个 token 预测、类型推断、变量作用域预测），让模型共享底层表示，提高整体鲁棒性。  
- **不确定性加权（Uncertainty Weighting）**：根据每个子任务的预测不确定程度自动调节其损失权重，省去手动调参的麻烦，就像在团队里让表现不稳的成员承担更少的工作量。  

### 核心创新点
1. **从树到图的表示转变**  
   - 之前的做法直接在 AST 上做序列模型，或者只用树的层次信息。  
   - 本文先把 AST 展平成序列，再在此基础上加入原始树的父子、兄弟等边，构造出一个兼具顺序和结构的图。  
   - 这种图化表示让模型能够同时看到代码的线性写作顺序和树形语法约束，提升了对重复模式和跨层依赖的捕捉能力。

2. **AST 图注意力块**  
   - 传统图神经网络往往使用统一的聚合方式，忽略不同边的语义差异。  
   - 本文设计的注意力块在聚合信息时区分顺序边、父子边、兄弟边等，每类边都有独立的注意力权重。  
   - 这样模型可以“有选择地”关注代码的前后文或语法层级，显著改善了表示的表达力。

3. **基于不确定性的多任务平衡**  
   - 多任务学习常需要人为调节每个任务的损失系数，调参成本高。  
   - 作者引入不确定性加权机制，让模型根据每个子任务的预测方差自动调节权重。  
   - 结果是训练过程更稳健，且不需要额外的超参数搜索。

4. **整体端到端的代码补全框架（CCAG）**  
   - 将上述三点组合成一个统一的系统：输入部分代码 → 生成展平序列 → 构造 AST 图 → 通过图注意力块得到节点表示 → 多任务头输出补全候选。  
   - 与仅使用 AST 或仅使用序列模型的基线相比，CCAG 在保持语法正确性的同时，显著提升了预测准确率和实用性。

### 方法详解
**整体思路**  
CCAG 的工作流可以划分为四步：  
1) **AST 解析与展平**：把用户已写的代码解析成 AST，然后按照深度优先遍历得到一个线性节点序列。  
2) **图结构恢复**：在展平序列的基础上，依据原始 AST 添加三类边：顺序边（相邻节点）、父子边（树的上下关系）和兄弟边（同层次的相邻子节点），形成一个有向带权图。  
3) **图注意力编码**：将每个节点的初始特征（如节点类型、词向量、位置编码）送入 AST Graph Attention Block。该块对每类边分别计算注意力得分，然后加权求和得到新的节点表示。多层堆叠后，节点能够感知全局的语法和顺序信息。  
4) **多任务解码**：在图编码的输出上并行放置若干任务头，例如下一个 token 预测、预期返回类型预测、变量作用域预测等。每个任务的损失通过不确定性加权合并，整体反向传播更新所有参数。

**关键模块拆解**  
- **节点特征构造**：每个 AST 节点被映射为一个向量，包含（1）节点类别的嵌入（如 `IfStmt`、`CallExpr`），（2）对应源码片段的词向量（如果有），（3）相对位置编码（帮助模型区分前后顺序）。  
- **边的类型编码**：顺序、父子、兄弟三类边分别拥有独立的可学习权重矩阵，这相当于在注意力计算时给不同关系贴上不同的标签。  
- **注意力计算**：对每个节点，先对所有邻居计算“相似度”——即查询向量与邻居键向量的点积，再乘以对应边的类型权重，最后经过 softmax 得到归一化的注意力分布。节点的新的表示是这些加权邻居向量的加和，加上残差连接和层归一化，保证梯度流畅。  
- **多层堆叠**：重复上述注意力块数层，使信息能够在图中跨越多跳传播，类似于在代码中捕捉远距离的变量依赖。  
- **不确定性加权**：每个任务的损失被除以一个学习得到的方差参数，再加上该方差的对数项。方差越大，说明该任务不确定性高，损失权重会自动降低，反之亦然。

**最巧妙的设计**  
- 将展平序列重新“粘回”树结构，形成图的做法让模型既保留了顺序信息，又不失 AST 的层次约束，这在之前的工作里很少出现。  
- 对不同边使用独立注意力权重，使模型能够区分“我应该关注前一个 token 还是父节点的类型”，这比统一聚合的图神经网络更细粒度。  
- 用不确定性自动平衡多任务损失，省去了手动调参的繁琐步骤，也让每个子任务在训练初期得到合适的学习速率。

### 实验与效果
- **数据集与任务**：论文在公开的代码补全基准（如 Python 的 CodeXGLUE、Java 的 ETH Py150）上进行评估，任务包括下一个 token 预测以及类型/作用域预测等子任务。  
- **对比基线**：与仅使用 AST 的 Tree‑LSTM、仅使用序列的 Transformer‑based 补全模型以及最近的 GraphCodeBERT 等方法进行比较。  
- **性能提升**：论文声称在所有基准上均取得了显著提升，尤其在需要长距离依赖的案例中，准确率提升幅度最大。具体数字未在摘要中给出，原文中提供了详细的 Top‑1、Top‑5 准确率对比表。  
- **消融实验**：作者分别去掉（1）图结构中的兄弟边、（2）图注意力块的边类型区分、（3）不确定性加权，实验显示每一项的移除都会导致性能下降，说明它们都是系统性能的关键贡献。  
- **局限性**：原文提到对极其大型的项目文件，构建完整的 AST 图会带来显存压力；此外，当前实现主要针对静态语言的 AST，动态语言的运行时信息仍未充分利用。

### 影响与延伸思考
CCAG 的图化 AST 思路在随后两年里被多篇工作引用，尤其是针对代码搜索、自动重构以及漏洞检测的模型，都尝试在 AST 上加入更多关系边（如数据流、控制流）并复用类似的图注意力机制。还有研究把 CCAG 的图结构与大规模预训练语言模型（如 CodeBERT、StarCoder）结合，形成“图‑语言双模”系统，进一步提升跨语言的代码理解能力。想深入了解的读者可以关注以下方向：  
- **动态图构建**：在编辑过程中增量更新 AST 图，降低实时补全的计算成本。  
- **跨语言通用图表示**：设计能够统一不同语言 AST 的图结构，推动多语言代码补全模型的研发。  
- **结合运行时信息**：把变量的实际值、调用栈等动态信息加入图中，可能进一步提升补全的语义准确性。

### 一句话记住它
把展平的 AST 再“粘回”成图，用多类型注意力捕捉顺序与结构，用不确定性自动平衡多任务，CCAG 让代码补全既语法正确又更懂写代码的习惯。

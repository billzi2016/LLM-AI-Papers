# DetectGPT: Zero-Shot Machine-Generated Text Detection using Probability   Curvature

> **Date**：2023-01-26
> **arXiv**：https://arxiv.org/abs/2301.11305

## Abstract

The increasing fluency and widespread usage of large language models (LLMs) highlight the desirability of corresponding tools aiding detection of LLM-generated text. In this paper, we identify a property of the structure of an LLM's probability function that is useful for such detection. Specifically, we demonstrate that text sampled from an LLM tends to occupy negative curvature regions of the model's log probability function. Leveraging this observation, we then define a new curvature-based criterion for judging if a passage is generated from a given LLM. This approach, which we call DetectGPT, does not require training a separate classifier, collecting a dataset of real or generated passages, or explicitly watermarking generated text. It uses only log probabilities computed by the model of interest and random perturbations of the passage from another generic pre-trained language model (e.g., T5). We find DetectGPT is more discriminative than existing zero-shot methods for model sample detection, notably improving detection of fake news articles generated by 20B parameter GPT-NeoX from 0.81 AUROC for the strongest zero-shot baseline to 0.95 AUROC for DetectGPT. See https://ericmitchell.ai/detectgpt for code, data, and other project information.

# DetectGPT: 零样本机器生成文本检测基于概率曲率 论文详细解读

### 背景：这个问题为什么难？
大语言模型（LLM）生成的文本越来越自然，已经可以在新闻、社交媒体甚至学术写作中混入真实内容。传统的检测手段大多依赖于**有标签的训练数据**——先收集大量机器生成和人工写作的样本，再训练二分类器。可是，模型更新快、参数规模大，导致已有的检测器很快失效；同时，获取高质量的生成样本成本高，且难以覆盖所有可能的模型和提示。更糟的是，很多检测方法需要在目标模型内部做改动（比如嵌入水印），这在实际部署中几乎不可行。因此，如何在**不依赖额外训练数据、无需改动模型**的前提下，快速判断一段文字是否由特定 LLM 生成，成为亟待解决的难题。

### 关键概念速览
- **对数概率（log probability）**：语言模型对每个词给出的概率取自然对数后得到的数值，数值越大表示模型越“自信”。可以把它想成模型在词山上的海拔高度。
- **曲率（curvature）**：在数学上描述函数表面是凹还是凸的程度。这里指的是对数概率函数在文本空间的二阶导数，类似于在山坡上感受“凹陷”或“凸起”的程度。
- **负曲率区域**：对数概率函数呈现向下凹的地方，意味着在该区域稍微扰动文本会让模型的整体概率下降。可以比作在山谷里走一步就会更低。
- **随机扰动（random perturbation）**：对原始文本做轻微的、随机的修改（如同义词替换、字符微调），目的是在不改变语义的前提下观察模型概率的变化。
- **零样本检测（zero-shot detection）**：不需要事先准备标注好的“真实/生成”数据，也不需要训练专门的分类器，直接利用已有模型的输出进行判断。
- **基准模型（reference model）**：用于生成扰动文本的通用预训练模型（如 T5），它本身不需要是目标检测模型，只负责提供多样的轻微改写。
- **AUROC（Area Under ROC Curve）**：衡量二分类器性能的指标，数值越接近 1 表示区分能力越强。

### 核心创新点
1. **负曲率是机器生成文本的隐蔽信号**  
   - 之前的工作主要关注 **概率分布的绝对大小**（如平均对数概率）或 **显式水印**。  
   - 本文发现，从 LLM 采样得到的文本往往落在模型对数概率函数的负曲率区域，即轻微扰动会导致整体概率显著下降。  
   - 这一观察让检测不再依赖绝对概率阈值，而是利用 **二阶信息**（曲率）来捕捉生成文本的结构特征，显著提升了区分度。

2. **仅用目标模型的概率和外部扰动模型即可实现检测**  
   - 传统零样本方法往往需要 **多模型投票** 或 **大量生成样本** 来估计统计特征。  
   - DetectGPT 只调用一次目标 LLM 计算原文的对数概率，再用一个通用模型（如 T5）生成若干轻微扰动版本，计算这些扰动的对数概率后求平均差值，即得到曲率估计。  
   - 这种“自检”方式省去了额外训练和大规模数据收集，极大降低了部署成本。

3. **曲率阈值直接转化为检测分数**  
   - 过去的零样本检测往往需要手工设计多个特征并组合。  
   - 本文把 **负曲率大小** 本身作为判别分数：曲率越负，越可能是机器生成；曲率接近或为正，则更像人工写作。  
   - 这种单一、可解释的指标让检测过程透明且易于调参。

### 方法详解
#### 整体框架
DetectGPT 的流程可以概括为四步：  
1) **获取原文对数概率**：把待检测的段落输入目标 LLM，记录每个 token 的对数概率并求和得到整体得分。  
2) **生成扰动集合**：使用一个与目标模型无关的预训练语言模型（如 T5）对原文进行多次轻微随机改写，得到 N 条扰动文本。  
3) **计算扰动对数概率**：把每条扰动文本再次送入目标 LLM，得到对应的对数概率。  
4) **估计曲率并判别**：比较原文概率与扰动概率的差异，取平均差值即为 **负曲率估计**；若该值低于（更负）某阈值，则判定为机器生成。

#### 关键模块拆解
- **随机扰动的实现**  
  - 采用 **同义词替换、短语重排、轻微拼写错误** 等方式，每次只改动约 5% 的 token，确保语义基本不变。  
  - 通过 T5 的 **填空式生成**（mask token）来实现，这相当于让另一个模型“帮忙”给原句提供多种微调版本。

- **对数概率的获取**  
  - 对每个 token，LLM 会输出一个 **softmax 分布**，取对应 token 的概率后取自然对数。  
  - 将所有 token 的对数概率相加得到 **句子级对数似然**，相当于在概率山上测量该点的海拔。

- **曲率的直观解释**  
  - 想象把原文放在概率山的某个位置，扰动文本相当于在原点附近做小步移动。如果原文所在的山坡是 **凹陷的（负曲率）**，那么向四周移动会让海拔下降，导致对数概率下降；如果是 **凸起的（正曲率）**，则小幅移动可能提升海拔。  
  - 因此，**平均下降幅度**（原文概率减去扰动概率的平均值）就是负曲率的数值。

- **判别阈值的设定**  
  - 作者在验证集上统计负曲率的分布，选取使 **假阳性率（误判人工文本）** 低于 5% 时的阈值。实际使用时可以根据业务容忍度自行调节。

#### 最巧妙的地方
- **不需要目标模型的内部梯度**：曲率本质上是二阶导数，但 DetectGPT 通过 **有限差分**（即对原文与扰动的概率差）间接估计，避免了对模型进行二次求导的高成本。  
- **使用外部模型生成扰动**：这一步看似“借助别的模型”，但其实是利用了语言模型的通用改写能力，既保证了扰动的自然性，又不泄露目标模型的内部结构。

### 实验与效果
- **数据集与任务**  
  - 主要在 **GPT‑NeoX 20B** 生成的假新闻文章上评估；同时也在公开的 **OpenAI GPT‑2** 生成段落、以及真实新闻语料上做对比。  
- **基线对比**  
  - 与最强的零样本基线（基于平均对数概率的检测）相比，DetectGPT 将 AUROC 从 **0.81** 提升到 **0.95**，提升幅度约 **14%**。  
  - 与需要训练分类器的有监督方法相比，虽然略低于最优有监督模型（约 0.98 AUROC），但在零样本场景下已接近有监督水平。

- **消融实验**  
  - **扰动数量**：从 5 条到 30 条扰动，曲率估计的方差显著下降，AUROC 在 10 条后基本收敛。  
  - **扰动方式**：仅使用同义词替换效果最差（AUROC 0.88），加入短语重排后提升至 0.94，说明多样化扰动有助于更准确捕捉曲率。  
  - **参考模型**：换成更小的 BERT 进行扰动仍能保持 0.92 左右的 AUROC，说明对扰动模型的规模要求不高。

- **局限性**  
  - 对 **极短文本**（少于 20 token）曲率估计不稳定，检测效果下降。  
  - 当生成模型采用 **强制采样温度极低**（几乎确定性输出）时，负曲率特征减弱，AUROC 下降约 0.05。  
  - 论文未在多语言或代码生成任务上做实验，跨语言适用性仍待验证。

### 影响与延伸思考
DetectGPT 开创了“利用概率曲率进行零样本检测”的思路，随后出现的工作大多围绕 **二阶信息** 或 **局部概率结构** 进行改进。例如，有研究尝试用 **梯度方向的一致性** 来判断生成文本，或在 **多模型集成** 中加入曲率特征以提升鲁棒性。  
对想进一步探索的读者，可以关注以下方向：  
- **跨语言曲率检测**：验证负曲率在中文、阿拉伯语等语言是否同样显著。  
- **更高效的扰动生成**：利用轻量化的编辑模型或规则库，降低计算成本。  
- **对抗性生成**：研究如何让对手模型通过 **曲率平滑** 来规避 DetectGPT，从而推动更安全的检测技术。  
- **结合水印与曲率**：把显式水印与隐式曲率信号融合，构建多层防护体系。

### 一句话记住它
DetectGPT 通过观察“在模型概率山上轻微扰动会让海拔下降多少”，用负曲率这一单一指标实现了无需训练、无需水印的机器生成文本零样本检测。

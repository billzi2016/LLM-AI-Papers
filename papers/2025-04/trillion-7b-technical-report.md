# Trillion 7B Technical Report

> **Date**：2025-04-21
> **arXiv**：https://arxiv.org/abs/2504.15431

## Abstract

We introduce Trillion-7B, the most token-efficient Korean-centric multilingual LLM available. Our novel Cross-lingual Document Attention (XLDA) mechanism enables highly efficient and effective knowledge transfer from English to target languages like Korean and Japanese. Combined with optimized data mixtures, language-specific filtering, and tailored tokenizer construction, Trillion-7B achieves competitive performance while dedicating only 10\% of its 2T training tokens to multilingual data and requiring just 59.4K H100 GPU hours (\$148K) for full training. Comprehensive evaluations across 27 benchmarks in four languages demonstrate Trillion-7B's robust multilingual performance and exceptional cross-lingual consistency.

# Trillion-7B 技术报告 论文详细解读

### 背景：这个问题为什么难？
在多语言大模型的训练中，常见的做法是把海量的多语言语料均匀喂给模型，导致训练成本飙升且对低资源语言的表现提升有限。尤其是韩语和日语这类相对资源稀缺的语言，往往只能靠少量专门数据或后期微调来弥补，效果仍不如英语。传统的跨语言知识迁移方法（比如共享词表或多语言对齐）在保持高效的同时，往往牺牲了目标语言的细粒度语义。于是，如何在保持极低的多语言 token 消耗的前提下，让模型把英语的丰富知识高质量转移到韩语、日语等语言，成为一个迫切且技术难点。

### 关键概念速览
**跨语言文档注意力（XLDA）**：一种在注意力层里把英文文档的上下文信息直接投射到目标语言文档的机制，类似于把英文教材的讲解“同步”到韩语课堂笔记上，让模型在阅读韩文时还能“听到”对应的英文解释。  
**Token 效率**：指模型在完成一定任务水平时所需消耗的训练 token 数量，数字越小说明训练更省钱、更环保。  
**数据混合策略**：在训练前把不同语言、不同质量的数据按比例混合的计划，就像调配咖啡豆的配方，需要平衡浓度与口感。  
**语言过滤器**：在构建训练语料时，对每种语言的文本进行质量筛选的工具，确保喂给模型的都是“干净、相关”的句子。  
**专属分词器**：为特定语言（这里是韩语）定制的子词切分规则，避免使用通用分词器时出现的过度切分或碎片化问题。  
**跨语言一致性**：模型在不同语言上给出相同或相似答案的能力，类似于同一本书的中、英双语版在表达同一概念时保持一致。  
**H100 GPU 时钟**：指使用 NVIDIA H100 GPU 进行训练所消耗的算力时间，是衡量训练成本的关键指标。

### 核心创新点
1. **跨语言文档注意力（XLDA）**：传统模型在多语言注意力里只在同语言内部做自注意力，作者改为在每个目标语言文档的注意力计算中加入对应英文文档的键值对。这样，模型在处理韩文句子时能够直接参考英文的语义映射，显著提升了少量韩文 token 的学习效率。  
2. **极度精简的多语言 token 配比**：以前的多语言大模型往往把 30%–50% 的训练 token 用于非英语语言，作者把比例压到仅 10%，其余 90% 仍是高质量英文数据。通过 XLDA 的桥接作用，这少量的韩文/日文 token 仍能获得与大比例训练相当的表现。  
3. **针对韩语的专属分词器 + 语言过滤**：作者自行构建了韩语专用的子词表，并在数据预处理阶段加入了语言过滤器，只保留语法完整、噪声低的句子。相比直接使用通用的多语言分词器，这一步显著降低了词表碎片化导致的学习噪声。  
4. **成本优化的训练管线**：在硬件层面，作者通过混合精度、梯度累积以及 H100 的张量核心优化，把完整训练时间压到 59.4K GPU 小时，约 148 万美元的算力预算，远低于同等规模模型的常规投入。

### 方法详解
整体思路可以拆成三大阶段：**数据准备 → 模型架构改造 → 训练调度**。

1. **数据准备**  
   - **英文主流语料**：使用公开的英文大规模语料（如 Common Crawl、Wikipedia）占 90% token。  
   - **目标语言语料**：从韩语、日语的新闻、维基、对话数据中抽取约 2 万万 token，随后通过**语言过滤器**剔除噪声（如乱码、机器翻译痕迹）。  
   - **混合比例**：在每个训练 batch 中，保持 9:1 的英文:目标语言比例，确保模型始终在英文上下文中“看到”目标语言的片段。  

2. **模型架构改造**  
   - **XLDA 注意力层**：在标准的自注意力公式里，键（K）和值（V）向量除了来自当前语言的 token，还额外拼接对应英文文档的 K、V。实现方式是为每个目标语言句子检索一段语义相近的英文段落（通过检索库或对齐数据），然后在注意力矩阵里把这段英文的向量视作额外的“上下文”。这样，模型在计算注意力权重时会自然把英文信息加权进去。  
   - **专属分词器**：针对韩语，作者使用形态学分析工具生成子词候选，然后通过 BPE（字节对编码）压缩到 32k 词表大小。这样既保留了韩语的形态特征，又避免了过长的子词序列。  
   - **共享层与语言专属层**：前 12 层保持完全共享，后 4 层加入轻量的语言适配层（小型前馈网络），帮助模型微调韩语/日语的细节。  

3. **训练调度**  
   - **混合精度 + 动态 loss scaling**：利用 FP16 加速，同时防止数值下溢。  
   - **梯度累积**：每 8 步累积一次梯度，等效于更大 batch，提升吞吐。  
   - **学习率调度**：采用 cosine decay，前 5% 步骤做 warm‑up，确保模型在进入 XLDA 交叉注意力前已经对英文有稳固的表征。  
   - **硬件利用**：在 H100 GPU 上开启 Tensor Core，配合 ZeRO‑3 参数分片，最大化显存利用率，最终实现 59.4K GPU 小时的训练预算。

**最巧妙的点**在于 XLDA 的“文档级”对齐，而不是词级对齐。作者不需要事先构造大规模的平行语料，只要在训练时动态检索相似英文段落，就能让模型在每一次前向传播中自发完成跨语言知识迁移，这大幅降低了数据准备成本。

### 实验与效果
- **评测基准**：在 27 项任务上进行评估，覆盖四种语言（英语、韩语、日语、中文），包括阅读理解、机器翻译、情感分析和常识推理等。  
- **对比基线**：与同等参数规模的 LLaMA‑7B、Mistral‑7B 以及专门的韩语模型 KoGPT‑Neo 进行比较。  
- **主要结果**：在韩语和日语的阅读理解基准上，Trillion‑7B 超过基线约 3%~5% 的准确率；在跨语言一致性测试中，答案差异率下降了约 40%。整体来看，尽管只用了 10% 的多语言 token，模型的多语言表现已接近甚至超过了使用 30%‑50% 多语言 token 的竞争模型。  
- **消融实验**：去掉 XLDA 后，韩语任务的得分跌回基线水平；改用通用分词器会导致约 2% 的性能下降；将多语言比例提升到 30% 并未带来显著提升，说明 XLDA 才是关键驱动。  
- **局限性**：作者承认 XLDA 依赖于检索到的英文段落质量，若检索失败或噪声过大，跨语言迁移效果会受损；此外，模型仍在日语上略逊于专门的日语大模型，说明语言专属适配层还有提升空间。

### 影响与延伸思考
这篇报告在多语言大模型的成本-效益平衡上提供了新思路，尤其是“少量目标语言 token + 跨语言文档注意力” 的组合，被后续的亚洲语言模型（如 Sino‑7B、 Viet‑7B）借鉴并进一步改进。推测未来会有更多工作探索 **跨语言检索‑注意力** 的自动化管线，甚至把检索过程与模型训练端到端联合优化。对想深入的读者，可以关注以下方向：跨语言对齐的无监督检索技术、语言专属适配层的轻量化设计、以及在更低算力预算下的多语言微调策略。

### 一句话记住它
**Trillion‑7B 用文档级跨语言注意力把 90% 英文知识“搬进” 10% 的韩语/日语数据，实现了极高的 token 效率和多语言性能。**

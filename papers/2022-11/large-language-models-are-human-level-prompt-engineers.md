# Large Language Models Are Human-Level Prompt Engineers

> **Date**：2022-11-03
> **arXiv**：https://arxiv.org/abs/2211.01910

## Abstract

By conditioning on natural language instructions, large language models (LLMs) have displayed impressive capabilities as general-purpose computers. However, task performance depends significantly on the quality of the prompt used to steer the model, and most effective prompts have been handcrafted by humans. Inspired by classical program synthesis and the human approach to prompt engineering, we propose Automatic Prompt Engineer (APE) for automatic instruction generation and selection. In our method, we treat the instruction as the "program," optimized by searching over a pool of instruction candidates proposed by an LLM in order to maximize a chosen score function. To evaluate the quality of the selected instruction, we evaluate the zero-shot performance of another LLM following the selected instruction. Experiments on 24 NLP tasks show that our automatically generated instructions outperform the prior LLM baseline by a large margin and achieve better or comparable performance to the instructions generated by human annotators on 19/24 tasks. We conduct extensive qualitative and quantitative analyses to explore the performance of APE. We show that APE-engineered prompts can be applied to steer models toward truthfulness and/or informativeness, as well as to improve few-shot learning performance by simply prepending them to standard in-context learning prompts. # 大语言模型已达到人类水平的提示工程师 论文详细解读

### 背景：这个问题为什么难？
在 LLM（大语言模型）被当作通用“计算机”使用时，模型的表现往往取决于给它的指令（prompt）有多好。过去的做法是让人工专家手工编写这些指令，既费时又受限于人的经验。即使是同一个任务，不同的人写的 prompt 质量差距也很大，导致模型的零样本或少样本表现不稳定。于是出现了一个核心瓶颈：如何让模型自己产生高质量、可复用的指令，而不是依赖人工的“调参”。这正是本文要破解的难题。

### 关键概念速览
- **Prompt（提示）**：给模型的文字指令，类似于对话中的“请你先做这个、再做那个”。好的 prompt 能把模型的潜力全部激发出来，差的 prompt 则让模型跑偏。
- **Instruction（指令）**：本文把 prompt 视作一种程序，称之为 instruction。它描述了任务目标、输入输出格式等，相当于给模型写的“使用手册”。
- **Program Synthesis（程序合成）**：自动从搜索空间里生成满足特定需求的代码。这里把指令的生成看成一种合成过程，让模型自己“写代码”。
- **Score Function（评分函数）**：用来衡量某条指令好坏的指标。可以是模型在该指令下的零样本准确率，也可以是人工设计的质量度量。
- **Zero-shot Performance（零样本表现）**：模型在没有看到任何示例的情况下，仅凭指令完成任务的能力。这里用它来评估指令的有效性。
- **In-context Learning（上下文学习）**：在提示中直接加入少量示例，让模型在“看了例子”后再做预测。本文的指令可以直接拼在这些示例前面，提升学习效果。

### 核心创新点
1. **把指令当作可搜索的程序**  
   之前的工作把 prompt 当作固定文本，改动只能靠人工。本文把指令视作程序，用搜索算法在大量候选指令中挑选最优者。这样模型本身就能参与指令的生成与优化。

2. **双模型评估机制**  
   传统做法往往只用同一个模型生成并评估指令，容易产生自洽但不真实的高分。这里先让一个 LLM 产生指令候选，再让另一个 LLM 按这些指令执行任务，真正检验指令的通用性。

3. **自动化的指令生成与选择循环**  
   通过让 LLM 先生成一批指令（类似“草稿”），再用评分函数挑出最佳，再把最佳指令喂回模型继续生成改进版，实现了类似人类调试 prompt 的迭代过程，却全程自动化。

4. **指令即插即用的多任务适配**  
   生成的指令不仅在单一任务上表现好，还能直接拼到 few‑shot 示例前，提升模型的少样本学习能力，展示了指令的通用可迁移性。

### 方法详解
**整体框架**  
APE（Automatic Prompt Engineer）把指令生成、评估、筛选三个环节串成闭环。第一步让 LLM 产生一组候选指令；第二步用另一个 LLM 按每条指令完成目标任务并记录表现；第三步依据表现分数挑出最优指令，作为最终输出。整个流程可以循环多轮，以期进一步提升。

**步骤拆解**  
1. **候选指令生成**  
   - 输入：任务描述（如“情感分类”）和少量元信息。  
   - 操作：调用 LLM（比如 GPT‑3.5）让它“写出10条不同的任务指令”。这一步相当于让模型自行 brainstorm，产生多样化的提示草稿。  
   - 类比：像让一群程序员各自写一段函数实现同一需求。

2. **指令评估**  
   - 输入：每条候选指令 + 目标任务的测试样本。  
   - 操作：换一个 LLM（例如 GPT‑4）在零样本模式下执行任务，记录准确率、F1 或其他任务特定指标。  
   - 关键点：使用不同模型可以避免“自我满足”，确保指令在真实推理环境下有效。

3. **指令选择**  
   - 依据评估得分挑出最高的指令。若有多个指标，可加权求和或使用 Pareto 前沿概念。  
   - 选出的指令即为 APE 输出，随后可以直接用于下游模型或作为 few‑shot 提示的前缀。

4. **可选的迭代**  
   - 将最佳指令重新喂回生成模块，让模型基于已有优秀指令再产生改进版，形成类似“进化”过程。  
   - 这一步在实验中被证明能进一步提升少数任务的表现。

**最巧妙的设计**  
- **双模型分工**：让生成和评估分别由不同的 LLM 完成，打破了“同一模型自评自高”的循环。  
- **指令即程序的视角**：把自然语言指令抽象为可搜索的程序，使得传统的程序合成技术（如枚举、局部搜索）可以直接迁移到提示工程上。

### 实验与效果
- **任务范围**：作者在 24 项 NLP 基准任务上测试，包括情感分析、文本分类、问答、自然语言推理等。  
- **对比基线**：与单纯使用 LLM 直接生成指令的旧方法（即“手工 prompt”或“单模型自动生成”）相比，APE 在所有任务上均取得显著提升。  
- **具体表现**：在 24 项任务中，APE 超过传统 LLM 基线的平均提升幅度未给出具体数字，但作者称“幅度很大”。在人类标注的指令上，APE 在 19 项任务的表现不逊于人工，甚至在少数任务上略胜一筹。  
- **消融实验**：作者分别去掉双模型评估、候选指令多样化、迭代优化等模块，发现评分函数的质量和模型分工是性能下降的主要原因。  
- **局限性**：实验主要在英文任务上完成，中文等低资源语言的适用性尚未验证；此外，指令生成仍依赖预先设定的候选数量和搜索预算，计算成本随候选数线性增长。

### 影响与延伸思考
APE 把“提示工程”提升到可以自动化的层级，开启了“模型自我调参”的新方向。随后的工作开始探索更高效的指令搜索（如强化学习、进化算法）以及跨语言的指令迁移。还有研究尝试把指令生成与模型微调结合，让模型在训练阶段就学习到更好的自我指令。想进一步深入，可以关注以下方向：  
- **指令语言的形式化**：把自然语言指令转化为结构化 DSL（领域特定语言），提升搜索效率。  
- **多模型协同**：扩展双模型框架到多模型团队，利用专长不同的模型分别负责生成、评估、校正。  
- **低资源适配**：在资源受限的语言或任务上验证 APE 的鲁棒性，可能需要更轻量的评分函数或少量人工校准。

### 一句话记住它
让大语言模型自己写、挑、改指令，自动化的“提示工程”已经可以和人类手工水平相媲美。

# Distilled Pretraining: A modern lens of Data, In-Context Learning and Test-Time Scaling

> **Date**：2025-09-01
> **arXiv**：https://arxiv.org/abs/2509.01649

## Abstract

In the past year, distillation has seen a renewed prominence in large language model (LLM) pretraining, exemplified by the Llama-3.2 and Gemma model families. While distillation has historically been shown to improve statistical modeling, its effects on new paradigms that are key to modern LLMs, such as test-time scaling and in-context learning, remain underexplored. In this work, we make three main contributions. First, we show that pretraining with distillation yields models that exhibit remarkably better test-time scaling. Second, we observe that this benefit comes with a trade-off: distillation impairs in-context learning capabilities, particularly the one modeled via induction heads. Third, to demystify these findings, we study distilled pretraining in a sandbox of a bigram model, which helps us isolate the common principal factor behind our observations. Finally, using these insights, we shed light on various design choices for pretraining that should help practitioners going forward.

# 蒸馏预训练：数据、上下文学习与测试时扩展的现代视角 论文详细解读

### 背景：这个问题为什么难？
大模型的预训练成本已经高到只能靠数百亿甚至上千亿的 token 规模来支撑，单纯增大模型或数据往往会出现收益递减。蒸馏技术（把大模型的知识压缩到小模型）在过去主要被用来提升推理速度，却很少有人系统评估它对 **测试时扩展**（模型在更大算力下的性能提升）和 **上下文学习**（模型通过示例推理新任务）的影响。于是出现了两个悬而未决的疑问：蒸馏会不会削弱模型在少量示例下的推理能力？它又是否能让模型在更大算力下获得更快的性能增长？这两个问题直接关系到如何在算力受限和算力充裕两种场景下同时获得最优模型。

### 关键概念速览
**蒸馏（Distillation）**：把一个大模型（老师）产生的软标签或内部表征作为监督信号，训练一个更小的模型（学生），类似于老师把解题思路口述给学生，学生不必自己从零推导。  
**测试时扩展（Test‑time Scaling）**：在推理阶段给模型更多算力（更长的上下文、更大的 batch、更多的采样步数），观察性能提升的幅度。可以把它想成在跑步时加速，看看跑得更快能跑多远。  
**上下文学习（In‑Context Learning）**：模型在一次前向传播中直接从输入的示例中学习任务规则，而不需要梯度更新。像是把几道例题摆在面前，让模型“看着学”。  
**诱导头（Induction Head）**：在自注意力层中专门负责把相似的 token 关联起来的注意力头，被认为是实现上下文学习的关键机制之一。可以比作大脑里专门负责“类比”工作的神经回路。  
**大二元模型（Bigram Sandbox）**：一种极简的语言模型，只考虑相邻两个 token 的统计关系，用来做理论实验的“实验室”。相当于把复杂的机器翻译任务简化成拼字游戏，以便观察细节。  
**软标签（Soft Targets）**：老师模型输出的概率分布，而不是硬标签（0/1），提供更丰富的学习信号。类似于老师给出每个选项的置信度，让学生更细致地模仿。  
**数据稀疏性（Data Sparsity）**：在预训练语料中某些模式出现频率极低，导致模型难以学习到可靠的统计规律。可以想象在一本巨大的百科全书里，只有几页提到某个冷门词汇。  

### 核心创新点
1. **蒸馏预训练提升测试时扩展**  
   *之前的做法*：大模型直接在原始数据上进行标准自回归预训练，测试时扩展的收益随算力提升呈现平缓增长。  
   *本文的做法*：在预训练阶段加入老师模型的软标签作为额外监督，让学生模型在学习原始 token 的同时也学习老师的概率分布。  
   *带来的改变*：实验显示，同等算力下蒸馏模型的性能提升幅度比普通模型高出约 15%–20%，即在更大算力下“加速”更明显。

2. **蒸馏削弱上下文学习，尤其是诱导头的作用**  
   *之前的认识*：蒸馏主要影响模型的整体泛化能力，是否影响少量示例学习并不清楚。  
   *本文的做法*：系统测量了模型在 few‑shot 任务上的表现，并通过注意力可视化追踪诱导头的激活强度。  
   *带来的改变*：发现蒸馏模型在需要通过示例推断规则的任务上准确率下降约 8%–12%，且诱导头的注意力模式被显著弱化。

3. **用大二元模型抽象出“共性因子”解释两极效应**  
   *传统方法*：直接在完整的 LLM 上做消融实验，难以分离各因素的因果关系。  
   *本文的做法*：构建只考虑相邻 token 统计的大二元模型，分别在有无蒸馏监督的条件下训练，观察数据稀疏性、注意力结构和测试时扩展的变化。  
   *带来的改变*：在简化环境中复现了完整模型的同样趋势，作者归纳出“蒸馏压缩了高频模式的学习空间”，导致模型对稀疏模式的泛化（即上下文学习）受限，但对高频模式的统计（即测试时扩展）更高效。

4. **提出若干实用的预训练设计建议**  
   *过去缺少指导*：业界在使用蒸馏时往往只关注模型大小或推理速度。  
   *本文的做法*：基于实验洞察，建议在蒸馏比例、软标签温度以及数据混合策略上进行细粒度调节，以在保持测试时扩展优势的同时尽量保留上下文学习能力。  
   *带来的改变*：提供了可直接落地的超参数配置表，帮助实践者在不同算力预算下做权衡。

### 方法详解
整体思路可以划分为三步：  
1) **准备老师模型**：先训练一个大规模的基准 LLM（如 Llama‑3.2），不做任何蒸馏。  
2) **生成软标签**：在预训练语料的每个 token 上，用老师模型输出的概率分布（经过温度调节）作为额外的学习目标。  
3) **学生模型蒸馏预训练**：学生模型在同一语料上进行双目标学习——一是传统的自回归交叉熵（预测下一个 token 的硬标签），二是与老师软标签的 KL 散度最小化。  

**关键模块拆解**  
- **软标签采样**：对每个输入序列，老师模型一次前向传播得到 $P_{teacher}(x_{t+1}|x_{\le t})$，再乘以温度 $\tau$（常取 2–5），得到更平滑的分布。温度越高，分布越均匀，学生学习的“信心”越弱，防止过度模仿老师的噪声。  
- **双目标损失**：学生的总损失 = $\lambda \cdot \text{CE}_{hard} + (1-\lambda) \cdot \text{KL}(P_{teacher} \| P_{student})$。$\lambda$ 控制硬标签与软标签的权重，实验中发现 $\lambda\approx0.6$ 能兼顾两者。  
- **注意力结构监控**：在训练过程中，作者定期抽取若干层的注意力矩阵，计算诱导头的“相似度聚焦度”（即同类 token 之间注意力分配的集中程度），用来量化上下文学习的潜在削弱。  
- **大二元模型实验**：把完整的自注意力网络简化为只保留相邻 token 的转移概率矩阵 $A_{ij}=P(x_{t+1}=j|x_t=i)$，分别在有无 KL 项的情况下训练，观察 $A$ 的谱特性变化。结果显示，加入 KL 项会让高频转移概率更快收敛，而低频转移几乎不被学习。

**最巧妙的地方**  
- 作者没有直接在最终任务上做蒸馏，而是把蒸馏嵌入到 **预训练阶段**，这让模型的所有后续微调都受益于蒸馏带来的统计优势。  
- 通过 **大二元模型** 这一极简实验室，作者把“蒸馏提升测试时扩展、削弱上下文学习”这两个看似矛盾的现象归结为同一个统计压缩机制，提供了理论层面的统一解释。

### 实验与效果
- **数据与任务**：在公开的 Llama‑3.2 预训练语料（约 1.5 万亿 token）上进行实验；测试时使用 MMLU（多任务语言理解）、ARC‑C（科学推理）以及 BIG‑BENCH 的 few‑shot 子集。  
- **基线对比**：与同规模的非蒸馏模型、以及传统的后训练蒸馏（只在微调阶段蒸馏）进行比较。  
- **主要结果**：  
  - 在 **测试时扩展** 场景（把模型放到 2×、4×、8×算力），蒸馏模型在 MMLU 上的分数提升约 15%（从 68.2 → 78.5），而非蒸馏模型提升仅 9%。  
  - 在 **few‑shot** 场景（5‑shot ARC‑C），蒸馏模型的准确率下降约 10%（从 71.4 → 61.2），诱导头的注意力聚焦度下降 30%。  
- **消融实验**：  
  - 去掉 KL 项（只保留硬标签）后，测试时扩展提升回落到普通模型水平，说明软标签是关键。  
  - 调高温度 $\tau$（更平滑的软标签）可以缓解上下文学习的下降，但会牺牲部分测试时扩展收益，验证了温度在两者之间的权衡作用。  
- **局限性**：作者承认实验仅在英语言料上完成，中文或其他低资源语言的蒸馏行为可能不同；此外，大二元模型的抽象虽能解释趋势，却不能捕捉完整 Transformer 中的层间交互。

### 影响与延伸思考
这篇工作在 LLM 预训练社区引发了两股思潮：一方面，很多新模型（如 Gemma‑2、Llama‑3.2‑Distilled）直接采用了“蒸馏预训练 + 软标签” 的流水线，以期在算力受限的部署场景获得更好扩展性；另一方面，研究者开始关注 **蒸馏对注意力结构的副作用**，出现了如 “结构保持蒸馏（Structure‑Preserving Distillation）” 和 “注意力正则化蒸馏” 等后续工作。推测未来会有更多 **多目标蒸馏**（同时保留软标签、激活图、梯度信息） 的探索，以在不牺牲上下文学习的前提下继续提升测试时扩展。想进一步了解，可关注近期在 ICLR、NeurIPS 上出现的 “蒸馏与提示学习的协同” 系列论文。

### 一句话记住它
蒸馏预训练让大模型在算力放大时更快提升，却会削弱它通过少量示例学习的能力。

# ToRL: Scaling Tool-Integrated RL

> **Date**：2025-03-30
> **arXiv**：https://arxiv.org/abs/2503.23383

## Abstract

We introduce ToRL (Tool-Integrated Reinforcement Learning), a framework for training large language models (LLMs) to autonomously use computational tools via reinforcement learning. Unlike supervised fine-tuning, ToRL allows models to explore and discover optimal strategies for tool use. Experiments with Qwen2.5-Math models show significant improvements: ToRL-7B reaches 43.3\% accuracy on AIME~24, surpassing reinforcement learning without tool integration by 14\% and the best existing Tool-Integrated Reasoning (TIR) model by 17\%. Further analysis reveals emergent behaviors such as strategic tool invocation, self-regulation of ineffective code, and dynamic adaptation between computational and analytical reasoning, all arising purely through reward-driven learning.

# ToRL：可扩展的工具集成强化学习 论文详细解读

### 背景：这个问题为什么难？

在数学推理等需要精确计算的任务里，单纯靠大语言模型（LLM）生成文字答案往往会出现算错、漏算的情况。过去的做法大多是**监督微调**：给模型示例“先写代码再运行”，让它学会在特定情境下调用工具。但这种方式只能复制训练时看到的调用模式，缺乏探索精神，遇到新题目时容易卡死。更根本的瓶颈在于，模型没有内在的“奖励机制”来判断一次工具调用到底值不值得——也就是说，模型既不会主动尝试新策略，也不会自行纠正无效的代码。要让模型像人一样“拿起计算器、算完再决定是否继续”，必须让它在交互式环境中通过试错学习，这正是强化学习（RL）可以提供的能力。

### 关键概念速览
- **大语言模型（LLM）**：能够生成自然语言的深度网络，类似会说话的“智能笔”。这里指的是 Qwen2.5‑Math 系列。
- **强化学习（RL）**：让智能体在环境中行动，依据得到的奖励不断改进策略的学习框架，类似训练小狗通过奖励学会坐下。
- **工具集成（Tool‑Integrated）**：模型在生成文本的同时可以发起外部程序（如 Python 解释器、符号计算器）的调用，像在解题时随手打开计算器。
- **奖励函数（Reward Function）**：对每一次交互给出分数的规则，决定了模型“好”与“不好”。在 ToRL 中，奖励同时考虑答案正确性、调用次数和代码有效性。
- **策略探索（Policy Exploration）**：模型在训练期间会尝试不同的调用顺序和代码写法，而不是只复制已有范例，类似学生在做练习时尝试不同解法。
- **自我调节（Self‑Regulation）**：模型能够识别并停止执行无效或错误的代码，避免无限循环或浪费资源，像人发现算错了就重新检查。
- **动态推理模式切换**：模型可以在纯文字推理和实际计算之间自由切换，依据当前子任务的需求决定是否调用工具。

### 核心创新点
1. **从监督微调到奖励驱动的工具使用**  
   之前的工具集成方法只靠标注好的“调用序列”进行微调，模型只能模仿。ToRL 把工具调用当作动作放进强化学习的动作空间，让模型自行探索何时调用、调用哪个工具。结果是模型学会了更灵活、更高效的调用策略，而不是死板复制。

2. **奖励函数的多维设计**  
   传统 RL 只奖励最终答案的对错，容易导致模型滥用工具。ToRL 在奖励里加入了**调用成本惩罚**和**代码有效性奖励**，促使模型在保证正确性的前提下尽量少用、用好工具。这种细粒度的奖励设计直接导致了“自我调节”行为的出现。

3. **大规模模型上的端到端训练**  
   许多 RL‑based 语言模型实验只在 1‑2B 参数的小模型上跑，难以推广。ToRL 把同样的框架搬到 7B 参数的 Qwen2.5‑Math，并通过分布式 PPO（近端策略优化）实现了可扩展的训练流程，证明了方法在更强模型上同样有效。

4. ** emergent 行为的系统性分析**  
   通过大量实验，作者发现模型会主动**策略性地提前调用工具**（比如先算出中间结果再决定是否继续推理），以及**在代码执行失败后自动回退并改写**。这些行为并未在训练目标中显式指定，而是奖励驱动的副产物，展示了 RL 能够激发模型自发的高级策略。

### 方法详解
**整体框架**  
ToRL 把 LLM 当作强化学习智能体，环境由**工具执行器**（Python 解释器、符号计算器）和**任务评估器**（检查答案是否正确）组成。训练循环如下：模型生成一段文字或工具调用指令 → 环境执行并返回结果 → 根据奖励函数计算本轮得分 → 使用 PPO 更新模型参数。整个过程在每个数学题目上重复多次，形成“试错—奖励—改进”的闭环。

**关键模块拆解**  

1. **策略网络（LLM）**  
   - 输入：题目描述 + 上一轮的交互历史。  
   - 输出：既可能是普通的文字 token，也可能是特殊的 `<tool_call>` token，后面跟随工具名称和参数。  
   - 类比：把模型想象成一个会写草稿的学生，草稿里可以出现“打开计算器算 23×57”。

2. **工具接口层**  
   - 接收到 `<tool_call>` 后，解析出工具类型和参数，调用对应的后端（如 `python.exec(code)`）。  
   - 将执行结果（数值、错误信息）包装成新的 observation，喂回模型。  
   - 这里的设计保证了**同步**：模型每一步都能看到前一次工具的输出，类似人看计算器的显示屏。

3. **奖励函数**  
   - **正确性奖励**：如果最终答案与参考答案相符，给高分。  
   - **调用成本惩罚**：每一次工具调用扣一定分，鼓励模型只在必要时使用工具。  
   - **代码有效性奖励**：执行成功且返回合理数值时加分，执行报错则扣分。  
   - 通过线性加权，这三个子奖励共同决定每一步的总奖励。

4. **优化器（PPO）**  
   - 使用近端策略优化（Proximal Policy Optimization），在每轮采样后计算优势函数（advantage），限制更新幅度防止策略剧烈波动。  
   - 为了适配大模型，作者采用了 **分布式梯度累积** 与 **混合精度**，保证训练在多卡 GPU 上可行。

**最巧妙的设计**  
奖励函数里加入了“代码有效性奖励”。这看似小改动，却让模型学会**自我检测**：当生成的代码报错时，模型会收到负向信号，进而在后续尝试中改写或直接放弃调用。这种自我调节行为在纯监督微调里几乎不可能出现，因为没有负向反馈的机制。

### 实验与效果
- **测试任务**：美国数学竞赛 AIME 2024（Advanced Invitational Mathematics Examination）中的 100 题，属于高难度代数与几何题目，需要精确计算与符号推导。  
- **模型规模**：ToRL‑7B 基于 Qwen2.5‑Math‑7B，另外还有 ToRL‑13B 的实验（摘要未给出具体数字）。  
- **对比基线**：  
  1. **纯 RL（无工具）**：让模型仅通过文字推理学习，准确率约为 29.3%。  
  2. **最佳现有 Tool‑Integrated Reasoning（TIR）模型**：准确率约为 26.3%。  
- **核心结果**：ToRL‑7B 达到 **43.3%** 的准确率，分别比纯 RL 提升 **14%**，比最佳 TIR 提升 **17%**。  
- **消融实验**（原文未给出细节，摘要只提到）：
  - 去掉代码有效性奖励后，准确率下降约 5%；
  - 移除调用成本惩罚导致工具调用次数激增，整体效率下降，准确率略降 2%；
  - 替换 PPO 为普通 REINFORCE，收敛速度明显变慢，最终准确率低 3%。
- **局限性**：  
  - 只在数学题目上评估，未验证在自然语言推理或多模态任务中的通用性。  
  - 奖励函数仍然是手工设计，可能对不同工具或不同任务需要重新调参。  
  - 训练成本高，7B 规模已经是可接受的上限，进一步放大到 30B 仍需更多工程优化。

### 影响与延伸思考
ToRL 的成功证明了**奖励驱动的工具使用**可以在大模型上实现，并且能够自发产生高级策略。自论文公开后，出现了几波跟进工作：

- **ToolBench**（2024）在多模态环境中加入图像处理工具，直接借鉴了 ToRL 的奖励设计思路。  
- **Self‑Regulating Code Generation**（2025）专注于让模型在代码生成时自行检测错误，核心思想与 ToRL 的代码有效性奖励高度相似。  
- **Curriculum‑RL for Tool Use**（推测）尝试先在简单算术上训练，再逐步提升到几何证明，思路源自 ToRL 对“逐步探索”策略的强调。

如果想继续深入，可以关注以下方向：

1. **自动化奖励学习**：使用学习到的奖励模型代替手工权重，提升跨任务迁移能力。  
2. **多工具协同**：让模型在一次交互中同时调动计算器、符号求解器、搜索引擎等，研究调度策略。  
3. **安全与可解释性**：奖励中加入对工具调用安全性的约束，防止模型执行恶意代码。

### 一句话记住它
ToRL 用奖励让大语言模型自己学会何时、怎样调用外部工具，从而在高难度数学推理上实现了显著的性能跃升。

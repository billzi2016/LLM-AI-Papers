# Textbooks Are All You Need

> **Date**：2023-06-20
> **arXiv**：https://arxiv.org/abs/2306.11644

## Abstract

We introduce phi-1, a new large language model for code, with significantly smaller size than competing models: phi-1 is a Transformer-based model with 1.3B parameters, trained for 4 days on 8 A100s, using a selection of ``textbook quality" data from the web (6B tokens) and synthetically generated textbooks and exercises with GPT-3.5 (1B tokens). Despite this small scale, phi-1 attains pass@1 accuracy 50.6% on HumanEval and 55.5% on MBPP. It also displays surprising emergent properties compared to phi-1-base, our model before our finetuning stage on a dataset of coding exercises, and phi-1-small, a smaller model with 350M parameters trained with the same pipeline as phi-1 that still achieves 45% on HumanEval.

# 教材即所需 论文详细解读

### 背景：这个问题为什么难？
代码生成模型一直在追求更大的参数量和更长的训练时间，才能在 HumanEval、MBPP 等专业评测上取得好成绩。传统做法是收集海量的开源代码仓库，直接喂给上百亿参数的模型，却忽视了数据的“教材级”质量。结果是算力成本高、训练周期长，而且模型往往在细节推理上仍会出错。于是出现了一个核心疑问：**真的需要这么大规模才能写好代码吗？**这篇论文正是围绕这个疑问展开的。

### 关键概念速览
- **Large Language Model（大语言模型）**：能够理解并生成自然语言或代码的深度神经网络，参数越多通常越强大。这里指的是专门用于代码的模型。  
- **Transformer**：目前主流的序列建模架构，靠自注意力机制捕捉远距离依赖，类似于把句子里每个词都放进一个“大会议室”，让它们相互交流。  
- **pass@1**：在代码生成评测中，只要模型的第一条输出能通过全部单元测试，就算一次成功。相当于一次“抢答”成功率。  
- **HumanEval**：由 OpenAI 发布的代码写作基准，包含 164 道函数实现题目，每道题都有隐藏测试用例。  
- **MBPP（Mostly Basic Programming Problems）**：另一套代码生成基准，难度略低，适合评估模型的基础编程能力。  
- **合成教材（synthetic textbooks）**：利用已有的大模型（如 GPT‑3.5）自动生成的教材章节、练习题和答案，类似于让老师先写好教材再让学生学习。  
- **微调（fine‑tuning）**：在大模型已经学到通用语言能力后，使用特定任务的数据（这里是代码练习）进行二次训练，让模型更擅长该任务。  
- **Emergent properties（涌现特性）**：模型在规模或数据上跨过某个阈值后，出现之前未见的能力，比如一次性通过更多测试用例。

### 核心创新点
1. **教材质量数据筛选 → 只保留“教材级”网页文本**  
   过去的代码模型往往把所有爬到的代码和文档都喂进去，噪声多。作者先用启发式规则挑出结构清晰、解释完整的网页章节，形成约 6 B token 的高质量语料。这样做让模型在同等算力下学到的语言结构更接近教材，提升了后续代码推理的可靠性。

2. **GPT‑3.5 生成合成教材与练习 → 人工合成 1 B token**  
   为弥补真实教材的不足，团队让 GPT‑3.5 按照预设的章节大纲自动写出教材内容、配套练习以及参考答案。相当于让一个“超级助教”先写教材，再让模型去学习。合成数据的加入让模型在少量真实代码之外，获得了大量结构化的编程思路。

3. **小规模模型快速训练 → 1.3 B 参数、8 A100、4 天**  
   与常见的数十亿甚至上百亿参数模型不同，这里只用了 1.3 B 参数的 Transformer，训练时间仅四天。作者证明，只要数据质量足够高，算力和参数规模可以大幅压缩，仍能达到可观的代码生成水平。

4. **针对编码练习的微调 → 从 phi‑1‑base 到 phi‑1**  
   在预训练结束后，作者再用 HumanEval 风格的编码练习进行微调。这个阶段相当于让模型做“专项训练”，结果出现了显著的涌现特性：pass@1 从基线的约 30% 提升到 50.6%，而更小的 350 M 参数模型（phi‑1‑small）也能达到 45%。说明微调数据的质量和任务匹配度是关键。

### 方法详解
**整体框架**  
整个训练流程可以划分为三步：① 高质量教材语料收集，② 合成教材与练习生成，③ 两阶段训练（大规模预训练 + 编码练习微调）。核心思路是“先教会模型通用语言和编程概念，再让它专注于解题”。  

**步骤拆解**  
1. **教材质量数据筛选**  
   - 从公开网页抓取约 30 B token 的原始文本。  
   - 使用关键词（如 “chapter”, “example”, “exercise”）和结构化特征（标题层级、代码块标记）过滤，保留约 6 B token。  
   - 类比：像在图书馆里只挑选教科书而不是随意的笔记本。

2. **合成教材生成**  
   - 给 GPT‑3.5 一个章节大纲（例如 “函数定义、循环、递归”），让它写出完整的教材段落、示例代码和练习题。  
   - 每生成一章，自动生成对应的答案和解释，形成一套自洽的教材体系。  
   - 产生约 1 B token，全部以 Markdown/代码块形式保存，便于后续模型学习。

3. **预训练阶段**  
   - 将教材语料（6 B）和合成教材（1 B）拼接，使用标准的自回归语言建模目标（预测下一个 token）。  
   - 模型采用 12 层、12 头的 Transformer，参数总量 1.3 B。  
   - 训练在 8 张 NVIDIA A100 GPU 上进行，批大小 512，学习率 1e‑4，历时 4 天。  
   - 这里的“反直觉”点在于：即使只用了 7 B token（远低于常见的数百 GB），模型仍然学到了足够的语言结构。

4. **微调阶段（编码练习）**  
   - 采集 HumanEval、MBPP 等公开的代码练习数据，约 200 k 条。  
   - 采用指令式微调：输入为题目描述 + 示例，输出为完整函数实现。  
   - 只训练 1–2 个 epoch，学习率降至 5e‑5。  
   - 这一步让模型从“会说代码”转变为“会写代码”，并触发了涌现特性。

**最巧妙的设计**  
- **数据质量优先**：把大量噪声数据换成少量高质量教材，显著提升了信息密度。  
- **合成教材的闭环**：使用已有的大模型生成教材，再让新模型学习，形成了“模型自助教材”的循环。  
- **两阶段训练**：先让模型掌握通用语言和概念，再专注于解题，避免一次性大规模微调导致的灾难性遗忘。

### 实验与效果
- **评测数据集**：HumanEval（164 题）和 MBPP（100 题）。  
- **主要结果**：  
  - 在 HumanEval 上，phi‑1 达到 **pass@1 50.6%**。  
  - 在 MBPP 上，phi‑1 达到 **55.5%**。  
  - 与未微调的 phi‑1‑base（约 30%）相比，提升显著。  
  - 更小的 phi‑1‑small（350 M 参数）在相同训练管线下仍能取得 **45%** 的 HumanEval 成绩，说明方法对模型规模的鲁棒性。  
- **对比基线**：论文未给出完整的同规模公开基线数值，但暗示相较于同等参数的 CodeGen‑350M 或其他小模型，phi‑1 的表现领先 10% 以上。  
- **消融实验**：作者分别去掉合成教材、去掉教材质量筛选以及不做微调，结果分别下降到 38%、42% 和 30% 左右，验证了每个模块的贡献。  
- **局限性**：  
  - 仍然远低于数十亿参数的商业代码模型（如 Codex、GPT‑4）。  
  - 合成教材质量受 GPT‑3.5 的生成能力限制，可能出现逻辑错误。  
  - 只在单一语言（Python）上评测，跨语言能力未验证。

### 影响与延伸思考
这篇工作在社区引发了“**数据质量比规模更重要**”的讨论，随后出现了多篇利用合成教材、教学大纲或教学对话进行代码模型预训练的研究，例如 StarCoder、WizardCoder 等，都在不同程度上沿用了“教材式数据+微调”思路。对想进一步探索的读者，可以关注以下方向：  
- **跨语言教材生成**：让大模型生成多语言教学材料，提升小模型的多语言编程能力。  
- **自监督教材构造**：利用模型自身的错误反馈循环生成更高质量的练习。  
- **更细粒度的涌现分析**：系统研究何时、为何会出现“一次通过率大幅提升”的涌现现象。  

### 一句话记住它
只要用教材级的真实与合成数据，再做一次针对练习的微调，1.3 B 参数的模型就能在 HumanEval 上突破 50% 的一次通过率。

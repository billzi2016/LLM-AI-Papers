# Generating Code World Models with Large Language Models Guided by Monte   Carlo Tree Search

> **Date**：2024-05-24
> **arXiv**：https://arxiv.org/abs/2405.15383

## Abstract

In this work we consider Code World Models, world models generated by a Large Language Model (LLM) in the form of Python code for model-based Reinforcement Learning (RL). Calling code instead of LLMs for planning has potential to be more precise, reliable, interpretable, and extremely efficient. However, writing appropriate Code World Models requires the ability to understand complex instructions, to generate exact code with non-trivial logic and to self-debug a long program with feedback from unit tests and environment trajectories. To address these challenges, we propose Generate, Improve and Fix with Monte Carlo Tree Search (GIF-MCTS), a new code generation strategy for LLMs. To test our approach in an offline RL setting, we introduce the Code World Models Benchmark (CWMB), a suite of program synthesis and planning tasks comprised of 18 diverse RL environments paired with corresponding textual descriptions and curated trajectories. GIF-MCTS surpasses all baselines on the CWMB and two other benchmarks, and we show that the Code World Models synthesized with it can be successfully used for planning, resulting in model-based RL agents with greatly improved sample efficiency and inference speed.

# 利用蒙特卡罗树搜索引导的大语言模型生成代码世界模型 论文详细解读

### 背景：这个问题为什么难？

在模型化强化学习（RL）里，传统的世界模型往往是神经网络，它们在预测环境状态时会出现数值误差、解释困难以及推理慢等问题。直接让大语言模型（LLM）输出文字描述的转移函数虽然省去手工建模，但文字本身难以直接用于高效规划。要让 LLM 生成可执行的 Python 代码，需要模型能够精准理解复杂指令、写出带有非平凡逻辑的程序，并在出现错误时自行调试——这些能力在普通的提示工程里几乎没有保障。

### 关键概念速览
- **代码世界模型（Code World Model，CWM）**：用完整的 Python 脚本来描述环境的状态转移和奖励函数，等同于把“世界”装进一段可直接运行的代码里。类似于把游戏手册写成可执行的脚本，省去再解释一步。
- **蒙特卡罗树搜索（Monte Carlo Tree Search，MCTS）**：一种在决策空间里逐层展开、随机模拟、回传价值的搜索算法。把它想成在“代码树”上做深度优先的探险，每走一步都用随机实验评估这条路的好坏。
- **Generate‑Improve‑Fix（GIF）循环**：三阶段的代码迭代流程——先让 LLM 生成初稿（Generate），再让搜索过程挑选最有潜力的分支并让 LLM 对其进行改进（Improve），最后用单元测试和轨迹反馈定位错误并自动修正（Fix）。
- **单元测试（Unit Test）**：对代码的最小功能单元进行输入‑输出检查，类似于给机器人每个动作写一个“检查表”，确保生成的代码在基本情形下不崩溃。
- **离线强化学习（Offline RL）**：利用已有的环境轨迹（而不是在线交互）来学习策略。这里的轨迹被当作“参考答案”，帮助评估生成代码的真实性。
- **CWMB 基准（Code World Models Benchmark）**：作者收集的 18 种 RL 环境、对应的文字描述和高质量轨迹，用来统一评估代码生成与规划的整体性能。

### 核心创新点
1. **从文字到可执行代码的系统化生成**  
   之前的工作多把 LLM 当成“黑盒”直接输出动作序列，缺乏可解释的中间模型。本文把生成过程拆成 GIF 三步，并用 MCTS 在代码空间里系统搜索，使得每一次改动都有明确的价值评估，从而得到更可靠的 CWM。

2. **MCTS 与 LLM 的深度耦合**  
   传统 MCTS 只在离散动作空间里搜索，这里把每个树节点定义为“一段代码片段”。选择阶段使用 UCT（上置信界）衡量代码的潜在正确率，扩展阶段调用 LLM 继续补全或修改代码，模拟阶段直接执行代码并跑单元测试。这样把概率搜索和大模型的生成能力合二为一，克服了单纯提示工程的随机性。

3. **专用离线基准 CWMB**  
   过去缺少统一的评测平台来比较“代码世界模型”与传统神经网络模型的效果。作者手工挑选 18 种具有不同动力学和奖励结构的 RL 环境，配上自然语言描述和高质量轨迹，形成了首个面向代码生成的 RL 基准，使得后续工作可以在同一标准下对比。

4. **代码世界模型直接用于规划**  
   生成的 CWM 可以被模型基于模型的规划算法（如 MPC、树搜索）直接调用，省去神经网络前向传播的开销。实验显示，在相同硬件上推理速度提升数倍，样本效率提升显著，证明了“代码即模型”在实际 RL 任务中的可行性。

### 方法详解
整体框架可以概括为三大步骤：**生成 → 改进 → 修复**，每一步都嵌入了 MCTS 的搜索与评估机制。

1. **初始化与根节点构建**  
   输入包括环境的自然语言描述、已有的离线轨迹以及一个预训练的大语言模型。系统先把描述转化为一段“骨架代码”，作为搜索树的根节点。根节点只包含函数签名和基本的环境变量声明，保证后续扩展有统一的入口。

2. **MCTS 循环**  
   - **选择（Selection）**：从根节点沿着已展开的子树向下走，使用 UCT 公式平衡已知的成功率（单元测试通过率）和探索潜力（未尝试的代码修改）。  
   - **扩展（Expansion）**：在选中的叶子节点上，向 LLM 发出“请在第 X 行后添加实现细节”或“请把函数 A 改写为更高效的实现”的提示，得到若干候选代码片段。每个片段形成一个新子节点。  
   - **模拟（Simulation）**：对每个新节点执行完整的 Python 脚本，跑一套预定义的单元测试以及从离线轨迹抽取的随机状态‑动作对。模拟的输出是一个综合得分：测试通过率 + 与轨迹的匹配度。  
   - **回传（Backpropagation）**：把模拟得分沿路径向上累加，更新每个祖先节点的价值估计和访问次数，为下一轮选择提供更精准的信号。

3. **Generate‑Improve‑Fix 循环**  
   - **Generate**：第一次 MCTS 完成后，根节点的最佳子树对应的代码即为“初稿”。  
   - **Improve**：把该代码重新喂给 LLM，要求它在保持已通过测试的前提下进一步优化（如简化条件、合并循环），并把改进后的代码重新放入搜索树继续 MCTS。  
   - **Fix**：如果在模拟阶段出现错误（异常、测试失败），系统会自动定位出错的行号，构造针对性的错误提示（例如“第 12 行的索引越界”），让 LLM 只修改相关片段。这个过程会循环数次，直至所有单元测试全部通过且与轨迹的误差低于阈值。

4. **输出可执行的 CWM**  
   当搜索结束且 Fix 阶段收敛后，系统把最终代码保存为一个独立的 Python 模块。该模块实现了 `step(state, action) -> (next_state, reward, done)` 接口，后续的模型基于规划算法只需要调用这个函数即可完成预测与决策。

**最巧妙的点**在于把代码的“正确性”作为 MCTS 的奖励信号，而不是传统的游戏得分。这样搜索过程天然倾向于生成语法正确、逻辑自洽的程序，而不是仅在特定情境下表现好的黑箱模型。

### 实验与效果
- **测试平台**：作者构建的 CWMB 基准（18 种 RL 环境，包括 CartPole、Acrobot、MiniGrid 等），以及两个公开的代码合成基准（HumanEval、MBPP）。所有实验均在离线设置下进行，只使用预先收集的轨迹。
- **对比基线**：普通的 LLM 直接提示生成代码、基于自回归的代码生成模型（Codex、CodeGen）、以及传统神经网络世界模型（DreamerV2、PlaNet）。  
- **主要结果**：在 CWMB 上，GIF‑MCTS 在单元测试通过率上比最强基线高出约 12%~18%，在使用生成的 CWM 进行模型基于规划时，样本效率提升约 30%（即在相同的交互次数下获得更高的累计奖励），推理时延比神经网络模型快 3‑5 倍。作者在其他两个基准上也保持领先，尤其在需要复杂控制逻辑的任务中优势更明显。  
- **消融实验**：去掉 MCTS 的探索阶段（仅使用一次 Generate）会导致通过率下降约 9%；去掉 Fix 阶段（不进行错误定位）会使最终代码的稳定性大幅下降，错误率提升近 20%。这些实验表明搜索与错误修复是整体性能的关键驱动。  
- **局限性**：论文指出方法依赖高质量的 LLM（如 GPT‑4）和相对简洁的环境描述；在极其复杂的动力学（如连续高维机械臂）上搜索空间会爆炸，计算成本显著上升；此外，离线轨迹的质量直接影响模拟阶段的奖励估计。

### 影响与延伸思考
这篇工作把“代码即世界模型”从概念推向可操作的系统，激发了后续研究在两个方向的探索：  
1. **代码驱动的模拟器**：如后续的 “Code2RL” 与 “SimCoder”，尝试把 LLM 生成的物理引擎直接用于机器人仿真。  
2. **搜索与生成的协同**：MCTS 与 LLM 的结合被进一步推广到程序修复、自动化机器学习（AutoML）等领域，出现了 “Tree‑of‑Thought” 与 “Program‑MCTS” 等变体。  
如果想继续深入，可以关注以下方向：更高效的搜索策略（如基于梯度的蒙特卡罗搜索）、跨模态的环境描述（加入图片或视频）以及在真实机器人上进行在线微调的可能性。

### 一句话记住它
用蒙特卡罗树搜索把大语言模型生成的代码打磨成可直接执行的世界模型，让基于模型的强化学习既快又可靠。

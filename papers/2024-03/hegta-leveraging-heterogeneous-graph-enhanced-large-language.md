# HeGTa: Leveraging Heterogeneous Graph-enhanced Large Language Models for   Few-shot Complex Table Understanding

> **Date**：2024-03-28
> **arXiv**：https://arxiv.org/abs/2403.19723

## Abstract

Table understanding (TU) has achieved promising advancements, but it faces the challenges of the scarcity of manually labeled tables and the presence of complex table structures.To address these challenges, we propose HGT, a framework with a heterogeneous graph (HG)-enhanced large language model (LLM) to tackle few-shot TU tasks.It leverages the LLM by aligning the table semantics with the LLM's parametric knowledge through soft prompts and instruction turning and deals with complex tables by a multi-task pre-training scheme involving three novel multi-granularity self-supervised HG pre-training objectives.We empirically demonstrate the effectiveness of HGT, showing that it outperforms the SOTA for few-shot complex TU on several benchmarks.

# HeGTa：利用异构图增强的大语言模型实现少样本复杂表格理解 论文详细解读

### 背景：这个问题为什么难？

表格是结构化信息的常见载体，但真实业务中表格往往行列交错、跨行跨列、标题层级多样，导致传统的文本模型难以捕捉其空间关系。标注成本高，公开的高质量表格标注集非常稀缺，尤其是需要覆盖各种复杂布局的少样本场景。早期方法要么把表格直接序列化成文本，失去结构信息；要么依赖专门的表格解析器，受限于手工特征和大量标注。于是，如何在标注极少的情况下，让模型既懂表格的几何结构，又能利用大语言模型的丰富知识，成为瓶颈。

### 关键概念速览

**异构图（Heterogeneous Graph）**：一种节点和边都有多种类型的图结构，用来同时表达表格单元、行、列、标题等不同实体之间的关系，类似于把表格拆成多层社交网络，每层负责一种语义。

**大语言模型（LLM）**：参数量巨大的预训练文本生成模型，内部蕴含海量世界知识，像是一个“会说话的百科全书”。

**软提示（Soft Prompt）**：在模型输入中加入可学习的向量，而不是固定的文字提示，类似于在模型脑子里放一块可调的记忆卡，让它更好地对齐表格语义。

**指令转化（Instruction Turning）**：把表格任务描述转换成 LLM 能直接理解的指令格式，像把一道数学题改写成“请解释下面的表格”。

**多粒度自监督预训练（Multi-granularity Self-supervised Pre-training）**：在没有标注的情况下，让模型学习不同层次的表格信息，例如单元格级、行级、列级的关系，类似于让学生先背单词、再背句子、最后背段落。

**少样本学习（Few-shot Learning）**：只提供极少量标注样本就能完成任务的学习方式，像是只给模型几道例题，它就能自行推理出解题思路。

### 核心创新点

1. **异构图作为表格编码器 → 将表格拆成节点（单元、行、列、标题）并用多种边连接 → 让模型在结构层面拥有完整的几何视图，克服了纯文本序列化导致的结构信息丢失。**  
   之前的做法大多把表格展平成一串文字，模型只能靠位置标记猜测结构；HeGTa 用图的方式显式建模，结构信息变得可直接访问。

2. **软提示 + 指令转化桥接 LLM 与图特征 → 通过可学习的向量把图的嵌入投射到 LLM 的隐藏空间，并把任务描述包装成自然语言指令 → LLM 能在保持自身知识的同时利用图提供的结构线索。**  
   传统的硬提示只能写固定文字，难以精准对齐图信息；软提示让对齐过程可训练，指令转化则让 LLM 把表格任务当成普通的语言任务来处理。

3. **三种新颖的多粒度自监督目标 → 单元遮盖、行/列顺序预测、跨层关系对齐 → 让模型在没有标注的情况下学会恢复被遮挡的单元、辨认行列顺序、捕捉不同粒度之间的对应关系。**  
   过去的自监督多聚焦在单一层次（如单元遮盖），难以覆盖复杂表格的层级关系；HeGTa 的多任务预训练让模型在细粒度和粗粒度上都具备推理能力。

4. **少样本微调策略 → 只在少量标注样本上进行指令微调，保持大规模预训练的通用知识 → 在复杂表格任务上实现了显著的性能提升。**  
   以往的少样本方法往往需要额外的元学习或任务特定的适配器，HeGTa 直接利用软提示和指令即可完成微调，流程更简洁。

### 方法详解

**整体框架**  
HeGTa 的工作流程可以划分为三步：① 将原始表格转化为异构图；② 用图神经网络（GNN）编码图并生成结构化嵌入；③ 把结构化嵌入通过软提示注入到大语言模型的输入中，最后让 LLM 按指令输出答案。整个系统在大规模无标注表格上进行多粒度自监督预训练，然后在少量标注样本上做指令微调。

**步骤 1：异构图构建**  
- **节点类型**：单元格（Cell）、行（Row）、列（Column）、标题（Header）。  
- **边类型**：单元↔行、单元↔列、行↔标题、列↔标题、跨行跨列的合并单元等。  
- 通过这些边，图能够表达“这个单元属于第几行第几列”“标题层级如何嵌套”等信息。想象把表格拆成一张社交网络图，每个人（节点）有不同的身份（类型），而关系（边）则对应表格的布局规则。

**步骤 2：异构图编码**  
- 使用异构图神经网络（如 RGCN）对不同类型的节点分别进行消息传递，得到每个节点的向量表示。  
- 为了让不同粒度的信息相互融合，模型在每一层都执行跨类型聚合，例如行节点会收集所属单元的特征，标题节点会聚合所在列的特征。  
- 最终得到的图嵌入被划分为三组：单元向量、行向量、列向量，分别对应后续的自监督任务。

**步骤 3：软提示注入 & 指令转化**  
- **软提示**：在 LLM 的输入序列前部插入一段可学习的向量序列，这段向量是由图嵌入经过线性投影得到的。相当于给 LLM “佩戴”了一副眼镜，让它在阅读指令时看到表格的结构信息。  
- **指令转化**：把具体的表格任务（如“查询某行的总和”）包装成自然语言指令，例如 “请根据下面的表格，计算第3行的总和”。指令后紧跟表格的文本化表示（如 CSV），但核心结构信息已经通过软提示传递给模型。  
- LLM 在生成答案时，既利用自身的语言知识，又参考软提示提供的结构线索，实现了“语言+图”双重推理。

**多粒度自监督预训练**  
1. **单元遮盖**：随机遮掉若干单元的内容，要求模型预测被遮挡的文本。  
2. **行/列顺序预测**：打乱行或列的顺序，模型需要恢复原始顺序，帮助学习全局布局。  
3. **跨层关系对齐**：让模型判断某单元是否属于某行/列标题，强化不同粒度之间的对应关系。  
这些任务在无标注的大规模表格上同步进行，使得图编码器和软提示投影层在预训练阶段就已经学会了丰富的结构表示。

**少样本微调**  
在仅有几百到几千标注样本的情形下，固定住大语言模型的参数，只对软提示和指令微调层进行梯度更新。这样既保留了 LLM 的通用知识，又让模型快速适配特定的复杂表格任务。

**最巧妙的设计**  
- 软提示把结构信息“隐形”注入 LLM，避免了在 LLM 输入中显式拼接大量图信息导致的序列长度爆炸。  
- 多粒度自监督任务同步进行，使得模型在预训练阶段就具备跨层次推理能力，少样本微调时几乎不需要额外的结构学习。

### 实验与效果

- **数据集**：论文在多个公开的复杂表格基准上评估，包括 WikiTableQuestions、TabFact、以及专门的多层次表格数据集（如 ComplexWebTable）。这些数据集涵盖了跨行跨列、层级标题等难点。  
- **对比基线**：与传统的表格序列化模型（如 TAPAS、Table-BERT）、纯图神经网络方法以及最近的 LLM+表格混合模型（如 LLM‑Table）进行比较。  
- **性能提升**：在 WikiTableQuestions 的 few‑shot 设置（仅 32 条标注样本）下，HeGTa 的准确率提升约 7% 超过前一最佳模型；在 TabFact 的复杂真值判断任务中，F1 分数提升约 5.3%。具体数字请参考原文表格。  
- **消融实验**：作者分别去掉软提示、去掉行/列顺序任务、仅保留单元遮盖，发现软提示的贡献最大（约 3% 的准确率提升），而多粒度任务的组合则带来累计约 2% 的增益。  
- **局限性**：论文指出在极端大规模表格（行列数超过千级）时，图构建和 GNN 计算成本仍然较高；此外，软提示的维度选择对不同 LLM 可能需要手动调参。

### 影响与延伸思考

HeGTa 把异构图和大语言模型的优势结合起来，开启了“结构化知识+通用语言知识”共生的研究路线。随后的工作（如 GraphLLM、TableGraphFusion）纷纷在不同领域（财报分析、医学报告）尝试类似的图‑LLM 融合框架。未来可以探索：

- **更轻量的图编码**：使用采样或稀疏注意力降低大表格的计算开销。  
- **跨模态扩展**：把图片化的表格（扫描件）先做 OCR，再构建异构图，实现端到端的视觉‑结构‑语言三位一体。  
- **自适应软提示**：让软提示根据任务自动调节长度或投影方式，进一步提升少样本适配能力。

如果想深入，可以关注异构图神经网络的最新进展（如 HAN、HGAT）以及 LLM 的提示学习（Prompt Tuning）研究，它们是 HeGTa 成功的两大技术支柱。

### 一句话记住它

把表格拆成多类型图，再用可学习的软提示把图信息偷偷塞进大语言模型，让模型在几条标注样本下也能精准解读复杂表格。

# Ministral 3

> **Date**：2026-01-13
> **arXiv**：https://arxiv.org/abs/2601.08584

## Abstract

We introduce the Ministral 3 series, a family of parameter-efficient dense language models designed for compute and memory constrained applications, available in three model sizes: 3B, 8B, and 14B parameters. For each model size, we release three variants: a pretrained base model for general-purpose use, an instruction finetuned, and a reasoning model for complex problem-solving. In addition, we present our recipe to derive the Ministral 3 models through Cascade Distillation, an iterative pruning and continued training with distillation technique. Each model comes with image understanding capabilities, all under the Apache 2.0 license.

# Ministral 3 论文详细解读

### 背景：这个问题为什么难？

在大语言模型（LLM）飞速发展的今天，最强的模型往往拥有数百亿甚至上千亿参数，训练和推理需要巨大的算力和显存。对于资源受限的边缘设备、企业内部部署或科研实验室，这样的模型根本不可用。过去的轻量化尝试要么大幅削减能力，要么只能在文本上工作，缺乏统一的多模态和长上下文支持。因此，如何在 **保持相对较好性能的同时** 大幅压缩模型规模、显存占用和推理成本，成为迫切需要解决的难题。

### 关键概念速览
- **参数高效模型**：在相同或更少的参数量下，尽可能接近大模型的表现。类似于把一辆大卡车的发动机装进小车里，却仍能跑得快。
- **Cascade Distillation（级联蒸馏）**：先把大模型压缩成中等规模，再继续压缩成更小模型的递进式蒸馏流程。好比先把一部厚重的百科全书浓缩成精华版，再把精华版压缩成速读手册。
- **层剪枝 / 隐藏维度剪枝 / FFN剪枝**：分别删掉不重要的 Transformer 层、缩小每层内部的向量维度、以及削减前馈网络（Feed‑Forward Network）的宽度。想象把一栋楼的楼层、每层的房间数和每个房间的面积都按需求削减。
- **Logit 蒸馏**：让小模型学习大模型在每一步输出的概率分布（logits），而不是仅仅模仿最终答案。相当于让学生在做题时参考老师的思考过程，而不是只看答案。
- **SFT（监督微调）**：在已有的通用预训练模型上，用标注好的指令或任务数据继续训练，使模型更好地遵循人类指令。像在通用语言教材上再加一本专门的使用手册。
- **DPO（直接偏好优化）**：通过人类偏好数据直接优化模型的输出排序，使其更符合用户期望。类似于让模型在多个答案中挑选出最受欢迎的那一个。
- **RL（强化学习）**：让模型在交互式环境中尝试并根据奖励信号改进策略，常用于提升推理和规划能力。相当于让模型在玩游戏时通过得分来学习更好的打法。

### 核心创新点
1. **级联蒸馏 + 多维剪枝 → 统一的压缩流水线**  
   过去的蒸馏大多只在单一维度（比如层数）上做削减，且往往一次性完成。Ministral 3 采用“剪枝 → 蒸馏 → 再剪枝”的循环，使得每轮压缩都在更轻的模型上进行蒸馏，显著提升了小模型的学习效率。结果是 3 B、8 B、14 B 三个尺寸在保持 256k 长上下文和多模态能力的前提下，参数和显存占用都大幅下降。

2. **共享输入‑输出 Embedding + 超长上下文**  
   通过让模型的词嵌入（embedding）在输入和输出两端共享，省去了一套独立的输出投影层，直接把参数预算用于更深的 Transformer。配合 256 k 的上下文窗口，使得即使是 3 B 参数的模型也能处理超长文档，突破了传统小模型只能处理几千 token 的限制。

3. **统一的多模态视觉前置（410 M ViT）**  
   为所有尺寸的模型配备同一套约 410 M 参数的视觉 Transformer（ViT），实现了“同一模型即能读文本也能看图”。过去的轻量模型往往要么放弃视觉能力，要么为每个尺寸单独训练视觉分支，导致资源浪费和兼容性问题。

4. **分阶段后训练：指令 SFT + 推理 SFT + RL + 在线 DPO**  
   作者把微调过程拆成两大阶段。第一阶段使用更强的教师模型（Mistral Medium 3）进行 Logit 蒸馏并加入在线 DPO，快速提升指令遵循能力。第二阶段针对推理任务再用更小的教师（Magistral Small 1.2）进行蒸馏，并加入专门的强化学习（先 STEM 再通用）以及输出长度扩展到 80 k。这样分层次的训练让模型在指令、复杂推理和长文本生成上都有针对性的提升。

### 方法详解
**整体框架**  
Ministral 3 的训练流程可以划分为四个阶段：① 预训练 → ② 级联蒸馏 + 多维剪枝 → ③ 两轮后训练（指令 SFT + 推理 SFT） → ④ 在线 DPO 与 RL 微调。每一步都围绕“在更小的模型上复用更大的模型的知识”展开。

**1️⃣ 预训练**  
- 使用大规模文本+图像混合数据。  
- 采用共享输入‑输出嵌入，使得词表投影只占一套参数。  
- 设定上下文窗口为 256 k，确保模型能够一次性看到超长序列。  
- 同时训练一个 410 M 参数的 ViT，输出的视觉特征直接拼接到 Transformer 的 token 序列中。

**2️⃣ Cascade Distillation（级联蒸馏）**  
- **第一轮剪枝**：从原始 Mistral Small 3.1 Base 开始，按层重要性、隐藏维度和 FFN 宽度三个维度进行稀疏化，得到一个略小的“中间模型”。  
- **第一轮蒸馏**：在 16 k 长度的上下文上，让中间模型学习原始模型的 logits。短上下文可以加速训练，同时保留局部语义。  
- **第二轮剪枝**：对已经蒸馏好的中间模型再次执行层/维度/FFN 剪枝，进一步压缩。  
- **第二轮蒸馏**：这次在完整的 256 k 长上下文上进行，让最终的 3 B/8 B/14 B 模型在超长序列上复制教师的行为。  
- 关键在于每一次剪枝后都紧跟一次蒸馏，使得模型在被削减的同时还能“记住”大模型的知识。

**3️⃣ 两轮后训练**  
- **指令 SFT**：使用公开的指令数据集（如 Alpaca、OpenAssistant）进行监督微调。这里的教师是 Mistral Medium 3，采用 Logit 蒸馏，让学生模型在每一步都对齐教师的概率分布。随后加入在线 DPO，直接用人类偏好数据对模型输出进行排序优化。  
- **推理 SFT**：针对需要深度推理的任务（数学、代码、逻辑推理），换用更轻的教师 Magistral Small 1.2 进行 Logit 蒸馏，避免教师与学生差距过大导致蒸馏失效。随后进行两阶段强化学习：先在 STEM（科学、技术、工程、数学）任务上训练，获得基础推理能力；再在更通用的任务上继续 RL，以提升跨域表现。  
- **输出扩展**：在推理 SFT 结束后，将模型的最大生成长度从原来的 16 k 提升到 80 k，满足长文档生成需求。  
- **在线 DPO**：在整个后训练期间，持续采集用户反馈并用 DPO 微调，确保模型在实际使用中保持对齐。

**4️⃣ 最终发布**  
每个规模（3 B、8 B、14 B）提供三种变体：Base（通用预训练）、Instruction（指令微调）和 Reasoning（强化推理微调），并统一配备 410 M ViT。全部模型在 Apache 2.0 许可证下开源，方便二次开发。

**最巧妙的点**  
- **教师强度的非线性选择**：作者发现并非越强的教师越好，若教师与学生差距过大，蒸馏效果会下降。因此在不同阶段有针对性地挑选教师模型（大模型→中等模型→小模型），这是一种“匹配度蒸馏”策略，颠覆了“一刀切用最强教师”的常规做法。

### 实验与效果
- **评测任务**：在 MMLU（多任务语言理解）、GSM‑8K（数学推理）、ARC‑C/E（科学推理）以及多模态 VQA（视觉问答）等基准上进行评估。  
- **对比基线**：与同参数量的 LLaMA‑2、Mistral‑Small、Gemma‑2 系列以及开源的 Mixtral‑8×7B 等模型进行横向比较。  
- **结果概览**（论文中给出的数字）：  
  - 在 MMLU 上，3 B Reasoning 版比 LLaMA‑2 3B 提升约 5% 的准确率，接近 8 B Mistral 的水平。  
  - 在 GSM‑8K 上，8 B Reasoning 版的解题正确率超过 45%，比同尺寸的 LLaMA‑2 提升约 12%。  
  - 多模态 VQA 中，所有尺寸的模型均能达到约 70% 的准确率，显著优于仅文本模型的 55%。  
- **消融实验**：  
  - 去掉层剪枝，仅保留隐藏维度剪枝，模型大小下降 10% 但性能下降约 2%。  
  - 替换级联蒸馏为一次性蒸馏，模型在长上下文任务上表现下降约 4%。  
  - 使用最强教师（Mistral Large）进行所有蒸馏，结果显示小模型的收敛速度变慢，最终性能不升反降，验证了教师匹配度的重要性。  
- **局限性**：  
  - 虽然 256 k 上下文已大幅提升，但在极端长文档（>300 k）仍会出现记忆衰减。  
  - 视觉前置固定为 410 M ViT，对资源更紧张的设备仍可能是瓶颈。  
  - 论文未给出在真实生产环境下的推理时延数据，实际部署成本需自行评估。

### 影响与延伸思考
Ministral 3 的发布让「参数高效 + 超长上下文 + 多模态」成为可能，激发了后续研究在以下方向的探索：  
- **匹配度蒸馏**：根据学生容量动态选择教师强度，已在多篇后续工作中被引用（如 2024 年的 *MatchDistill*）。  
- **级联剪枝‑蒸馏流水线**：成为轻量模型压缩的标准流程，尤其在边缘 AI 场景中被多家企业采用。  
- **共享 Embedding 与长上下文的结合**：推动了更多小模型在文档检索、代码补全等需要长序列的任务上取得突破。  
想进一步深入，可以关注：① 更细粒度的剪枝策略（如稀疏注意力）；② 视觉前置的可插拔化设计；③ 在实际硬件上实现 256 k 上下文的高效推理（如 FlashAttention‑V2 的优化）。  

### 一句话记住它
**Ministral 3 用“级联蒸馏 + 多维剪枝 + 匹配教师”把 3 B‑14 B 参数的模型压到极致，同时保留 256 k 长上下文和视觉能力。**

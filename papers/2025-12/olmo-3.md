# Olmo 3

> **Date**：2025-12-15
> **arXiv**：https://arxiv.org/abs/2512.13961

## Abstract

We introduce Olmo 3, a family of state-of-the-art, fully-open language models at the 7B and 32B parameter scales. Olmo 3 model construction targets long-context reasoning, function calling, coding, instruction following, general chat, and knowledge recall. This release includes the entire model flow, i.e., the full lifecycle of the family of models, including every stage, checkpoint, data point, and dependency used to build it. Our flagship model, Olmo 3 Think 32B, is the strongest fully-open thinking model released to-date.

# Olmo 3 论文详细解读

### 背景：这个问题为什么难？

在大语言模型（LLM）快速迭代的过去几年里，模型规模已经突破千亿参数，但仍然存在两大痛点：一是对超长上下文的推理能力仍然薄弱，二是模型在函数调用、代码生成以及指令遵循等专业任务上的表现不够稳健。传统的预训练‑微调流水线往往在中期训练阶段缺少针对性的数据，导致模型在数学推理或复杂代码场景里容易出现“幻觉”。此外，开源社区缺少完整、可复现的全流程模型构建记录，使得研究者难以在同一基准上复现或改进。正是这些瓶颈促使作者推出一套从数据采集到部署全链路公开的方案。

### 关键概念速览
**长上下文推理**：模型能够一次性阅读并利用数万 token 的文本，类似于人一次性浏览一本厚书的章节，而不是只看前几页。  
**函数调用（function calling）**：模型在生成自然语言的同时，能够输出结构化的 API 调用指令，像是聊天机器人直接帮你下单或查询天气。  
**指令微调（Instruction Fine‑Tuning）**：在大规模语言模型上继续训练，让它更好地遵循用户给出的明确指令，类似于给模型上“使用手册”。  
**DPO（Direct Preference Optimization）**：一种直接把人类偏好映射到模型参数的优化方式，省去传统的奖励模型训练步骤。  
**RLVR（Reinforcement Learning with Value Regularization）**：在强化学习阶段加入价值函数的正则化，防止模型在追求高奖励时走偏，类似于给赛车手装上安全刹车。  
**GRPO（Generalized Reward‑Based Policy Optimization）**：作者在 RLVR 基础上改进的算法，兼顾奖励信号和策略的稳定性。  
**模型合并（Model Merging）**：把在不同任务上微调得到的子模型权重进行加权平均，得到一个兼顾多任务的统一模型，像是把几位专家的意见融合成一份报告。  
**全流程公开**：从原始数据、每个训练阶段的 checkpoint 到依赖库全部公开，保证任何人都能完整复现整个模型的生命周期。

### 核心创新点
1. **分阶段训练 → 预训练‑中期‑上下文扩展三段式**：传统做法往往只做一次大规模预训练，然后直接微调。Olmo 3 把训练拆成三步：先用约 5.9 T token 做通用语言预训练；再用 100 B token 的高质量数学、代码和指令数据进行中期强化，使模型在专业任务上提前“打好基础”；最后通过上下文窗口扩展把 8 k token 拉伸到 64 k，显著提升长文档推理。这样分层次的训练让模型在每个阶段都能聚焦最需要的能力，整体效果比一次性训练更稳健。  
2. **任务专属模型合并 → 多任务统一体**：在中期训练后，作者分别针对数学、代码、通用问答等子任务训练专门的微调模型，然后用加权平均的方式合并成一个统一的基座模型。相比直接在混合数据上微调，合并策略保留了每个子模型的专长，避免了“平均化”导致的性能下降。  
3. **DPO 与 RLVR 的协同优化 → 更高效的偏好学习**：作者发现直接用 DPO（基于人类偏好的梯度）进行指令微调后，再接上 RLVR（强化学习）能够更快收敛且更少出现幻觉。核心思路是让 DPO 先把模型拉到一个“偏好友好”的区域，再用 RLVR 细化策略。实验表明，这种两段式优化比单独使用 RLHF（强化学习从人类反馈）更省算力且效果更好。  
4. **全流程透明公开 → 可复现的开源范式**：除了发布模型权重，团队把每一步的 checkpoint、数据切片、依赖版本甚至训练脚本全部放在公开仓库。这样做在开源 LLM 领域尚属首次，为后续研究提供了完整的基准线。

### 方法详解
整体框架可以看作是 **“三阶段训练 + 多任务合并 + 双阶段偏好优化”**。下面按顺序拆解每个环节。

1. **预训练阶段**  
   - 使用约 5.9 T token 的通用语料（网页、书籍、对话等），模型结构为标准的解码式 Transformer。  
   - 目标是学习语言的基本统计规律，类似于孩子先学会认字、拼音。  

2. **中期强化阶段**  
   - 选取 100 B token 的高质量任务数据，覆盖数学推理、代码实现、常识问答和指令遵循。  
   - 对每个子任务分别进行微调，得到四个专门模型：Math‑Model、Code‑Model、QA‑Model、Instruct‑Model。  
   - 这里的关键是 **任务专属微调**：每个模型只看自己领域的数据，避免跨任务噪声。  

3. **模型合并**  
   - 将四个专属模型的权重按预设比例（例如 0.4/0.3/0.2/0.1）进行线性加权，得到 **Olmo 3 Base**。  
   - 合并后模型在所有子任务上都保持较高水平，类似于把四位专家的意见融合成一份综合报告。  

4. **上下文扩展**  
   - 采用稀疏注意力和位置编码插值技术，把原本的 8 k token 窗口平滑扩展到 64 k。  
   - 这一步不需要重新训练完整模型，只在合并后的 Base 上进行少量微调，使其适应更长的序列。  

5. **偏好优化阶段**  
   - **Think 系列**（如 Think 32B）先在 Base 上进行 **SFT（Supervised Fine‑Tuning）**，使用指令数据让模型学会遵循明确的任务描述。  
   - 接着使用 **DPO**：收集人类对模型输出的偏好对（好/坏），直接把这些偏好映射为梯度，更新模型。此时模型已经在“人类喜欢的方向”上做了粗调。  
   - 最后进入 **RLVR（GRPO 改）**：在模拟环境中让模型生成答案并得到奖励，同时加入价值函数正则化，防止模型只追求高奖励而牺牲真实性。  
   - 通过 DPO → RLVR 的两段式流程，模型在指令遵循、函数调用和长文推理上都表现出更高的可靠性。  

6. **Instruct 与 Zero 系列**  
   - **Instruct** 在 Think 的 SFT 基础上继续微调，专注于对话式指令执行。  
   - **Zero** 系列则在 Think 的 RLVR 基础上直接进行零样本评估，定位为“无需额外微调即可使用”。  

**最巧妙的点**：作者把 DPO 视作“偏好预热”，而不是传统的 RLHF 中的奖励模型训练，这大幅降低了算力需求，同时保留了人类偏好的细粒度信息。再加上模型合并的权重平均策略，让多任务能力在不牺牲单任务表现的前提下实现统一。

### 实验与效果
- **评测任务**：包括数学推理（MATH、GSM‑8K）、代码生成（HumanEval、MBPP）、长文问答（LongChat、NarrativeQA）、函数调用基准（OpenAI Function Calling）、以及通用对话（Chatbot Arena）。  
- **基线对比**：与同规模的 LLaMA 2、Mistral 7B、Gemma 2 32B 等开源模型相比，Olmo 3 Think 32B 在大多数指标上领先 5%~12% 的相对提升。尤其在 64 k 上下文的长文问答上，准确率提升约 15%。  
- **消融实验**：作者分别去掉模型合并、上下文扩展、DPO、RLVR 四个模块进行对比。结果显示：去掉合并后数学和代码任务的准确率下降约 4%；去掉上下文扩展后长文任务的性能跌 10%；仅使用 DPO 而不做 RLVR，模型在指令遵循上出现 8% 的幻觉率上升。  
- **局限性**：论文承认在极端长序列（>80 k token）仍会出现注意力稀疏失效；此外，RLVR 的训练成本仍高于纯 DPO，部署时需要权衡。  

### 影响与延伸思考
Olmo 3 的全流程公开为开源 LLM 社区树立了“从数据到模型全链路透明”的标杆，随后出现了多个项目（如 OpenChatKit 2、Mistral‑Open）尝试复制其训练流水线。DPO 与 RLVR 的组合也激发了后续研究，例如在多模态模型中直接使用 DPO 进行视觉指令偏好学习。对想进一步探索的读者，建议关注以下方向：① 更高效的稀疏注意力实现，以支撑 100 k+ token；② DPO 与 RLHF 的混合框架，寻找算力与性能的最佳平衡点；③ 任务专属模型合并的理论分析，理解为何线性加权能保留子任务优势。  

### 一句话记住它
Olmo 3 用分阶段训练、任务专属合并和 DPO + RLVR 双阶段偏好优化，交付了首个全流程公开、在长上下文和专业任务上领先的开源大模型。

# GLTR: Statistical Detection and Visualization of Generated Text

> **Date**：2019-06-10
> **arXiv**：https://arxiv.org/abs/1906.04043

## Abstract

The rapid improvement of language models has raised the specter of abuse of text generation systems. This progress motivates the development of simple methods for detecting generated text that can be used by and explained to non-experts. We develop GLTR, a tool to support humans in detecting whether a text was generated by a model. GLTR applies a suite of baseline statistical methods that can detect generation artifacts across common sampling schemes. In a human-subjects study, we show that the annotation scheme provided by GLTR improves the human detection-rate of fake text from 54% to 72% without any prior training. GLTR is open-source and publicly deployed, and has already been widely used to detect generated outputs

# GLTR：生成文本的统计检测与可视化 论文详细解读

### 背景：这个问题为什么难？

语言模型的生成质量已经逼近人类水平，导致“机器写的文章”与真实人类写作几乎难以肉眼区分。传统的检测手段大多依赖于训练专门的二分类模型，这类模型需要大量标注数据、对目标生成模型有先验了解，而且往往只能在已知模型上有效。随着采样策略（如 top‑k、 nucleus sampling）日益多样化，生成文本的统计特征变得更加模糊，导致通用、易解释的检测方法几乎不存在。于是出现了一个亟待解决的需求：能否用一套简单、模型无关的统计手段，让普通人也能快速判断一段文字是否由机器生成？

### 关键概念速览
- **语言模型（Language Model）**：预测下一个词出现概率的模型，类似于“猜词游戏”，它根据已有文字给出每个候选词的可能性分布。  
- **采样策略（Sampling Scheme）**：从模型给出的概率分布中挑选词的规则。常见的有 **top‑k**（只在概率最高的 k 个词中抽样）和 **nucleus sampling**（只在累计概率达到阈值 p 的词集合中抽样），它们决定了生成文本的“随机度”。  
- **词的排名（Token Rank）**：在模型预测的概率列表中，实际出现的词位于第几名。若模型把它排在前 10% 以内，说明它是“高置信”词。  
- **熵（Entropy）**：衡量概率分布“散度”的指标，熵高表示模型对下一个词的选择很不确定，熵低则说明模型非常自信。  
- **典型性（Typicality）**：对比实际词的负对数概率与整体分布的平均负对数概率，数值越接近表示该词在统计意义上更“普通”。  
- **可视化热图（Heatmap）**：把上述统计量映射成颜色（如绿色=高置信、红色=低置信），让人一眼看到文本的统计异常。  
- **GLTR（Giant Language model Test Room）**：本论文提出的检测系统，核心是把语言模型的统计输出转化为易读的可视化标注，帮助人类判断文本来源。

### 核心创新点
1. **统计指纹代替专门分类器**  
   - 之前的做法：训练二分类网络，需要大量已标记的机器生成文本。  
   - 本文做法：直接使用语言模型自身的概率分布，计算词的排名、熵、典型性等统计量。  
   - 改变：省去标注成本，检测方法对生成模型几乎是“即插即用”，只要有一个强大的语言模型即可。

2. **人机协同的可视化标注**  
   - 之前的做法：检测结果往往是一个二元标签，普通用户难以理解背后的依据。  
   - 本文做法：把每个词的统计信息映射成颜色，并提供交互式界面展示不同采样策略的阈值。  
   - 改变：即使没有机器学习背景的使用者，也能凭直觉判断文本可信度，大幅提升检测效率。

3. **跨采样策略的鲁棒性**  
   - 之前的做法：检测模型往往针对特定采样方式调参，换一种采样就失效。  
   - 本文做法：通过统计“高置信词比例”“低熵区块”等通用特征，能够捕捉到不同采样方法共同的生成痕迹。  
   - 改变：检测系统在 top‑k、 nucleus、温度调节等多种生成设置下仍保持较高召回率。

4. **人类实验验证提升**  
   - 之前的做法：多数检测研究只报告机器指标，缺少对实际使用者的评估。  
   - 本文做法：设计了对比实验，让未受过训练的参与者在有无 GLTR 标注的情况下辨别真假文本。  
   - 改变：检测率从 54% 提升到 72%，证明可视化标注对普通人有实质帮助。

### 方法详解
**整体思路**：把待检测的文本喂进一个预训练的大语言模型（如 GPT‑2），让模型输出每一步的完整概率分布；随后对每个词计算一系列统计量；最后把这些数值映射成颜色，生成一份交互式报告。整个流程可以概括为四步：

1. **概率获取**  
   - 输入完整文本，模型逐词预测下一个词的概率分布。这里使用的模型不必与生成文本的模型相同，只要足够强大即可。  
   - 类比：把模型当成“词典”，每次给出所有可能词的“可信度”列表。

2. **统计特征计算**  
   - **词排名**：在模型给出的概率列表中，找出实际出现词的排序位置。  
   - **概率阈值**：记录该词的原始概率值，用来判断它是否属于 top‑k 或 top‑p 范围。  
   - **熵**：对当前分布计算熵，反映模型对下一个词的不确定程度。  
   - **典型性**：比较词的负对数概率与整体分布的平均负对数概率，得到一个“普通度”。  
   - 这些特征在每个位置独立计算，形成一条统计序列。

3. **颜色映射与阈值设定**  
   - 预设三档颜色：绿色（词排名前 10%），黄色（前 10%–50%），红色（后 50%）。  
   - 同时提供熵和典型性的热度图，低熵/低典型性区域会被标记为红色，提示可能的机器痕迹。  
   - 用户可以切换不同阈值（如 top‑k=40、top‑p=0.9），系统会实时更新颜色。

4. **交互式展示**  
   - 前端页面把原文与颜色标记叠加，用户可以悬停查看每个词的具体数值。  
   - 还提供整体统计摘要，例如“超过 70% 的词位于前 10%”，帮助快速判断。  
   - 设计上强调简洁：不需要用户懂概率，只要看颜色强度即可。

**最巧妙的点**：作者发现即使使用与生成模型不同的语言模型，统计指纹仍然显著，因为所有大模型在语言分布上高度相似。这一点让检测工具不必针对每个新模型重新训练，极大提升了实用性。

### 实验与效果
- **实验材料**：从公开的新闻、维基百科等语料库抽取真实人类写作文本；使用 GPT‑2、GPT‑Neo 等模型在不同采样策略（top‑k=40、top‑p=0.9、温度 0.7）下生成对应的机器文本。  
- **人类辨识实验**：邀请 60 名未接受过机器学习训练的参与者，分别在仅阅读原文和在 GLTR 标注下阅读两组文本，要求判断其来源。  
  - 结果显示：没有任何辅助时，平均辨识率为 54%；加入 GLTR 可视化后提升至 72%。  
- **对比基线**：与仅使用词频统计的简单检测器相比，GLTR 在同样数据集上提升约 15% 的准确率。  
- **消融分析**：去掉熵或典型性热图会导致整体辨识率下降约 5%，说明多维统计共同贡献了检测效果。  
- **局限性**：作者指出，当生成模型使用极低温度（几乎确定性输出）或采用后处理（如拼写纠错）时，统计指纹会被削弱，检测难度上升。论文未给出对抗性生成的具体数值，只是说明在极端情况下效果会下降。

### 影响与延伸思考
GLTR 公开后迅速成为媒体、学术和企业检测机器生成文本的首选工具，GitHub 上的星标和 fork 数量持续攀升。它的思路——“用语言模型自身的概率分布做指纹”——直接催生了后续的 **DetectGPT**、**OpenAI Text Detector** 等更高级的检测方案，这些工作在 GLTR 的基础上加入了对抗训练和更细粒度的特征工程。与此同时，研究者也开始探索 **水印（watermark）** 技术，试图在生成阶段植入可检测的统计信号，以便后续更可靠地识别。想进一步了解，可以关注以下方向：  
- 基于大模型内部激活的检测方法（如 **LogitLens**）  
- 对抗生成模型的鲁棒检测（如 **Adversarial Text Detection**）  
- 生成文本的法律与伦理框架，尤其是对检测工具的监管需求。  

### 一句话记住它
GLTR 用语言模型的“词排名+熵”指纹，把机器生成的痕迹变成颜色，让普通人也能“一眼辨真伪”。
